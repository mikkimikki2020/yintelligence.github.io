Binary Search Tree

Video + Slide 124~ 168

树是一种分层数据结构，存储具有父子关系的数据。 数据存储在节点node中，而节点之间的关系由边edge表示。 由于每个节点都有一个父节点（除了最上面的节点称为根节点root）并且每个节点可以有多个子节点，因此这种结构类似于倒置树。

在图中，从节点 A 到节点 B 的箭头线表示 A 是 B 的父节点，因此 B 是 A 的子节点。没有任何子节点的节点称为叶节点leaf node。

当从根节点开始并在树中向下直到到达目标节点时，就在树中形成了一条路径path。

例如，在图中，根节点 A 和节点 I 之间的路径由节点 A、D、H、I 组成。 树中节点的深度depth是根和节点之间路径的长度。树的深度是树的节点深度的最大值。

这里定义的树是层次结构，即树中的每个节点都可以充当由该节点和所有子节点组成的较小树的根节点。 这棵较小的树称为子树subtree。

通常，一棵树在每个节点中可以有可变数量的子节点。 每个节点最多有两个孩子的树称为二叉树binary tree，每个节点最多有三个孩子的树称为三叉树ternary tree。

Basic operations

// the class for implementing a node in the tree .
// contains a value , a pointer to the left child and
// a pointer to the right child

public class TreeNode implements Comparable
{
    private Comparable value;
    private TreeNode leftNode;
    private TreeNode rightNode;
    
    public TreeNode ( Comparable v )
    {
        value = v ;
        leftNode = null ;
        rightNode = null ;
    }
    
    public TreeNode (Comparable v ,TreeNode left,TreeNode right){
        value = v ;
        leftNode = left ;
        rightNode = right ;
    }
    
    public TreeNode getLeftTree ()
    {
        return leftNode ;
    }
    
    public TreeNode getRightTree ()
    {
        return rightNode ;
    }
    
    public Comparable getValue ()
    {
        return value ;
    }
    
    public int compareTo ( Object o )
    {
        return value.compareTo(((TreeNode) o).value);
    }
}

如上所述，树将是一种递归结构，其中每个节点都可以被视为较小子树的根。 因此，最简单的方法是首先创建一个小的帮助类来表示一个节点，然后再定义一个使用这个 TreeNode 类的树类。 这个 TreeNode 类存储节点的值，一个指向左子树根的指针（这也是一个 TreeNode）和一个指向右子树根的指针（也是一个 TreeNode）。

Searching for an element in a binary search tree

由于二叉搜索树的性质是对于每个节点 K，K 的左子树的所有节点都小于 K，而 K 的右子树的所有节点都大于 K，因此可以很容易地搜索树中的元素 递归定义。 在二叉搜索树中搜索元素时可能存在四种情况：

  1. 树是空的，所以元素不存在于树中。

  2. 树的根等于元素，所以我们在树中找到了元素。

  3. 树的根比元素大。 这意味着如果元素是树的一部分，它应该在左子树中

    所以，我们应该继续在左子树中搜索。

  4. 树的根比元素小。这意味着如果元素是树的一部分，它应该在右子树中。

    所以，我们应该继续在右子树中搜索。

这四种情况一一翻译成如下代码，用于在二叉搜索树中搜索给定元素：

private boolean findNode ( Comparable element,TreeNode current)
{
    if( current == null ) return false ;
    else if ( element . compareTo ( current . value ) == 0)
        return true ;
    else if ( element . compareTo ( current . value ) < 0)
    {
        return findNode ( element , current . getLeftTree ());
    }
    else return findNode ( element , current . getRightTree ());
}

public boolean find (Comparable element)
{
    return findNode (element , root);
}

Inserting an element in a binary search tree

插入操作有点复杂，因为我们不能在树中的任何位置插入元素，以免违反二叉搜索树的条件。 与 find 方法类似，我们也递归地定义了 insert 方法，并区分以下五个条件：

  1. 树是空的，所以为了将新元素添加到树中，我们需要让 root 指向它。
  2. 如果根等于我们要插入的元素，则该元素已经在树中，所以没有什么可做的。
  3. 如果要添加到树中的项小于根节点，则应将其插入到左子树中的某处。因此，我们在左子树上递归调用 insert 方法。
  4. 如果要添加到树中的项大于根节点，则应将其插入到右子树中的某处。因此，我们在右子树上递归调用 insert 方法。
  5. 然而，通过这种方式，由于多次递归调用 insert 以将元素插入右子树或左子树，我们可能会在空子树上调用 insert 方法。在这种情况下，我们应该将元素作为子元素插入到这个空子树的父元素之下。这要求我们在插入方法期间知道任何节点的父节点。因为通常不可能从树数据结构中知道节点的父节点，所以我们在 insert 方法中添加了一个额外的参数来跟踪当前节点的父节点。
public void insert ( Comparable element )
{
    insertAtNode (element, root, null);
}

private void insertAtNode (Comparable element,
TreeNode current, TreeNode parent)
{
    // if the node we check is empty
    if(current == null)
    {
        TreeNode newNode = new TreeNode (element);
        // the current node is empty , but we have a parent
        if(parent != null)
        {
            // do we add it to the left ?
            if( element.compareTo (parent.value) < 0)
            {
                parent.leftNode = newNode ;
            }
            // or do we add it to the right ?
            else
            {
                parent.rightNode = newNode ;
            }
        }
        
        // the current node is empty and it has no parent ,
        // we actually have an empty tree
        else root = newNode ;

    else if( element . compareTo( current.value ) == 0)

}
    // if the element is bigger than current , go right
    else insertAtNode (element,current.getRightTree() ,current );
}

4. The print method you now have, is actually an example of a more global method to traverse the tree. Consider the following code:

public void traverse (TreeAction action)
{
    Queue t = new Queue ();
    t. push (root);
    while (! t. empty ())
    {
        TreeNode n = (TreeNode) t.Pop();
        action.run(n);
        if( n.getLeftTree() != null)
            t.push (n.getLeftTree());
        if( n.getRightTree() != null)
            t.push (n.getRightTree ());
    }
}

这个动作是遍历树并在每个元素上调用“TreeAction”类的“run”方法。 TreeAction 类是一个抽象类，只有 run 方法：

public abstract class TreeAction
{
    public abstract void run ( Tree . TreeNode n );
