Hypertext Markup Language 超文本标记语言 (HTML)

HTML is an application of the Standard Generalized Markup Language 标准通用标记语言 (SGML)

  • simple human-readable markup language with a number of markup tags that can be manipulated by any text editor

    简单的，人类可读的标记语言markup language，带有许多可由任何文本编辑器操作的标记标签markup tags

Various markup tags to define the structure and presentation of a HTML document (webpage)

用于定义HTML 文档（网页）的结构和表示的各种标记标签

  • root element (<html>), header (<head>), body (<body>) and title (<title>)
  • headings (<h1>, ..., <h6>) and paragraphs (<p>)
  • tables (<table>, <tr> and <td>) and lists ( <ol>, <ul> and <li>)
  • images (<img>)

w3schools provides a detailed list of tags, tags can be nested 嵌套 and attributes (key/value pairs) can be added to a tag

Document Object Model 文档对象模型 (DOM)

HTML documents are transformed into a Document Object Model (DOM) by the browser

HTML 文档被浏览器转换为文档对象模型 (DOM)

  • tree of elements (and attributes) with textual content

    具有文本内容的元素（和属性）树

  • HTML DOM defines objects and properties for HTML elements

    HTML DOM 为 HTML 元素定义对象和属性

    • document node, element nodes, text nodes, attribute nodes and comment nodes

      文档节点、元素节点、文本节点、属性节点和注释节点

  • standard to create, read, update and delete HTML elements

    创建、读取、更新和删除 HTML 元素的标准

Hyperlinks to connect different HTML documents

  • only unidirectional embedded hyperlinks are supported

    仅支持单向嵌入超链接

History of HTML 历史

  • HTML 1.0 (1993), prototype already in 1991

  • HTML 2.0 (1995)

    at that time the Netscape Navigator offered much more functionality than the HTML standard

    当时 Netscape Navigator 提供了比 HTML 标准更多的功能

    • "browser war" between Netscape and Internet Explorer

      Netscape 和 Internet Explorer 之间的“浏览器战争”

  • HTML 3.2 (1997)

    • first version that was developed exclusively by the Word Wide Web Consortium (W3C)

    • introduced tables 表

    • introduced a lot of new elements for the visual appearance of a document (that was not the original idea of HTML!)

      为文档的视觉外观引入了许多新元素（这不是 HTML 的最初想法！）

      • e.g. <font> element or color attribute
      • most elements now deprecated and cascading style sheets should be used

  • HTML 4.0 (1997) and HTML 4.01 (1999)

    • internationalisation (Unicode) 国际化
    • introduction of Cascading Style Sheets (CSS)

  • In 1998 the W3C decided to not further evolve HTML!

  • XHTML 1 (2000) and XHTML 1.1 (2001)

    • XML version of HTML (draconian error handling)

      HTML 的 XML 版本（严格的错误处理）

  • XHTML 2.0 (never finished, discontinued in 2009)

    • revolutionary changes → breaking backwards compatibility

      革命性变化 → 打破向后兼容性

  • Web Hypertext Application Technology Working Group (WHATWG) founded in 2004 (led by Ian Hickson)

    • Web Forms 2.0 and Web Applications 1.0 → HTML5

  • In 2006 the W3C decided to work on HTML again

    • based on WHATWG's Web Applications specification

  • HTML5 specification is currently developed simultaneously by the WHATWG and the W3C HTML Working Group

    • HTML

      • Living Standard, WHATWG

    • HTML5

      • A Vocabulary and Associated APIs for HTML and XHTML

        HTML 和 XHTML 的词汇表和相关 API

  • HTML5 is a W3C Recommendation since October 2014

    • at least two browsers fully implementing the standard

Problems 问题

  • Mix of content, structure and presentation

    内容、结构和演示的混合

    • no separation of concerns

      没有关注点分离

    • structure of content defines the presentation in the browser

      内容结构定义了浏览器中的呈现

  • "Forgiving" browsers weaken the standard

    • an HTML document with errors (e.g. missing tags etc.) will still be rendered without any error messages

      带有错误（例如缺少标签等）的 HTML 文档仍将被呈现而没有任何错误消息

    • HTML documents can be checked against the standard

      可以根据标准检查 HTML 文档

    • most existing websites (>99%) contain errors

      Exercise: can you find a webpage without any errors?

HTML Markup 标记

  • Some elements have been added

    添加了一些元素

    • structural elements such as <article>, <section>, <header>, <footer> or <nav>
    • media elements including <video>, <audio> or <embed>
    • a <canvas> drawing element

  • Other elements have been removed

    其他元素已被删除

    • <big>, <font>, <strike>, <u>, ...

  • New form functionality (originally Web Forms 2.0)

    新的表单功能（最初是 Web Forms 2.0）

    • form elements such as or <input> or <output>

    • input types such as date, color, email, tel, range, ...

    • native functionality for client-side validation (without scripting)

      客户端验证的本机功能（无脚本）

HTML5 forms simplify the client-side validation of input fields

HTML5 表单简化了输入字段的客户端验证

      改进语言以更轻松地编写 Web 应用程序

  • Utility 利用率

    • solve real problems (pragmatic approach)

      解决实际问题（务实的方法）

    • separation of content and presentation (CSS)

      内容和表现分离 (CSS)

  • Interoperability 互操作性

    • interoperable browser behaviour

      可互操作的浏览器行为

    • identical error handling across browsers resulting in the same DOM tree for broken HTML

      跨浏览器的相同错误处理导致损坏的 HTML 的相同 DOM 树

  • Universal Access 通用接入

    • features should preferably work across different platforms (cross-platform), devices and media

      功能应该最好跨平台、设备和媒体工作

    • design features should be accessible to users with disabilities

  • Simple is better 简单更好

    • HTML5 APIs

    • plug-ins can be disabled or blocked (e.g. Adobe Flash on iPad)

      可以禁用或阻止插件（例如 iPad 上的 Adobe Flash）

localStorage and sessionStorage JavaScript objects to store data (key/value) on the client

Drag and Drop 拖放

Drag items and drop them anywhere in the browser

  • define draggable elements via the draggable attribute

    通过draggable属性定义可拖动元素

  • define elements that can accept drops

    定义可以接受放置的元素

  • exchange information via the dataTransfer object

    通过dataTransfer对象交换信息

Items can also be drag and dropped from the desktop to the browser

项目也可以从桌面拖放到浏览器

<!DOCTYPE html>
<html>
    <head>
    // style sheet information for the div elements
        <script>
        function drag(event) {
        	event.dataTransfer.setData("text/plain", event.target.id);
        }
        function drop(event) {
        	event.preventDefault(); // avoid default behaviour of opening as link
        	var data=event.dataTransfer.getData("text/plain");
            event.target.appendChild(document.getElementById(data));
        }
        function allowDrop(event) {
        	event.preventDefault();
        }
        </script>
    </head>
    <body>
        <div id="div1" ondrop="drop(event)" ondragover="allowDrop(event)">
        	<img src="wise.gif" draggable="true" ondragstart="drag(event)" id="d1">
        </div>
        <div id="div2" ondrop="drop(event)" ondragover="allowDrop(event)"></div>
    </body>
</html>

Web Workers 网络工作者

When executing JavaScript in an HTML page, the page becomes non-responsive until the script is finished

在 HTML 页面中执行 JavaScript 时，页面将变为无响应，直到脚本完成

  • Web Workers can be used to execute JavaScript code in a background process (thread)

    Web Workers 可用于在后台进程（线程）中执行 JavaScript 代码

    • to avoid the complexity of multi-threaded programming, Web Workers have independent JavaScript contexts and can only interact with each other via event-driven message passing

      为了避免多线程编程的复杂性，Web Workers 拥有独立的 JavaScript 上下文，并且只能通过事件驱动的消息传递相互交互

Note that Web Workers have no access to window.document (web page or DOM API)

请注意，Web Workers 无权访问 window.document（网页或 DOM API）

  • "clean up" Web Workers via terminate() method

    通过terminate()方法“清理”Web Workers

    var worker = new Worker("myScript.js");

		alert("The worker received this: " + event.data);

        // add a listener to receive messages from the web worker
        worker.addEventListener("message", handle, false);
        function handle(e) {
        // process message from the web worker
        }

        worker.postMessage("Hello worker!");
</script>

WebSocket 网络套接字 API

Bidirectional, full-duplex socket connection between the server and browser for real-time web applications (low latency) with asynchronous partial updates

服务器和浏览器之间的双向全双工套接字连接，用于具有异步部分更新的实时 Web 应用程序（低延迟）

  • server-initiated updates become possible!

    服务器启动的更新成为可能！

  • client and server upgrade from the HTTP protocol to the WebSocket protocol (initial HTTP handshake)

    客户端和服务器从 HTTP 协议升级到 WebSocket 协议（初始 HTTP 握手）

    • via Connection: Upgrade and Upgrade: websocket HTTP header fields

    • browser as well as server have to support the Web Socket protocol

      浏览器和服务器都必须支持 Web Socket 协议

  • reduced "overhead" since no HTTP headers

    由于没有 HTTP 标头，因此减少了“开销”

  • no longer necessary to do any polling or long polling

    不再需要进行任何轮询或长轮询

    • faster since in the polling approach the server can send nothing while a client request is transmitted

      更快，因为在轮询方法中，服务器在传输客户端请求时不能发送任何内容

      轮询（Polling）是一种CPU决策如何提供周边设备服务的方式，又称“程控输入输出”（Programmed I/O）。

      轮询法的概念是：由CPU定时发出询问，依序询问每一个周边设备是否需要其服务，有即给予服务，服务结束后再问下一个周边，接着不断周而复始。

  • similarly an EventSource object can be used if only the server has to push information (server-sent events)

    同样，如果只有服务器必须推送信息（服务器发送的事件），则可以使用 EventSource对象

<script>
    var socket = new WebSocket("ws://chat-server.com");
    socket.onopen = function(e) { alert("Opened connection ..."); };
    socket.onmessage = function(e) {
    var message = JSON.parse(e.data));
    alert("Message received.");
    ...
    };
    socket.onclose = function(e) { alert("Closed connection."); };
    socket.onerror = function(e) { alert("WebSocket Error" + e); };
    socket.send("Hellow World");
    // connection stays open and server can send multiple things
    ...
    socket.close();
</script>

Geolocation 地理定位 API

  • Standard interface for accessing geographical location information on the client device

    • transparent access to different location information sources

      透明访问不同的位置信息源

      • GPS, GSM cells, IP address, RFID, Wi-Fi connection etc.
  • Firefox uses the Google Location Service as default lookup service

    • send IP address and information about nearby wireless access points to the Google Location Service and an approximate location will be computed

      将 IP 地址和有关附近无线接入点的信息发送到 Google 位置服务，然后将计算出大致位置

  • JavaScript access to the Geolocation API

    • access via the geolocation child object of the navigator object

      通过导航器对象的地理定位子对象访问

function showPosition(position) {
	alert(position.coords.latitude + " " + position.coords.longitude);
}

function showError() {
	alert("Your current position cannot be computed!");
}

navigator.geolocation.getCurrentPosition(showPosition, showError,
{timeout:10000});

we can also continuously monitor the client's position

我们还可以持续监控客户的位置

navigator.geolocation.watchPosition(showPosition);

Offline Web Applications 离线网络应用

  • Application cache is used for

    应用程序缓存用于

    • offline browsing where users can use the web application while they are offline

      离线浏览，用户可以在离线时使用 Web 应用程序

    • increased performance due to cache hits and reduced server load

      由于缓存命中和服务器负载降低而提高了性能

      A cache hit is a state in which data requested for processing by a component or application is found in the cache memory.

      高速缓存命中是在高速缓存存储器中找到组件或应用程序请求处理的数据的状态。

      It is a faster means of delivering data to the processor, as the cache already contains the requested data.

      这是将数据传送到处理器的更快方法，因为缓存已包含请求的数据。

  • Managed via a cache manifest (on every page)

    通过缓存清单管理（在每个页面上）

<!DOCTYPE html>
<html lang="en" manifest="myApp.appcache">...</html>

we can adapt behaviour if a page is not visible (e.g. minimised)

  • change update frequency (e.g. checking for emails) to save resources

    更改更新频率（例如检查电子邮件）以节省资源

  • pause video or game

    }

Vibration 振动 API

Provide simple tactile feedback 提供简单的触觉反馈

  • simple vibration or pattern of on/off pulses

    简单的振动或开/关脉冲模式

  • feedback for mobile phones and other devices

    对手机和其他设备的反馈

  • for devices not supporting vibrations, the methods have no effect

    对于不支持振动的设备，该方法无效

// single vibration for 300ms
navigator.vibrate(300);
// pattern with pauses of 50ms
navigator.vibrate([300, 50, 300, 50, 1000]);
// ongoing vibrations can be cancelled at any time by calling the vibrate
method with a value of zero
navigator.vibrate(0);
