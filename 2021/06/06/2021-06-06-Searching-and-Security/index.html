Background 1 + 2 + Question + Video

Searching and PageRank

The simplified PageRank algorithm is defined as: \[ PR(u)=\sum_{v∈B_{u}}\frac{PR(v)}{L(v)} \] with:

  • PR（u）, the PageRank of page u
  • Bu, the set of pages that link to page u
  • L（v）, the amount of outgoing links of page v

So to calculate the PageRank of a given page u, we need to do the following:

  1. Get all pages that link to page u, the resulting set of pages is Bu
  2. For every page v in Bu, calculate \(\frac{PR(v)}{L(v)}\)
  3. Add the results together

The PageRank computations require several passes, called “iterations”, to adjust approximate PageRank values to more closely reflect the theoretical true value. During each iteration, the PageRank values from the previous iterations are used. So in order to calculate the PageRank for a graph, you need to do the following steps:

  1. Start by assigning every page with an initial PageRank of \(\frac{1}{1\#pageswhere}\) #pages is the amount of pages in the graph
  2. Calculate the new PageRank for every page in the graph
  3. Go to step 2

For more information, see here

Note: If a page has no outgoing links, we have a s o-called “sink”, which will accumulate most of the PageRank over time. To fix this, we assume that these sinkns are connected to every other nodes during the calculation.

Calculate PageRank for Grouped Sites

Consider the following graph:

Since page D is an internal sink, most of our website’s PageRank is accumulated there. However, by linking to external sites from D we are “leaking” a lot of PageRank away from our website, which is not desirable. What can we do to minimize the leakage? Can it be achieved without removing any existing links?

Solution

By adding links from D to our internal pages, we are diluting the amount that is leaked.

  • originally half goes to external site 1 （ex1）, half goes to external site 2 （ex2）
  • if we add one internal link （e.g. from D to A）, 1/3 goes to ex1, 1/3 goes to ex2, 1/3 is returned to one of our pages
  • if we add another internal link, 2/4 is lost to external pages, 2/4 is returned
  • if we add another internal link, 2/5 is lost to external pages, 3/5 is returned
  • ...

One could also add additional pages, and link to those from D.

Security

Cross-site Scripting （XSS）

Image a web page with the following PHP code:

<html>
<body>
    
<?php$name = "nobody";
if(isset($_GET['name']) && $_GET['name'] != ""){
    $name = $_GET['name'];
}
echo "Hello, ".$name;
?>
    
</body>
</html>

This page takes the “name” GET parameter from the URL and displays it. For instance, if we would go to page.php? name=Kugou, the page would display “Hello, Kugou” with the following outputted HTML

<html>
<body>
Hello, Kugou
</body>
</html>

You can see that it just outputs the value of the name parameter right into the HTML. Now imagine that we modified the url to this:

page.php?name=<script>alert(’test’);</script>

The page will take the value and put it in the HTML like this:

<html>
<body>
    Hello,
    <script>
        alert('test');
    </script>
</body>

As you can see, we can sometimes abuse an URL to inject JavaScript into the page. This is called a cross-site scripting （XSS） vulnerability.

  • Modify the site （CSS, HTML, ...） to show other information （e.g. showing fake news on a reputable news site）
  • Execute actions as the logged in user （e.g. post a message）
  • Create a script that intercepts the password that you enter and performs an AJAX request that sends it to your server
  • Create a script that sends all the cookies to your server （cookie stealing）
