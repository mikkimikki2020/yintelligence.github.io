Security Aspects

  • Authenticity 真实性

    • knowing the sender or receiver of data

      了解数据的发送者或接收者

      • who is trying to access data on a web server
      • who is offering a service
      • who sent an email
      • …

  • Privacy 隐私

    • keeping information private 保密信息

    • protect credit card information that is sent to a server

      保护发送到服务器的信用卡信息

    • protect information sent in emails

      保护通过电子邮件发送的信息

    • …

  • Integrity

    • ensuring that information is not changed when transferred

      确保信息在传输时不会改变

HTTP Authentication

  • Native authentication functionality offered by HTTP

    HTTP 提供的本机身份验证功能

    • instead of directly sending a response for a given request, the server can always respond with an authentication challenge (401 status code)

      服务器可以始终以身份验证质询（401 状态代码）进行响应，而不是直接为给定请求发送响应

  • HTTP is extensible to support different authentication protocols and offers the following two standard protocols

    HTTP 可扩展以支持不同的身份验证协议，并提供以下两种标准协议

    • basic access authentication 基本访问认证
      • simple Base64 encoding of the string
    • digest access authentication 摘要访问认证

  • Protected resources can be grouped in security realms with different sets of authorised users or groups of users

    受保护的资源可以在具有不同授权用户集或用户组的安全领域中分组

Basic Access Authentication 基本访问认证

Base64 Encoding

  • Base64 encoding can be used to represent binary data in a portable format (alphabet)

    Base64 编码可用于以可移植格式（字母表） 表示二进制数据

    • used by Media Types (MIME) for content transfer encoding

      媒体类型 (MIME) 用于内容传输编码

    • used to embed binary data in XML files (e.g. in XML-RPC)

      用于在 XML 文件中嵌入二进制数据（例如在 XML-RPC 中）

    • note that Base64 encoded data needs more space

      注意 Base64 编码的数据需要更多空间

  • Takes a sequence of bytes (8-bit) and breaks it into 6-bit chunks

    获取字节序列（8 位）并将其分解为 6 位块

    • padding with 0s to make it a multiple of 24 (LCM of 6 and 8)

      用 0填充以使其成为 24 的倍数（LCM 为 6 和 8）

    • complete 6-bit padding chunks are represented by the special character '='

      完整的 6 位填充块由特殊字符“=”表示

Example

  • Each 6-bit chunk is then represented by a character from a 64-character alphabet

    然后每个 6 位块由 64 个字符的字母表中的一个字符表示

  • Let us encode the string 'No' to Base64

    • padding to 24 bit
    • lookup of 6-bit chunks in index table
    • use '=' for completely padded 6-bit chunks

  • Example configuration for an Apache HTTP Server

    Apache HTTP 服务器的示例配置

      或者将信息添加到 httpd.conf

Problems of Basic Access Authentication 基本访问认证问题

  • Basic access authentication is not secure 基本访问认证不安全

    • username and password are sent almost in "cleartext"

      用户名和密码几乎以“明文” 形式发送

      • Base64 value can be very easily decoded

        Base64 值可以很容易地解码

    • easy to do replay attacks 容易进行重放攻击

      • simply reuse the Base64-encoded username and the password

        只需重复使用 Base64 编码的用户名和密码

  • Potential solutions 潜在的解决方案

    • combine the basic access authentication with an encrypted data transfer (e.g. via TLS/SSL)

      将基本访问身份验证与加密数据传输相结合（例如通过 TLS/SSL）

      • does not necessarily prevent replay attacks

        不一定能防止重放攻击

    • use of alternative digest access authentication

      使用替代摘要访问身份验证

Digest Access Authentication 摘要访问认证

  • Password is no longer sent in cleartext

    密码不再以明文形式发送

    • only a one-way digest that is computed out of the password (one-way hash function) is sent to the server

      只有从密码中计算出的单向摘要（单向哈希函数）被发送到服务器

    • Message Digest #5 (MD5) is a popular digest function

  • What about digest replay attacks?

    • server sends a special token (nonce) that changes frequently

      服务器发送一个经常变化的特殊令牌（nonce）

    • client adds the nonce to the password before computing the MD5

      客户端在计算 MD5 之前将随机数添加到密码中

      • any changes of the nonce result in changes of the digest which helps to prevent replay attacks

        once 的任何变化都会导致摘要的变化，这有助于防止重放攻击

// Computed response based on MD5
h1 = MD5(username:realm:password)
h2 = MD5(httpMethod:requestedURI)
response = MD5(h1:nonce:h2)

  • The Authorization-Info: nextnonce="..." is used to send the next nonce in advance 用于提前发送下一个随机数

    • client can send the computed hash value already with the original request (pre-emptive authorisation)

      客户端可以发送已计算的哈希值与原始请求（抢先授权）

  • The quality of protection (qop) field is used to negotiate different protection mechanisms 用于协商不同的保护机制

    • auth

      • authentication

    • auth-int

      • authentication and message integrity protection

        身份验证和消息完整性保护

      • add an MD5 of the body

Transport Layer Security 传输层安全 (TLS)

  • Cryptographic protocol to ensure secure network communication

    加密协议，以确保安全的网络通信

    • successor of the Secure Socket Layer (SSL) protocol

      安全套接字层 (SSL)协议的继任者

    • situated at the TCP/IP Application Layer or OSI Presentation Layer

      位于 TCP/IP应用层或 OSI表示层

  • Types of authentication 类型

    • unilateral authentication 单边认证
      • only server authentication 仅服务器身份验证
    • mutual authentication 相互认证
      • client and server authentication 客户端和服务器验证

Cryptography 密码学

  • In cryptography a cipher (coding scheme) is used in combination with a key to create a ciphertext out of a plaintext

    在密码学中，密码（编码方案）与密钥结合使用以从明文中创建密文

  • Cryptanalysis tries to get information out of the ciphertext without having access to the secret information (key)

    密码分析试图从密文中获取信息，而无需访问秘密信息（密钥）

Symmetric Key Cryptography 对称密钥密码术

  • A symmetric key cipher uses the same key for the encoding and decoding of a plaintext message

    对称密钥密码使用相同的密钥用于编码和解码一个明文消息的

  • Many existing symmetric key ciphers 许多现有的对称密钥密码

    • DES, Triple DES, Blowfish, Rijndael/AES, ...

  • The algorithms are often common knowledge and the key is the only secret thing

    算法通常是常识，密钥是唯一的秘密

    • key has to be kept secret 密钥必须保密

  • Brute force attack (enumeration attack) tries all keys

    蛮力攻击（枚举攻击）尝试所有密钥

  • The key length defines the number of potential keys

    密钥长度定义了潜在密钥的数量

    • e.g. 128 bit key considered safe today

      • can change with more powerful machines

        可以用更强大的机器改变

  • One problem of symmetric key cryptography is that we have to secretly share the common key before we can exchange any messages

    对称密钥密码学的一个问题是我们必须秘密地共享公共密钥，然后才能交换任何消

    • this has to be repeated with different keys for any two partners willing to establish a secret communication

      对于愿意建立秘密通信的任何两个合作伙伴，必须使用不同的密钥重复此操作

    • how should we establish the exchange over the Internet?

      我们应该如何在互联网上建立交换？

      • initially only an insecure channel is available

        最初只有一个不安全的通道可用

    • where should we secretly store all those keys?

      我们应该把所有这些密钥秘密存储在哪里？

Public Key (Asymmetric) Cryptography 公钥（非对称）密码术

  • Instead of a single key, public key cryptography uses an asymmetric pair of keys

    公钥密码术不是使用单个密钥，而是使用非对称密钥对

    • publicly available key for the encoding

      公开可用的编码密钥

    • secret key for the decoding

      用于解码的密钥

  • Each party has only a single public key which is used by everybody to encode messages to this party

    每一方只有一个公钥，每个人都使用它来编码发送给该方的消息

    • only the receiver can decode message with their private key

      只有接收者可以用他们的私钥解码消息

  • Public key cryptography can be used to establish secure Internet connections to any computer around the world without having to secretly share a key beforehand

    公钥加密可用于与世界各地的任何计算机建立安全的 Internet 连接，而无需事先秘密共享密钥

  • An asymmetric public key cipher has to ensure that an attacker cannot compute the private key based on any information they can intercept

    非对称公钥密码必须确保攻击者无法根据他们可以拦截的任何信息计算私钥

    • public key 公钥

    • ciphertext (with corresponding plaintext)

      密文（带有对应的明文）

      • can easily be created by any party by using the public key

        任何一方都可以使用公钥轻松创建

  • A well-known public key algorithm is the RSA cipher

    著名的公钥算法是RSA 密码

  • A drawback of asymmetric public key cryptography is the fact that the algorithms are much slower than symmetric ciphers

    非对称公钥密码术的一个缺点是算法比对称密码慢得多

  • Hybrid solutions combine public key with symmetric key cryptography

    混合解决方案将公钥与对称密钥加密相结合

    • the public key encryption is only used in the setup phase to securely exchange a pair of symmetric keys

      公钥加密仅在设置阶段使用，以安全地交换一对对称密钥

    • afterwards a secure channel is established based on the symmetric keys

      然后根据对称密钥建立安全通道

  • Security of public key cryptography?

    • new developments (e.g. quantum computing) might break public key cryptography

      新的发展（例如量子计算）可能会破坏公钥密码学

RSA Cipher (Rivest, Shamir and Adleman)

  • Public-key cipher that can be used for encryption as well as signing

    可用于加密和签名的公钥密码

    • published in 1978 by Rivest, Shamir and Adleman while they were at MIT

  • The public and private keys are generated based on two large distinct prime numbers

    公钥和私钥是基于两个不同的大素数生成的

    • the potential attacker will know about the product of the two prime numbers but nothing about the numbers themselves

      潜在的攻击者会知道两个素数的乘积，但对数字本身一无所知

    • use modular arithmetic for the encoding/decoding

      使用模块化算法进行编码/解码

    • as long as the attacker is not able to do a factorisation into the two prime numbers, RSA is assumed to be secure

      只要攻击者无法对两个素数进行因式分解，就认为 RSA 是安全的

Digital Signatures 数字签名

  • A digital signature can be used for two purposes 数字签名可用于两个目的

    • to prove the authenticity of a message

      证明消息的真实性

    • to guarantee that a message has not been changed during the transfer (integrity)

      保证消息在传输过程中没有被改变（完整性）

  • Sender creates a plaintext digest, encodes it with the private key and adds it as a signature to the message

    发件人创建一个明文摘要，用私钥对其进行编码并将其作为签名添加到消息中

    • the receiver creates the same digest and compares it with the decoded signature

      接收者创建相同的摘要并将其与解码的签名进行比较

  • Information about a person/company that is digitally signed by a certificate authority (CA)

    有关由证书颁发机构 (CA)数字签名的个人/公司的信息

    • owner's name 业主姓名
    • validity time 有效期
    • signature of the CA
    • owner's public key 所有者的公钥

HTTP Secure (HTTPS)

  • Secure version of HTTP

    • combines HTTP with asymmetric, symmetric and certificate based cryptography

      将 HTTP 与非对称、对称和基于证书的加密相结合

    • HTTP sent over TLS/SSL

      通过 TLS/SSL 发送的 HTTP

  • HTTPS protocol is selected by the https:// URL prefix

  • Browser connects to the HTTPS default port (port 443)

    浏览器连接到 HTTPS 默认端口（端口 443

    • Initial SSL handshake 初始 SSL 握手

      • negotiate protocol versions 协商协议版本

      • negotiate common cipher 协商通用密码

      • authentication 验证

      • generate temporary symmetric session keys

        生成临时对称会话密钥

Email Security 电子邮件安全

  • Emails are generally sent as unencrypted plain text

    电子邮件通常以未加密的纯文本形式发送

  • An email is stored on multiple intermediary servers before reaching its target

    电子邮件在到达目标之前存储在多个中间服务器上

    • relatively easy to intercept 比较容易拦截
    • would you also put anything you write in an email on a postcard?

  • Note that the sender of an email can easily be faked

    请注意，电子邮件的发件人很容易被伪造

  • If we want to fix these problems we have to use third-party tools such as Pretty Good Privacy (PGP)

    如果我们想解决这些问题，我们必须使用第三方工具，例如 Pretty Good Privacy (PGP)

    • privacy 隐私
      • strong encryption 强加密
    • authentication 验证
      • digital signatures 数字签名

Email SPAM 垃圾邮件

  • Abuse of an electronic messaging system (email) to deliver unwanted messages

    滥用电子信息系统（电子邮件）发送不需要的信息

  • A major part of all SPAM is sent by only a few hundred spammers

    只有几百个垃圾邮件发送者发送了所有垃圾邮件的主要部分

  • It is estimated that SPAM costs businesses more than 100 billion dollars per year

    据估计，SPAM 每年给企业造成的损失超过 1000 亿美元

  • SPAM is illegal in many countries and some spammers have already been sentenced to jail

    垃圾邮件在许多国家都是非法的，一些垃圾邮件发送者已经被判入狱

  • "Solutions"

    • SPAM filters 垃圾邮件过滤器
    • micropayments for emails 电子邮件的小额支付

  • Phishing attacks 网络钓鱼攻击

    • send emails that look like coming from an official authority and contain a request for sensitive data (e.g. password)

      发送看似来自官方机构并包含敏感数据（例如密码）请求的电子邮件

    • send emails with links to websites that look like official companies (e.g. your home bank)

      发送带有链接的电子邮件，链接到看起来像官方公司的网站（例如您的家庭银行）

  • Spammers often use botnets to send their SPAM

    垃圾邮件发送者经常使用僵尸网络发送他们的垃圾邮件

Botnets 僵尸网络

  • Computers infected by malicious software become part of a large botnet that can be remotely controlled

    被恶意软件感染的计算机成为可远程控制的大型僵尸网络的一部分

    • the largest botnets contain more than 1 million machines

      最大的僵尸网络包含超过 100 万台机器

  • An attacker can buy part of such a botnet to perform various harmful tasks including

    攻击者可以购买此类僵尸网络的一部分来执行各种有害任务，包括

    • the distribution of SPAM 垃圾邮件的分布
    • distributed denial of service attacks (DDOS)
    • 分布式拒绝服务攻击 (DDOS)
      • Distributed denial of service attacks are a very powerful weapon as it has for example been shown when Estonia was attacked in May 2007
      • 分布式拒绝服务攻击是一种非常强大的武器，例如在 2007 年 5 月爱沙尼亚遭到攻击时就表明了这一点
        • cannot easily be detected and filtered by firewalls since the traffic is created by many different machines
        • 由于流量是由许多不同的机器创建的，因此防火墙无法轻易检测和过滤

Firewalls 防火墙

Software and hardware firewalls introduce artifical "bottlenecks" that have to be passed by all the traffic

软件和硬件防火墙引入了所有流量必须通过的人为“瓶颈”

  • block specific ports 阻止特定端口

  • filter and block content 过滤和阻止内容

  • protect private intranets from incoming Internet traffic

    保护私有 Intranet 免受传入 Internet 流量的影响

    • often only a subnetwork (demilitarised zone) is connected to the Interne

      通常只有一个子网（非军事区）连接到互联网

Privacy 隐私

  • While users access information over the Internet, there is a continuous logging of their requests

    当用户通过 Internet 访问信息时，他们的请求会被持续记录

  • Each server stores information about clients who accessed specific resources

    每个服务器存储有关访问特定资源的客户端的信息

  • Data mining techniques can be used to combine this logging information and create user profiles

    数据挖掘技术可用于组合这些日志信息并创建用户配置文件

    • can for example be used for user-targeted advertising

      例如可用于针对用户的广告

  • Users also "deliberately" publish personal information

    用户还“故意”发布个人信息

    • e.g. on Facebook

  • Published information often cannot be easily deleted

    已发布的信息往往不容易删除

    • e.g. still accessible via Internet Archive

Web Log 网络日志

  • Log entry created every time a web server is accessed

    每次访问 Web 服务器时都会创建日志条目

  • A log entry typically contains information about

    日志条目通常包含有关

    • IP address of the requesting machine

      请求机器的 IP 地址

    • accessed URL 访问的 URL

    • request time 请求时间

    • refer link 参考链接

      • previous page accessed by the client

        客户访问的上一页

      • sent as part of the HTTP Request

        作为 HTTP 请求的一部分发送

    • browser type 浏览器类型

    • errors that occurred 发生的错误

    • ...

  • Web logs can be combined with other information

    网络日志可以与其他信息结合

    • e.g. login information can be used to reveal a user's identity

      例如，登录信息可用于揭示用户的身份

  • Refer link 参考链接

    • enables access to potentially private information

      允许访问潜在的私人信息

    • e.g. if previous request was an HTML form request using the GET method then all the data will be available as part of the URL

Web Log File Analysis 网络日志文件分析

  • Site owner can use various tools to analyse the log files 网站所有者可以使用各种工具来分析日志文件

    • e.g. Webalizer

  • How much information do we give away when accessing a website?

    我们在访问网站时会泄露多少信息？

  • What is happening with the logged data?

    记录的数据发生了什么？

    • combined with other information to reveal IP addresses?

      结合其他信息来揭示 IP 地址？

    • combined with log files from other sites?

      与来自其他站点的日志文件相结合？

      • user profiling 用户分析

    • intended use of data should be mentioned in the privacy policy

      隐私政策中应提及数据的预期用途

Cookies Revisited

  • Persistent cookies can be used to track a user over time

    持久性 cookie可用于随着时间的推移跟踪用户

    • similar to IP address but more precise

      类似于 IP 地址，但更精确

  • Third-party cookies can be used to build an anonymous user profile

    第三方 cookie可用于构建匿名用户配置文件

    • if a website contains elements that have to be accessed from another server (e.g. banner ads), then the server can set a cookie

      如果网站包含必须从另一台服务器访问的元素（例如横幅广告），则服务器可以设置 cookie

      • the third-party server creates a unique resource URL for every page on which the resource has been embedded

        第三方服务器为每个嵌入资源的页面创建一个唯一的资源 URL

      • the user can be tracked on any site that uses the same service (e.g. banner ads) and an anonymous user profile can be created

        可以在使用相同服务（例如横幅广告）的任何站点上跟踪用户，并且可以创建匿名用户配置文件

  • Cookies should not be used for authentication 不应用于身份验证

    • can be modified by a user to forge identity (cookie poisoning)

      可以被用户修改以伪造身份（cookie 中毒

Web Bugs 网络错误

  • User tracking based on the same idea as with third-party cookies

    基于与第三方 cookie 相同的想法的用户跟踪

    • Embed a small object (e.g. 1 pixel image) in a webpage and get informed every time the webpage is accessed

      在网页中嵌入一个小对象（例如 1 像素图像）并在每次访问网页时获得通知

    • request containing the IP address is sent to the server

      包含 IP 地址的请求被发送到服务器

  • The web bugs approach cannot only be used for webpages but also for other resources such as email, Word documents etc.

    Web bug 方法不仅可以用于网页，还可以用于其他资源，例如电子邮件、Word 文档等

    • if the user reads an email containing an embedded HTML web bug, the server knows when the email has been read but also gets information about the IP address of the mail client

      如果用户阅读了一封包含嵌入式 HTML 网络漏洞的电子邮件，服务器会知道电子邮件何时被阅读，但也会获取有关邮件客户端 IP 地址的信息

Other Services with Privacy Issues

  • Google Earth shows a lot of sensitive information

    谷歌地球显示大量敏感信息

    • e.g. military bases etc.

  • Google Street View shows not only streets and buildings but also citizens

    谷歌街景不仅显示街道和建筑物，还显示市民

    • privacy of individuals might be violated since they are shown at strange places or in weird situations
    • since the blurring of faces and number plates does not always work, some countries would like to stop the service

  • Many other free services from Google as well as other companies harvest personal information and use it, for example, for customer-targeted advertising

    Google 和其他公司提供的许多其他免费服务会收集个人信息并将其用于例如针对客户的广告

Google Analytics 谷歌分析

  • Very nice tool for web administrators to analyse their web traffic

    非常好的网络管理员分析网络流量的工具

    • easy to "install" over the Web 易于通过网络“安装”

    • website administrators have to add a piece of JavaScript code (tracking code) to their website

      网站管理员必须在他们的网站上添加一段 JavaScript 代码（跟踪代码）

      • similar to web bug approach shown earlier

        类似于前面显示的网络错误方法

    • Google gets information about site visitors

      Google 获取有关网站访问者的信息

  • While a user can normally choose to use a free service (e.g. Gmail) or not, the user has no choice when it comes to the tracking via Google Analytics

    虽然用户通常可以选择是否使用免费服务（例如 Gmail），但在通过 Google Analytics 进行跟踪时，用户别无选择

  • How save is the captured data? 捕获的数据如何保存？

    • what if somebody manages to steal the data?

      如果有人设法窃取数据怎么办？
