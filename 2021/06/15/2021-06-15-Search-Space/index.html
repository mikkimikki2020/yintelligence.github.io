Search strategies

An agent is in Arad, Romania, at the end of a touring holiday. It has a ticket to fly out of Bucharest the following day. The ticket is non-refundable, the agent's visa is about to expire and there are no seats available on flights for six weeks. Our holiday agent has adopted the goal of driving to Bucharest using the following map:

Draw the search tree for the following strategies:

  1. depth first search
  2. breadth first search
  3. uniform cost search
  4. A*

Solution

General search algorithms

add initial state to agenda;
	while agenda not empty and solution not found do
	remove the first node N from the agenda;
	if N is the goal then
		stop searching and return the solution;
	else
		find all successors of N and add them to the agenda;
	end
end

The outline of a general tree search algorithm is sketched above. What data structure must be used for the agenda to turn this outline into:

  1. depth first search
  2. breadth first search
  3. uniform cost search
  4. A*

For each of those search algorithms, start from the general algorithm above and add more details to it, such that it becomes the desired search algorithm.

Solution

  1. for depth first search, the agenda should operate as a stack: new nodes are added to the front of the agenda and taken from the front.
  2. for breadth first search, the agenda should operate as a queue
  3. for best first search , the agenda should be a priority queue with priorities f(n) = h(n)
  4. for A*, the agenda should be a priority queue with priorities f(n) =g(n) + h(n)

Search spaces

Consider the state space illustrated in the figure below.

  1. S is the initial state
  2. G1 and G2 are goal states
  3. Arcs show actions between states
  4. e.g., the successor function for state S returns {A, B}).
  5. Arcs are labelled with actual cost of the action (e.g., the action from S to A has a cost of 3)
  6. The numeric value in parentheses in each state is the state’s h-value (e.g., h(A) = 5).

Give the sequence of states that will be visited, together with the total cost of reaching the goal state, for (1) depth first search, (2) breadth first search and (3) A*. You should assume the following on the operational details of the algorithms:

  1. The algorithm does not check if a state is revisited, so there may be several nodes with the same state in the search tree.
  2. The algorithm terminates only when it selects a goal node for expansion, not when it is generated by the successor function.
  3. The successor function always orders states alphabetically.

Solution

  1. Depth first search: S – A – C – D – B – E – G2 = 17

  2. Breadth first search: S – A – B – C – D – E – G2 = 16 (the parent links trace the shortest path back to root S – B – G2)

  3. A*: S – A – C – D – G1 = 14

    For A*, the queue contains the following items after each pass:

    • S = 10
    • A = 8, B = 15
    • C = 7, D = 11, B = 15
    • D = 10, B = 15, (S = 16)
    • G1 = 14, B = 15, (S = 16, B = 19)
    • G1

    Note that A* does not find the path with the least cost (S – B – E – G2 with a cost of only 13).

    This is because the heuristic is not admissible: A* is guaranteed to find the shortest path only when the heuristic used is admissible.

    This means that the heuristic should never overestimate the true cost.

    In this assignment, the heuristic is not admissible. Example: In B the cost is estimated to be 8 while the true cost to reach G2 is only 6 (B – E – G2).

Path planning

Imagine a simple block’s world as shown below. A block can be moved a single cell at a time in each of the four directions. We have to find the shortest path from configuration A to configuration B.

  1. Discuss whether depth first search and breadth first search will always find a path from A to B. Will they find the shortest path?

  2. Suppose that we want to use informed search in order to guide our search. We will use the Manhattan distance as a search heuristic.

    Discuss whether best first search will find the shortest path, use the left drawing to add f, g and h costs. Will A* do so? Use the right drawing to add f, g and h costs.

  3. Now suppose that there is a direct underground connection from C to B. Hence, the shortest path from A to B now is over C (it is only five steps). Argue whether A* will find this shortest path.

Solution

  1. If it is ensured that the algorithms do not get stuck in loops, then they will find a solution. Depth first search will return the first path to the goal state it happens to encounter, which is not guaranteed to be the shortest one. Breadth first search, will find the shortest path.
  2. In this particular case, we can easily see that there are actually two possible ways of going from A to B in an efficient manner, either travel above the obstacle or travel below the obstacle. If you calculate the f-cost of every cell in the grid, it is observed that both paths have exactly the same f-cost when using the best first search. However, the lower path is shorter. So, with best first search, it is not guaranteed to find the shortest path. With A* the shortest path is found, because the heuristic is admissible.
  3. The shortcut through the underground connection is not reflected in the A* heuristic. At position C, the f-cost will be 11 (g = 4, h = 7).So, our heuristic overestimates the cost to the goal, thus the heuristic is not admissible. A* will prefer the solution found in B.

Board game

A new game comes with a board which contains 7 cells (as shown below).There are 6 pivots, each containing a number. Each pivot has to be in one cell and each cell can contain at most one pivot. Hence, there is always one empty cell. A pivot can be moved one cell to the left or one cell to the right (if it does not break the rules given above) with a cost of 1. A pivot can also jump over another pivot, with cost 2 (if it does not break the rules given above). The goal of the game is to have the empty cell in the middle, such that the sum of the blocks at the left of the empty cell is equal to the sum of the blocks at the right of the empty cell.

  1. Model this as a state space search problem.
  2. Which of the search algorithms studied in class is best suited for solving this puzzle? Motivate (max 5 lines).
  3. Give a sequence of states leading to the goal state.

Solution

  1. The state can be modelled by taking the vector of the board, that encodes the position of each element and of the white space, along.

    From this encoding we can extract the position of the empty cell(to check if it is in the middle) and also if the state is a goal state or not (by computing the two sums).

    For example, for the starting position the state could be <8,1,5,−,2,3,7>. At each step there are 4 available actions: swapping the empty cell with the first (cost1) or the second (cost 2) cell on the left and the same for the right direction.

    If we consider the graph search version, the branching factor is at most 3, except for the expansion of the initial state, which is 4.

  2. There are multiple goal states for the problem, as we can split the elements in any combination that outputs the same sum for both sides, while the order of the elements inside the groups does not matter:<1,7,5,−,8,3,2>,<7,1,5,−, 8,2,3>,.... A simple uninformed search approach for the problem can be UCS.

    It is not straightforward to apply informed search approaches for this case: designing a heuristic (admissible for A*) proves to be hard as there is not clear connection between the performed actions and their effect on the sum balance.

    However a nice intuition can be given by the difference between the left and right sums. If left−right is positive that means we have an excess of elements in the left side so for sure the empty cell should move to the left. This could be used to prune the search tree during the search process.

  3. 815-237 (0), 81-5237 (1), 8125-37 (3), 81253-7 (4), 812-357 (6), 8-21357 (8), 82-1357 (9), 821-357 (10), 8213-57 (11), 82-3157 ( 13), 823-157 (14)-10 steps, cost 14

Farmer, fox, chicken and grain

A farmer is on the left bank of a river with a fox, a chicken and a bag of grain, all of which he wishes to get to the right bank. There is a boat that can carry the farmer and at most ONE of those items at a time across the river. The farmer must be careful about what is left behind when he gets into the boat; if the fox is left alone with the chicken, the fox will eat the chicken; similarly, if the chicken is left with the grain, the chicken will eat the grain. Note that the fox will never eat the grain and that neither the fox nor the chicken will eat as long as the farmer is on the same bank as them.

  1. Formulate the problem as a search problem by specifying an initial state, a general state, the operators, the goal test and the path cost.
  2. Using your problem formulation, describe and illustrate the operation of the breadth-first and depth-first search algorithms (Note that you need not conduct the search to completion, only so far as to demonstrate the operation of the algorithms.)
  3. Which of these algorithms better solves the problem? Justify your answer with reference to space and time complexity, completeness and optimality.
  4. Would you recommend an alternative search algorithm for either problem? Again, justify your answer.

Solution

  1. A general state needs to encode the position of the farmer Fa (that is also the position of the boat), the fox Fo, the chicken C and the grain G.

    For example a state can be a couple of sets representing the variables on the left and right bank. The set of admissible states is the set of states where nothing gets eaten.

    The initial state is ({Fa,Fo,C,G},{}).The goal state is ({},{Fa,Fo,C, G)).

    There are two types of operators:

    • moving with a good b: (X∪{Fa,b},Y)↔︎(X,Y∪{Fa,b})
    • moving empty: (X∪ {Fa},Y)↔︎(X,Y∪{Fa})

    Each operator can be used from left to right or from right to left. Only the operators that transform an admissible state into another admissible state. Without specification in the problem, one can use a uniform path cost of 1 per operator can be used.

  2. BFS and DFS

    • For BFS:({Fa,Fo,C,G},{})

      • 1 transition: move with C({Fo,G},{Fa,C})
      • 2 transitions: move with C or move empty({Fa,Fo,G},{C}),({Fa,Fo,C,G},{})
      • 3 transitions: move with Fo, with G or empty({Fa,Fo,C,G},{}),({G},{Fa,Fo,C}),({Fo},{Fa,G,C}),({Fo,G},{Fa,C})
      • 1 transition: move with C({G},{Fa,Fo,C}),({Fo},{Fa,G,C}),({Fo,G},{Fa,C}),({Fo,G},{Fa,C})

    • For DFS: ({Fa,Fo,C,G},{})

      • 1 transition: move with C({Fo,G},{Fa,C})

      • 2 transitions: move with C or move empty({Fa,Fo,G},{C}),({Fa,Fo,C,G},{})

      • 3 transitions: move with Fo, with G or empty({G},{Fa,Fo,C}),({Fo},{Fa,G,C}),({Fo,G},{Fa,C}),({Fa,Fo,C,G},{})...

        If we had put the “move empty” transitions first, then the algorithm would have looped.

  3. There are cycles in this graph (after taking one transition, one canal ways go back) so DFS is not guaranteed to converge. Moreover,even if it converges, it might not find the shortest path. Moreover the branching factor is low. If the farmer is with 3 other items in his bank, then there are n+1 possible transitions (moving with one of the items or moving alone), and if n=3, the only admissible transition is to move with the chicken, so the branching factor is always 3 or lower(most often lower). This makes BFS a more suitable algorithm.

  4. With a uniform path cost, UCS is equivalent to BFS. IDS is complete and optimal, so it could be used instead of BFS if memory proves to be a problem. A human would typically use a DFS coupled with a cycle detection to allow only transitions that lead to a new state.

Sudoku

In the sudoku, you need to fill a 9×9 grid with numbers from 1 to 9such that each number is only present once on each line, column and 3×3 smaller grid. For example starting from the following grid:

  1. Formulate the problem as a search problem by specifying an initial state, a general state, the operators, the goal test and the path cost.
  2. How many steps does it take to reach a goal? Is iterative-deepening search a good algorithm for this problem? Why?
  3. How many moves are possible from the initial state (you can stop counting when you feel it is too much)? Compare to the length of the solutions found in previous question. Would you rather use breadth-first or depth-first search?
  4. Describe informally how you would solve this problem by hand. Why do you think it is better than depth-first search?
  5. Formulate your approach as a best-first search. It might require you to choose another path cost function.

Solution

  1. A general state is an encoding of the grid, such an 81-tuple containing the value of each slot (with 10 possible values: 1 to 9 or ∅ ). The set of admissible states is the set of states that do not break the Sudoku rules. The initial state is the starting grid. A final state is an admissible state where all the slots have an integer value.

    The operators consist in filling one slot, replacing a value ∅ with an integer. Only the operators that go from an admissible state to another one are considered. With no further specification, one can consider a uniform cost of 1 per operator.

  2. Each operator fills exactly one slot so any path to a goal has a length n where n is the number of empty slots in the starting grid, and any failed path is shorter. IDS is useless in that case: the first n−1 iterations cannot find a solution, so one can directly use a DFS search instead.

  3. In this example there are 49 empty slots in the starting grid so there are 49∗9 = 441 possible transitions. At least 49 of them are admissible if the grid has a solution, and the actual number of admissible solutions must be around 200. Although the branching factor de-creases as more slots get filled, BFS is untractable for this problem.

  4. A simple idea is to find slots with only one admissible value and fill them greedily. That solution is not complete (on a complex grid,there might be a point where each empty slot has several admissible values), but it works for simple grids. Compared to DFS, this approach does not require backtracking since a slot is only filled when one is sure of the value.

  5. The idea is to penalise transitions with several possible admissible values. For example, one can use a transition cost of n, where n is the number of admissible solutions when filling a given slot.

Labyrinth

This labyrinth is a 10×10 grid with walls randomly placed between adjacent cases. An agent is trying to find the shortest path from the entrance in the upper left to the exit in the lower right.

  1. Formulate the problem as a search problem by specifying an initial state, a general state, the operators, the goal test and the path cost.

  2. Give an estimate of the branching factor of the problem, the number of states and the length of a solution. Would you rather use breadth -first of depth-first search for such an algorithm? Justify your answer. What if the labyrinth gets larger? How about best first search?

  3. Propose an informal heuristic to find the shortest path faster. Formulate it as an admissible heuristic.

  4. Demonstrate the execution of the A* algorithm on this labyrinth. If your heuristic does not work as well as you expected, try to find abetter one.

  5. Now consider that the agent is a robot that moves physically inside the labyrinth. It does not know the map of the labyrinth but it can sense its own orientation, which ways are open out of its current case and it knows that the exit is in the lower right corner. The robot wants to reach the exit as fast as possible. What does it change to the problem? Which algorithm would you use?

Solution

  1. A state is a slot in the labyrinth. The starting state is the upper-left slot. The goal is the lower-right slot. The operators are moving up,left, right or down if there is no wall between the initial and final state.

  2. The branching factor is lower than 4. The denser the labyrinth (ie the more walls), the lower it gets. Any solution takes at least 18 steps and up to 99 for a path that goes through every slot. The denser the labyrinth, the longer the shortest solution will be on average (the less direct). BFS is more suited for very dense labyrinths and DFS is better for very sparse labyrinths. Since the cost is 1 per transition, UCS is equivalent to BFS.

  3. The idea is to favour moving in the general direction of the goal over moving back. One can use the Manhattan distance to the goal defined as the difference between the line indices of two slots plus the difference of their column indices.

    For example, the Manhattan distance between the starting point and the goa l is 9+9=18. This heuristic is admissible because it would be the length of the shortest path if there were no walls at all. If there are walls, the length of the shortest path goes up.

  4. Here is a map of the cost+heuristic of different slots. First, all the nodes with the lowest C+H of 18 are expanded, and some nodes with a higher cost are queued in the agenda. Once all the nodes with C+H 18 have been expanded, then all the nodes with a C+H 20 are expanded and the goal is still not found. Then some nodes with a C+H of 22 are expanded, some nodes with a higher C+H are queued in the process and the goal is reached with a cost of 22. The order in which two nodes with the same C+H are expanded depends on the implementation of the algorithm.

  5. Not knowing the whole map does not change anything: in A*, actual map is also replaced by the heuristic for the part of the graph that has not been explored yet. The difference is the cost function: now back tracking has a cost. Rather than the length of the final path,the robot wants to minimise the total computation time before finding the exit. In that case DFS seems more indicated with several modifications:

    1. when several transitions are available, give a higher priority to those moving right and down
    2. introduce randomness in the choice of the next transition to avoid getting trapped in a cycle.

The MAXONE problem

Suppose we want to maximize the number of ones in a string of l binary digits. Is it a trivial problem? It may seem so because we know the answer in advance. However, we can think of it as maximizing the number of correct answers, each encoded by 1, to l yes/no difficult questions. Assuming l = 10:

  1. Define the genome of an individual
  2. Define the fitness function
  3. Create a random population of size 6
  4. Using selection and mutation operators (with your own defined probabilities), evolve your population 3 times.

Solution

An individual is encoded (naturally) as a string of l binary digits. The fitness function is the number of ones in the genetic code of an individual. A possible population of size 6 is

  • s1 = 1111010101
  • f(s1) = 7
  • s2 = 0111000101
  • f(s2) = 5
  • s3 = 1110110101
  • f(s3) = 7
  • s4 = 0100010011
  • f(s4) = 4
  • s5 = 1110111101
  • f(s5) = 8
  • s6 = 0100110000
  • f(s6) = 3
