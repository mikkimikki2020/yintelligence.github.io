Background + Question + Video + Solution

Try your solution in IDP. Use it to find a valid colouring of the following graph:

Graph Colouring (bis)

Assume a set Node of nodes and Colour of colours is given. Write a first-order theory that states that the predicate Colouring: Node × Colour→ B is a valid colouring of a graph represented by a binary predicate Edge ranging over colours.

  • there are 5 nodes: N1,N2,N3,N4,N5
  • the graph has the following edges: (N1, N3), (N1, N4), (N2, N3), (N2, N4), (N2, N5), (N3, N5).
  • There are three colours: Blue, Red and Green
vocabulary V{
	type node
	type color
	coloring(node, color)
	edge(node,node)
}

theory graphFO :V{
 // Put your theory here
    ! x[node] y[node]: edge(x,y) => ~? z: coloring(x,z) &  coloring(y,z).

	! n: ?1 c: coloring(n,c).
}


structure instance :V{
	//This is the instance given in the assignment:
	node = {n1;n2;n3;n4;n5} 
	color = {blue;red;green}
	edge = {n1,n3;n1,n4;  n2,n3; n2,n4; n2,n5; n3,n5} 
}

procedure main(){
 //Generate the valid colorings 
	print(onemodel(graphFO,instance))
}

N-Queens problem

The problem is to put N queens on a chess-board of dimension N so that they cannot attack each other. A queen can attack another queen if it is on the same column, row or diagonal. First determine which types (sets to quantify over) and predicates you will need. Then, write a first-order theory that models the solutions to the N-Queens problem. (you will need some arithmetic for the diagonals, for that you can assume to have access to the normal functions such as addition of numbers).

vocabulary QueensV{
	type Queen
	type Row isa int
	type Col isa int
	
	RowOf(Queen):Row
	ColOf(Queen):Col
}

theory QueensT : QueensV{	
//NOTE: this is a solution that uses FUNCTIONS which we did not see in class (e.g. a function that maps a queen to a row. Try doing it without fuction symbols yourself! )
	!q1 q2: q1~= q2 => RowOf(q1)~=RowOf(q2).
	!q1 q2: q1~= q2 => ColOf(q1)~=ColOf(q2).
	!q1 q2: q1~= q2 => abs(ColOf(q1)-ColOf(q2))~=abs(RowOf(q1)-RowOf(q2)).
}

structure Queens7 : QueensV{
	Queen = {1..7}
	Row  = {1..7}
	Col  = {1..7}
}

structure Queens8 : QueensV{
	Queen = {1..8}
	Row  = {1..8}
	Col  = {1..8}
}

procedure main(){
	print("Searching for a solution for the 7-queens problem")
	print(modelexpand(QueensT,Queens7)[1])
	print("Searching for a solution for the 8-queens problem")
	print(modelexpand(QueensT,Queens8)[1])
}

Jugs

You must obtain exactly 4 liters of water using a 5 liter jug, a 3 liter jug and a water pump.

Tasks:

  1. Find a representation in first-order logic.
  2. Find a solution for the question using IDP.
  3. What is the smallest number of steps necessary to solve this?
vocabulary V{
	//Which types do you need (sets to quantify over)?

	//Hint: if you want to use arithmetic over, say coordinates, in IDP you can do it as follows
	type Time isa nat
	Last(Time) //Special predicate indicate the last time point
	// This specifies that time points are subsets of the natural numbers and hence that you can add them up, ... 
	
	//Which functions and predicates do you need? Many of them will be time-dependent! 
	//How do you model a state? What are the actions? 
    
    
    //input:
    type Volume isa int 
    type Jug
    Capacity(Jug,Volume)
    
    
    //state
    Holds(Jug,Volume,Time)
    
    //actions
    fill(Jug,Time)
    empty(Jug,Time)
    pourFromTo(Jug,Jug,Time)
    
    //whether or not Jug is involved in an action
    change(Jug,Time)
    
    
}

theory jugs :V{
//In order to model all the temporal information, we have to be careful. This will be executed in the context of a **finite** domain. So 
// don't just write constraints about t+1 when t is a time point. 
//Therefor, all our sentences will be conditional: For all times EXCEPT the last time... This is how time T and time T+1 relate... 
// IN LOGIC: ! t [Time]: ~Last(t) => (). 
    
    //Actions cannot be concurrent
   ! j t: ~(fill(j,t) & empty(j,t)).
    ! j j2 t: ~(fill(j,t) & pourFromTo(j,j2,t)).
    ! j j2 t: ~(empty(j,t) & pourFromTo(j,j2,t)).
    ! j j2 t: ~(fill(j,t) & pourFromTo(j2,j,t)).
    ! j j2 t: ~(empty(j,t) & pourFromTo(j2,j,t)).
	! j j2 t: ~(pourFromTo(j2,j,t) & pourFromTo(j,j2,t)).
    
    
	//Definition of when something is changed
    ! j t: change(j,t) <=> (fill(j,t) | empty(j,t) | ? j2: (pourFromTo(j2,j,t) | pourFromTo(j,j2,t))).

    //Holds is functionql
	! j v v2 t: Holds(j,v,t) & Holds(j,v2,t) => v = v2. 
    
	//INERTIA:
	! j t v: ~change(j,t) & Holds(j,v,t) & ~Last(t) => Holds(j,v,t+1).
        
    //Definition of state in terms of actions: 
    //initially: empty
    ! j v: Holds(j,v,0) <=> v =0.
    
    //If you fill them they become full
    ! t j v : ~Last(t) & fill(j,t) & Capacity(j,v) => Holds(j,v,t+1)   .
    //If you empty them they become empty
    ! t : ~Last(t) &  empty(j,t)  =>  Holds(j,0,t+1)  .
    
    //Rules for pouring. Two cases:
    // First one goes completely in the second: We just pour it out in the second
    ! t [Time] j j2 v1 v2: ~Last(t) & pourFromTo(j,j2,t) & Holds(j,v1,t) & Holds(j2,v2,t) & Capacity(j2,c) & v1+v2 =< c   => Holds(j2,v1+v2,t+1) & Holds(j,0,t+1)    .
    //Otherwise: the second is full and the first contains the rest
    ! t [Time]: ~Last(t) => ( !j j2: pourFromTo(j,j2,t) & Holds(j,v1,t) & Holds(j2,v2,t) & Capacity(j2,c) & v1+v2 > c   => Holds(j2,c,t+1) & Holds(j,v1+v2-c,t+1)    ).


    //Goal: in the end a jar has four
	? t j: Last(t) & Holds(j,4,t).



    
}

structure instance :V{
	Time = { 0..7}
    Last = {7	}
    Volume = {0..5}
    Capacity = {j1,5;j2,3}
}

procedure main(){
print(onemodel(jugs,instance))
}
