Wireless Sensor Network

WSN 是指一组空间分散的专用传感器，用于监测环境的物理条件

  • 通信依赖于无线连接wireless connectivity。
  • 传感器（节点）遭受：
    • 有限的内存容量memory capabilities → 有限的计算过程computational processes
    • 电池供电 → 有限的能量容量energy capacities
    • 在不可靠的无线电环境中运行 → 耗尽drain传感器节点的电池
  • 这些限制对系统的整体性能和网络使用寿命有重大影响。

目标：最大化节点的生命周期

一种解决方案：Radio Duty Cycling 机制

Nowadays, WSNs are used in many industrial fields applications such as:

  • Environment (I.e: water catchments monitoring)
  • Heath (I.e: body areas networks)
  • Safety (I,e: Remote sensing for disasters)

在 WSN 中，已经考虑了几个因素，并且已经为通信堆栈的所有层提出了新的节能协议

  • COAP " Constrained Application Protocol " (With Observation extension)
  • UDP " User Datagram Protocol "
  • 6lowPAN "IPV6 over Low power wireless Personal Area Network"
  • IEEE 802.15.4 standard

原理

Radio Duty Cycling (RDC) Protocols 无线电工作循环 (RDC) 协议

  • 在 WSN 中，功耗至关重要。在不需要的时候关闭无线电可以降低功率需求。
  • 在传统的 MAC 协议中增加了无线电工作循环 (RDC) 协议。
    • 这个想法是让一个节点“休眠”，只要没有其他节点想要通信。
    • 可视为MAC 层机制。

我们将区分两大类 RDC 协议

异步Asynchronous RDC Protocols (Random medium access随机媒体访问)

异步工作的 MAC RDC 协议的一些示例：

  • NullRDC
  • X-MAC
  • LPP Low Power Probing“低功耗探测”

问题

  • 发送者在聆听时消耗能量（可能是一个完整的睡眠期）
  • 接收者可能花费的能量与发送者等待来自未知发送者的数据包一样多。
  • 多个节点想要将一个数据包发送到一个共同的目的地时，它们的数据包会发生冲突collide。
    • CSMA-CA algorithm can be used to avoid collision
    • CSMA 在重传未确认数据包时引入的随机延迟random delays可以缓解该问题。

Contiki-MAC

  • Idle mode “no packets to be sent or received”:

    • 只要信号强度低于 RSSI 阈值，节点就会保持休眠状态。一旦检测到（RSSI 高），节点开启接收模式。然后进入发送模式发送ACK

  • Transmitting a packet to a specific destination:

    • 送方在一个完整的睡眠周期（大于发送 2 个 CCA）或直到它收到一个 ACK。
    • 它计算接收器何时醒来。 下一次传输将发生在它醒来之前不久，从而减少了传输次数--> 减少流量和功耗。

  • Broadcasting a packet:

    • 广播包不需要确认。
    • 发送方在完全唤醒期间重复发送数据包，以确保所有邻居都收到了。

  • 节点定期发送 CCA“Clear Channel Assessment”以感知传入数据

  • 发送 CCA 消耗更少的能量CCA：是网络活动的载波侦听机制carrier senses mechanism

  • CCA 依赖 RSSI “Received Signal Strength Indicator

    备注：

    • CCA 检测其他活动，可以是误报 --> 能耗
    • 节点发送两个 CCA，那么 CCA 不会落在两个连续的
    • *发送者的帧**
    • 最短帧的长度不能介于两个 CCA 之间
同步 Synchronous RDC Protocols
  • 在contention free的媒体访问环境中运行
  • 要求节点间时间同步tight time synchronization
  • 使用定时通讯time-scheduled communication
    • MAC 协议将通信信道划分为时隙time slots，并为通信对分配时隙。

同步工作的 MAC RDC 协议示例：

TSCH Time Slotted and Channel Hopping"“时隙和信道跳跃”使用多个通道以同步方式进行节点通信，以增加网络可靠性。

  • Time Slotted:
    • 及时将通信信道切成槽slots（适合traffic size）
    • 多个equal length and priority的时隙的组合称为时隙帧Slot frame，它随时间不断重复
    • 每个时隙都分配给一对通信节点 (Tx/Rx)，没有随机访问 => 冲突显着减少
    • 时隙持续时间必须足够长才能发送数据包并接收 ACK。
      • 如果需要重传数据包，会在下一次迭代中发生
  • Channel Hopping:
    • 16 channels are used for communication at 2.4GHz
    • Multiple communications happen at different frequency channels at the same time.
    • 如果一个信道发生干扰，则可能跳到另一个信道，从而获得更大的成功机会。
    • 注意：如果通道彼此靠近则无济于事

Key Features

  • 时隙访问减少冲突并确保无争用访问同时使用不同的信道传输数据增加了数据传输的容量网络并减少延迟。
  • 每次通信使用多个信道减少多径衰落效应，干涉。
  • 这增加了通信的可靠性。

问题

如果网络上没有活动（没有传输），节点仍然必须在预定的时间段醒来。 --> 与理想相比更高的能耗

前景

无线传感器网络是一种小型、低成本、支持无线通信的传感器，用于构建各种监控网络，用于不同领域，如工业自动化、过程控制、农业、医院监控系统、资产跟踪、 气候感知等。

在医疗领域，使用可植入和可穿戴医疗设备的无线体域网进行的持续健康监测被设想为一种变革性的医疗保健方法。生物医学传感器、低功耗电子产品和无线通信的快速发展使这一愿景成为现实。随着连接的医疗保健物联网设备的增加，WSN有望获得显着增长。例如，在欧盟，物联网解决方案正在快速增长，该市场为医疗保健部门提供了很多机会。物联网医疗保健活跃连接的数量预计将在预测期内增加，从而推动 WSN 市场。

5G Networks

5G 的目标是改进 4G 并提供数 Gbps 的峰值数据速度、超低延迟、可靠性、海量网络容量、高性能和效率。它由一个完全基于 IP 的模型组成，专为无线和移动网络设计。

该系统由一个用户终端和多个自主无线接入技术组成。5G主核基本上是其他技术的汇聚点。 这种设计便于主核运行到并行的多模式，包括IP网络模式和5G网络模式这样，它控制了 Radio Access Networks和Different Access Technologies.的所有网络技术。

原理

除了拓扑的新改进和新的通信架构

  • 毫米波5G
    • 运行在更高的无线电频率上，更不杂乱。频率约为 28 GHz 和 38-39 GHz
    • 4G 目前在 700 MHz 至 2.5-2.6 GHz 无线电频谱频段上运行
  • Mesh 网络：5G 的骨干网络将从铜缆和光纤连接到毫米波无线连接，通过基站之间的协作实现快速部署和网状连接
    • 虽然这些更高的频段可以更快地传输信息，但在远距离发送该信息时存在问题。
  • MIMO 天线：这些波很容易被物理物体阻挡。为了解决这个问题，5G 使用 MIMO 天线来增强整个网络的信号和容量。
    • 5G 将使用更小的发射器，而不是使用单个独立的桅杆
    • 据估计，5G 将能够比 4G 支持多达 1,000 台设备/米
  • 虚拟网络：5G 还可以将一个物理网络“切片”成多个虚拟网络。
    • 运营商将能够根据重要性提供正确的网络切片。

技术特点

  • Beamforming
    • 智能波束成形技术可以通过将radio frequency energy集中在downlink上的特定方向上，并使基站接收器能够从特定方向捕获能量，从而在上行链路上减少噪声和干扰，从而扩展基站的覆盖范围。
    • 基本上，5G NR 大规模 MIMO 技术将在能够进行 3D 波束成形的基站使用 2D 天线阵列，以利用中频段频谱的较高频段。
    • 准确及时的信道知识对于实现这种 3D 波束成形的全部优势至关重要
  • OFDM Orthogonal frequency-division multiplexing
    • 与 4G LTE 一样，5G 基于 OFDM（正交频分复用），但也使用 5G NR 空中接口和更宽的带宽技术，如 sub-6 GHz 和毫米波。
    • 5G NR 空中接口基本上进一步增强了 OFDM，以提供更高程度的灵活性和可扩展
      • 3G 信号广播是垂直极化的，在那里你有“上下”传播的波
      • 4G LTE MIMO 波是倾斜极化的，其中每个波都从水平方向旋转 45 度，经过镜像处理，因此第一个波在 45 度，另一个在 135 度。
    • 这个聪明的技巧被称为polarisation diversity，它允许您的调制解调器在蜂窝塔分配的相同频率上区分两个独立的数据流。
    • MIMO 由调制解调器基于与蜂窝塔的协商打开和关闭，具体取决于接收/传输信号的质量被评估。

Li-Fi (Light Fidelity)

LiFi 是一种光通信技术，它利用可视光通信或 Visual Light Communication 的特性来传输和接收数据。

  • Wi-Fi: Radio Frequencies (40GHz wide) 无线电频率频谱相当有限
  • Li-Fi: visible light (400THz wide)
  • Interference? Mobile devices + limited spectrum
  • Infrastructure = in place
  • 只需要更换高效的 LED 灯
  • VLC的基本原理就像传输数据的点对点源
  • LED 的密度可以以非常高的速度进行调制
    • 平均而言，LED 灯每秒可以打开和关闭约 10.000 次。 在这个速度下，人眼将无法检测到光密度的任何变化
    • 接收器对光密度的细微变化感兴趣，这会导致信息流。

原理

  • Signal: LED intensity (≠ amplitude or phase)

LiFi attocell network

  • Different attocells → interference

    不同的 attocell（或接入点，AP）会对网络造成干扰位于 attocell 区域边缘的用户

    E.g.: user 1 & 3 (SIC)

可以通过将 NOMA 与光空分多址或 SDMA 相结合来提供解决方案。 这是通过协调多点辅助联合传输技术实现的。 在传输预编码（TPC）的帮助下，不同地方的用户将同时得到服务。

→ NOMA + optical space division multiple access (SDMA)

→ Multi-point aided joint transmission technique

  • Different users served with TPC

  • Signal → TPC + SDMA ➔ mitigate interference

    当信号传送到接收端时，TCP 和 SDMA 将确保减轻干扰。

  • Spatial signatures → TPC-vector

    将根据两个用户的空间特征设计一个 TPC 向量然后将其添加到信号中，以提高用户 1 和用户 2 的性能。

前景

LiFi 是一项相对较新的技术，具有很多潜在用途。越来越多的设备连接到互联网，数据传输在接下来的几十年只会增加。由 3G 和 4G 等技术引起的数据传输量的增加将揭示电磁频谱的局限性。

LiFi 可以用作支持 Wi-Fi 系统的补充资产，更具体地说，可以卸载很大一部分无线数据流量。预计未来几十年该市场将高速增长，因为它具有广泛的应用。因此，LiFi 未来应该以积极的方式发展，因为它似乎是 LaaS（Light as a Service）的先驱。

随着大多数灯现在被节能 LED 灯所取代，这种增长将得到加强，这将拓宽 LiFi 技术可以运行的基础设施。这将作为一个背驮式骑行系统。

LiFi 技术的主要未来优势将是其安全特性。 它最重要的用途是在医疗保健业务中。 在医疗保健和医院中，设备对电磁污染非常敏感。

不建议在此处使用 Wi-Fi，因为它以无线电频率运行，这会对设备造成干扰。 因此，LiFi 可以解决这个问题，因为光线不会干扰该设备。 同样的原理也可以用在飞机上，光线不会干扰驾驶舱的设备，乘客也不必关掉手机。

未来如果 LiFi 技术应用于汽车制造行业，也就不足为奇了。 由于车祸在世界范围内仍然是一个大问题，LiFi 通信可以用于汽车的大灯。 然后汽车可以相互通信并发送信息。 当前方行驶的汽车由于某种原因不得不非常突然地停车时，这可以传达给周围的汽车，以便他们可以对此进行预测。

2020 年全球 LiFi 市场估值达到 3 亿美元左右，预计到 2026 年将成长为 40 亿美元的产业，未来五年累计异常增长率每年约为 70%。

Body Area Networks

体域网络 (BAN) 是人体上或人体内部的传感器集合，它们与位于人体上或人体附近的中央汇节点进行通信，主要侧重于医疗监测。虽然传感器可以通过物理连接（即有线）与汇聚节点进行通信，但在这里我们将只关注无线体域网 (WBAN)。

WBANs (Wireless Body Area Networks)

  • WBAN 与其他短距离通信标准之间的主要区别在于WBAN 应该具有非常低的功耗

WBAN architecture

  • Sink aka Personal Server (PS) aka Base Station

    • Process the collected data from sensors in Tier 1 and transmit it to Tier-2

  • Tier-1: intra-WBASN

    • Between sensors and sink

      放置在身体上或身体内的传感器节点（例如 ECG 或葡萄糖传感器）与接收器（也称为个人服务器或基地站），通过低功率通信方法

  • Tier-2: inter-WBASN

    • Between sensor and access point(s) (AP)

      对其进行预处理

      可以访问互联网的接入点，例如智能手机或 PC

    • (Without an AP, the sink could theoretically communicate over 3G/4G)

  • Tier-3: beyond-WBASN communication

    • Reach end recipient 超越禁令，到达最终接收者医生、救护车

Power Consumption vs Data Rate (critical)

WBAN 比个人区域网络 (PAN) 具有更具体的用途，因为 PAN“不符合某些应用环境的医疗（接近人体组织）和相关通信法规。它们也不支持广泛满足广泛的体域网应用所需的可靠性 (QoS)、低功耗、数据速率和无干扰的组合。

Ultra-Wide Band

超宽带 (UWB) 是一种替代通信机制，也可以服务于需要低功耗的应用。

  • UWB Operates below FCC mandated power spectral density load
  • Central node wants to communicate to 2 different devices
  • Uses Time Hopping
  • Time is split into time frames (Tf)
    • Each time frame is split into chips (slots) 1 bit is encoded by a transmission of pulses in 3 successive time frames
    • For each device, the bit is determined to be a 0 or a 1 depending on a time delay delta
  • Requires clock synchronization

大量数据在传播时可以更快地传输跨越很宽的带宽，让发射机保持更短的持续时间并减少耗电量显着。 这意味着，对于相同的消耗功率，UWB 可以传输更多的数据

UWB 通信的带宽要大得多：它发送和接收带宽在 500 MHz 和几 GHz 之间但功率谱要低得多的数据密度。由于它们使用的低功率密度和高频率，信号可以只行进有限的距离，不会干扰其他波。他们是一十手机发出的功率的千分之一到十万分之一，因此它们对人类健康要低得多。这使得它们非常适合低功率设备在人体附近操作。

UWB 很有用，因为它涵盖了广泛的频率范围，并且功率谱密度为低于 FCC 门槛

未来

安全可能是最大的WBAN 将面临的问题。由于传感器体积小，功率有限，计算能力和执行加密等安全措施的存储仍然存在难的。

  • System and device level security
  • Invasion of privacy
  • Sensor validation
  • Data consistency
  • Sensor resource constraints
  • Worldwide operability

市场

由于体域网是一个通用的网络架构概念，因此没有可以附加到其上进行分析的包罗万象的产品。相反，人们可以分析彼此兼容的不同类型的传感器和接收器，以便了解市场的发展方向。虽然体域网有很多应用，但我将重点介绍医疗体域网组件，因为它们具有最大的未来潜力。如前所述，也有许多不同类型的传感器可用于医学评估（心电图、血氧、血糖等）。另外，我将放大智能手表市场，因为它目前是最成熟的，并且看看它多年来在市场份额和越来越多的医疗功能方面是如何发展的。

从全球来看，智能手表行业主要由苹果、三星、华为、Fitbit（现为谷歌所有）等大型电信和电子电器公司主导。

显然，苹果的市场份额最大，为 40%，如 2020 年第 4 季度所示。然而，很明显，市场是由高端品牌驱动的，由于技术仍然是新的和专门。这将继续成为未来 4-5 年的趋势，直到小公司找到方法来复制这些设备的功能，同时降低制造成本，因为他们的目标是使技术从高端市场转向大众市场。

Vehicle-to-everything (V2X) Communication

车辆通过传感器变得更加智能，现在可以连接车辆，将单个自治单元转变为协作系统。合作智能交通系统 (C-ITS) 需要始终保持连接 目标是提高道路安全和交通效率车载自组织网络 (VANET) 发挥着重要作用。

主要使用两种技术（DSRC 和 C-V2X）

AUTONOMOUS DRIVING

  • Six Levels of Autonomy (L0-L5).
  • First L3 vehicles are available.
  • V2X will be needed for fully automated systems.
  • Level 4 and 5 require always on communication
  • 25 Gbps per hour projected requirement for L5 Vehicles communication

V2X 通信必须持续捕获和解释有关周围环境的完整数据。因此，车辆和实体之间的通信需要始终能够传输和接收有关周围环境的足够信息。

也就是说，可以根据通信实体和应用程序将 V2X 分析为进一步的子类别。 这些类别是

  • Bidirectional communication of between two vehicles 两辆车之间的双向通信(V2V)
  • Vehicle to Infrastructure (V2I)
  • Vehicle to Pedestrian行人 (V2P)
  • Vehicle to Network/Cloud (V2N)
  • Vehicle to Grid电网 (V2G)

原理

DSRC

  • 基于名为 ITS-G5 的 IEEE 802.11p 的欧洲标准化工作。
  • 5.9 GHz频段分配给欧美的V2X通信
  • Decentralized Congestion Control减少了空气中的拥塞。去中心化拥塞控制 (DCC) 是一种机制，有助于网络的稳定性和资源的分配。
  • 范围有限，无忠诚度loyalty free的临时通信。
  • 欧洲市场上的可用车辆配备。

无线网络相比，在 BSS (OCB) 模式之外简化了操作在 IEEE 802.11 术语中称为基本服务集 (BSS) 的网络，因为它禁用管理程序，例如信道扫描、身份验证和关联，并使用通配符BSS 标识符。启用 OCB 的 ITS-G5 站可以直接和立即传输消息没有耗时的控制帧交换延迟。

网络和传输层分为两层，

  1. 第一个是使用 GeoNet 工作和基本传输协议 (BTP)
  2. 第二种解决方案是已知的来自 Internet 协议，尤其是在 IPv6 中，其中 UDP、TCP 和其他传输协议被执行

CAM 和 DENM 都组织在带有 ITS PDU 标头的容器中

CELLULAR V2X(C-V2X)

  • Standardized in 3GPP R14 and next Releases.

  • A vertical of use cases ultra-reliable low latency communications (URLLC)

    垂直用例超可靠低延迟通信

  • LTE Direct extension to support V2X communication.

  • 长期演进 (LTE) 可用作此类通信的机制。

  • 主要问题之一是现阶段的延迟。

  • Use of the current LTE infrastructure

  • V2X 将通过使用 R16 超低延迟增强和宽带载波提供高吞吐量（例如共享大量传感器数据）、更高的可靠性和更低的延迟支持（载波聚合）。

  • 它可以被视为超越 4G LTE V2X，但它将与 4G LTE V2X 共存。5G New radio(NR) V2X 侧重于 5G NR 在部署高级 V2X 服务中的作用，作为 3GPP V2X 的一部分。

    • 这些先进的 V2X 服务代表了超出 LTE R15 V2X 范围开发的服务，需要增强型 5G NR 系统和新的 5G NR

  • 车载设备分配 Vehicular Device to Device (D2D) 无线电通信链路满足要求。 在这些要求中，需要有一个灵活的设计来支持具有低延迟和高可靠性要求的服务，以及支持更高的容量和更好的覆盖范围。

C-ITS STACK 应用根据需求使用不同的零件

  • Facilities layer ITS applications are provided with functions, information or services.
  • Network and transport layer Communication heterogeneity covers various ITS-specific and general protocols.
  • Physical layer and data link layer enables V2X communication in ad-hoc networking environment

COOPERATIVE AWARENESS MESSAGE (CAM) 车辆状态的连续传输

CAM 协议携带关键的车辆状态信息。安全和交通效率应用程序使用这些消息，接收这些消息的车辆能够跟随其他车辆”，因为它们正在连续发送。包中有两个容器，一个是是车辆状态，被高频发送。 （速度，位置）在较低的频率下，还发送附加信息。

  • Vehicle position and basic vehicle data. (Location,acceleration, Size)
  • Periodically sent as one hop geographical broadcast.
  • Frequency 1-10 Hz and size up to 800 bytes.

DECENTRALIZED ENVIRONMENTAL NOTIFICATION MESSAGE (DENM) 事件触发消息通知情况

DENM 是事件驱动的，并在地理范围内提供安全信息。地区。这些消息由应用程序控制，并在检测到情况时发送。一种为该 ITS 站添加了唯一的动作标识符以提供信息。

  • Notification to road users on specific events.
  • Event triggered.
  • Examples: Stationary, Vehicle, Roadworks, Weather Conditions

GeoNetworking 使用地理信息的网络层功能

GeoNetworking 堆栈主要用于通过 ITS-G5 进行临时通信，并且可以用于具有地理寻址的单跳和多跳通信。地理寻址使用 IPv6，通过蜂窝网络与基于 IP 的基础设施进行通信。它也可以通过 GeoNetworking 发送 IPv6 数据包。

  • Provides ad hoc networking based on geographical addressing and geographical routing.
  • Can work on different ITS access technologies.
  • Several addressing and forwarding mechanisms (Onecast, Unicast,Broadcast)
  • IPv6 Support
  • Geo-Networking Flow can be single-hop and multi-hop.
    • Single-hop is mainly a beacon and is only broadcasted once (CAM).
    • Multi-hop can be a Geo-Broadcast (DENM).

应用

  • Emergency Electronic Brake Light (EEBL) application
    • 该应用程序向车辆的本地追随者通知硬中断。
    • 需要车辆的能力，因此紧急电子刹车灯被激活并在 V2X DEN 消息中广播。
    • “结果是它会通知主（自我）车辆的驾驶员前方远程车辆突然刹车。 通过提供预警，驾驶员将有更多时间采取必要的行动来避免追尾事故。 当急刹车的车辆被其他车辆（驾驶员视线之外）挡住时，此应用程序在密集驾驶情况下特别有用。
  • Intersection Movement Assist (IMA)
    • 驾驶员在前往十字路口时会被警告与交叉路口发生碰撞。 经过解码和分析 CAM 消息，IMA 应用程序计算轨迹并提供碰撞即将发生时发出警告。
  • Left Turn Assist (LTA)
    • 驾驶员在即将左转时收到警报，以防止碰撞，当来自相反方向的车辆以相似的轨迹行驶。
    • 如果车辆之间的视线受阻，则需要使用路边单元 (RSU)。RSU 将是一种能够传递信号或检测并发出碰撞风险信号的设备
  • Traffic light optimal speed advisory
    • 在这种情况下，交通灯会广播与状态相关的计时数据（例如，到达绿灯时间）。
    • 这有助于驾驶员了解状态并调整其速度以获得绿灯波
  • Cooperative adaptive cruise control
    • 需要使用 V2X CA 消息和单播交换。 跟随的车辆，是获取前方车辆动态和一般交通情况。 然后，适应ACC并提高其性能。
  • Cooperative vehicle-highway automation system (Platoon)
    • 通过使用 CA 消息和单播交换，车辆可以作为一个排在高速公路或特定车道上安全运行。
  • Vulnerable road user Warning
    • 在存在易受伤害的道路使用者的情况下向车辆提供警告。

市场

根据世界卫生组织和道路安全报告，每年约有 135 万人死于道路交通事故。大多数国家的道路交通事故损失占其国内生产总值的 3%。为了减少这些数字，安全车辆在避免碰撞和降低受伤概率方面发挥着重要作用。 20 至 5000 万人遭受非致命伤害，道路交通伤害造成的经济损失为 1677.524 亿美元（2005 年）数据）。欧盟委员会设定了零愿景，旨在消除事故的数量接近于零。

V2X 可以帮助实现这一目标。这项技术还将提高交通效率，消除因拥堵而浪费时间和燃料的交通拥堵成本。这将有助于减少排放[18]，因为交通拥堵使二氧化碳排放量增加了 300%。在美国，这一成本为每年 1,240 亿美元，每户约 1,700 美元，预计到 2030 年约为 2.8 万亿美元，而在英国德国 法国 成本为 497.54 亿美元，预计到 2030 年将降低 42%。因此，减少排放有助于缓解拥堵、速度管理和交通顺畅。如果将这些策略结合起来，可以减少30%的排放，同时增加旅行体验。

预计2019年至2022年V2X的复合年增长率（CAGR）将超过170%。 根据[17]到2022年底，V2X的市值将达到12亿美元。 继 2019 年底公开提供的第一个解决方案之后[20]，全球配备 V2X 的车辆预计将接近 600 万辆。[17]。 根据 [3]，到 2028 年，市场预计将达到 128.59 亿美元左右，其中欧洲最大，其次是亚洲，其次是北美。
