What is mathematical induction数学归纳法?

A technique to prove properties of natural numbers P(n) with n ∈ N.

  1. Proof for P(0) (base case)
  2. Proof ∀n ∈ N : P(n) ⇒ P(n + 1) (induction step)

为什么在本课程中提到数学归纳法？

  • 它是递归的基础：从数学归纳法到结构归纳法。
  • 证明算法的性质是一种很好的技术

Recursion

A function is recursive if it is defined in terms of itself

算法处理给定输入所需的时间通常取决于输入的大小：例如，预计在 1000 个元素的vector中搜索元素比在 10 个元素的vector中搜索花费更多的时间， 仅仅是因为有更多元素可以与搜索键进行比较。 例如，让我们再次检查代码以检查vector是否包含给定的搜索键。

public boolean contains(Object obj){
    for(int i=0;i<count;i++){
        if(data[i]. equals(obj))
            return true;
    }
    return false;
}

该代码由一个单个for循环组成，用于遍历vector并按顺序访问vector的每个元素。 对于vector的每个元素，检查它是否等于键key。 因此，如果我们假设将键与元素进行比较所花费的时间不取决于元素本身，则算法的总运行时间仅取决于 for 循环的迭代次数。 为了估计这将有多少迭代，我们必须区分几种不同的情况:

  • 在最好的情况下，我们要搜索的元素是vector的第一个元素。 在这种情况下，只有一次迭代，因此需要进行一次比较。 正式地，我们将其写为 T（n）= 1，这意味着该算法的 runtime T 在大小为 1 的输入上运行。
  • 在最坏的情况下，元素不在vector中。 在这种情况下，for循环需要遍历整个vector。 因此，如果vector中没有元素，则算法需要进行比较或形式上T（n）=n。
  • 在平均情况下，在遍历vector的一半后找到元素。如果vector具有元素并且键在不同可能位置的每个位置或不存在的可能性相等，则要检查的平均元素数是T（n） = （1 + 2 +...+n）/n 等于 T（n） =n（n+1）/2n=（n+1）/2。

从上面，我们了解到在平均和最坏情况下在vector中搜索给定元素所需的时间取决于vector中的元素数量。此外，在这两种情况下都存在简单的线性关系。因此，在两倍大的vector中搜索平均也将花费两倍的时间。

在计算机科学中，分析算法的时间性能是至关重要的，因为对于许多算法来说，输入大小与平均运行时间之间并不存在线性关系。这种关系可以是二次的，甚至是指数的。在这种情况下，对于较小的输入大小，运行时间变得过长。

因此，从这个角度来看，能够知道这种关系是线性的还是二次的……比知道需要执行的操作的确切数量要重要得多。因此，对算法运行时性能的分析是关于比较数量级orders of magnitude，而不是比较精确数字。

Orders of magnitude and the big-O notation

下面定义的 big-O符号正是用于形式化数量级概念。

  • Definition 1 (big-Oh).

    • T(n) is O(F(n)) if there are positive constants c and n0 such that T(n) ≤ cF(n) when n ≥ n0.

      引入了 O 符号并定义了算法time complexities的上限。

      例如，如果 F 是一个线性函数，则定义表明 T(N) = O(n) 如果存在某个 c 对于从n0开始，T(n) ≤ cn。

      根据这个定义，在向量中找到一个元素的时间复杂度确实是 O(n)，因为确实存在一个 n0和一个 c，其中 T(n) =（n+1）/2 ≤ cn。

      请注意，如果上限是 O(n)，那么它也是 O(n2) 等等，因为等式仍然成立。显然，在算法分析中，我们对上限的准确估计感兴趣，因此我们使用 O(n)。

  • Definition 2 (big-Omega).

    • T(n) is Ω(F(n)) if there are positive constants c and n0 such that T(n) ≥ cF(n) when n ≥ n0.

      类似地，第二个定义给出了一个下限

  • Definition 3 (big-Theta).

    • T(n) is Θ(F(n)) if and only if T(n) is O(F(n)) and T(n) is Θ(F(n)).

      第三个定义，T(n) = Θ(n)，结合了下限和上限来表示 T(n) 的增长率等于 F(n)。 因此，尽管这是三者中最精确的估计，但并不常用。

需要注意的是，在报告time complexities时，需要考虑一些规则：

  • 不要在符号中添加任何常量，不要在符号中添加任何常量，例如 无论 T(n) = n 还是 T(n) = 2n 或 T(n) = 0.5n，它们都是 O(n)。 所以，O(2n) 或 O(0.5n) 是无效的，即使它们给出了一个真正的边界。这是因为在算法分析中，只有数量级order of magnitude是相关的。
  • 不要在符号中放置任何低阶项lower order terms，例如 无论 T(n) =n2 或 T(n) = 2n2+n 或 T(n) = 0.5n2+7，它们都是 O(n2)。 所以，O(n2+n) 或 O(0.5n2+7) 是无效的，即使它们给出了一个真正的边界。 这是因为在算法分析中，只有数量级是相关的。 对于足够大的 n 值，高阶项将始终支配低阶项。

Some examples

  • T(n) = 3n2+ 2n+ 7 = O(n2) (quadratic).
  • T(n) = n+n+n= O(n) (linear).
  • T(n) = 7 =O(1) (constant).
  • T(n) = 32n+ 7n2+ 11 = O(2n) (exponential).
  • T(n) = 2log(n) + 1 = O(log(n)) (logarithmic∗).

Analysing Divide and Conquer algorithms

分治算法的基本思想是递归地将问题细分为更小的问题，直到问题变得小到难以解决为止。 因此，分而治之并不是一种算法，而是一种开发算法以解决各种问题的范式。

假设总问题的size 是n。 解决问题的分治算法范式可以描述如下：

  1. 它将问题分成若干个，每个大小为n/b。 （请注意，这本身并不等于 b，因为子问题可能例如重叠。）
  2. 它解决了每个部分的问题。 （这是递归步骤）
  3. 它将解决方案聚合成一个单一的解决方案。

如果我们假设聚合 a 块大小为 n/b 的解的成本是 f(n)，那么这个分治范式的时间复杂度可以用以下递推关系表示：

  • T(n) =aT(n/b) +f(n)

这个方程表达了一个常识，即解决问题的成本 T(n) 是解决较简单问题的成本 T(n/b)，乘以较简单问题的数量，再加上聚合不同的较简单问题的成本 f(n)。

存在各种数学方法来求解这种递推关系。但是，如果我们做出两个非常合理的假设，递推关系很容易解决：

  • 我们假设当问题达到规模 1 时我们停止将问题分解为更小的问题
  • 我们假设那是 b 的幂

第二个假设的结果是大小为 1 的子问题总是在整数个除法步骤中获得（因为问题大小在每一步除以一个因子箱）。 因此，这两个假设共同导致算法总是在整数步中停止递归。

The time complexity of Binary Search

Theorem 3. The time complexity of the Binary Search algorithm on a sorted vector is O(logn).

证明：当对一个已排序的数组进行二分查找时，在算法的每一步，我们都需要决定是在左子问题还是在右子问题中继续搜索。 因此，在每一层，都要进行一次比较，以决定是向左还是向右，并且在每一步中，子问题的大小都减少了 2 倍。

  • T(n) = T(n/2) + 1
  • T(n) = logn+ 1 = O(logn)
