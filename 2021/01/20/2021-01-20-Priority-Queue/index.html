Priority Queue

Video + Slide 99 ~ 99

与 Stack 和 Queue 数据结构类似，Priority Queue是一种支持 push 和 pop 方法的简单结构。 但是，pop 方法不会返回结构中最顶层的元素，而是返回priority最高的元素。 因此，push 方法不仅存储数据，还存储优先级priority。 正如堆栈和队列一样，优先队列的实现可以基于Vector 和 Linked List。 此外，对于 Vector 和 Linked List，都有两种不同的方法：

  • 第一种方法包括维护按priority排序的数据结构。 这意味着push操作更加复杂，因为它现在需要在已经排序的数据结构中的正确位置插入元素，以确保结构保持排序。 因此，具有最高priority的元素将始终位于数据结构的末尾或开头，这样弹出操作仍然非常简单。 在这种方案下，push 的时间复杂度为 O(n)，而 pop 的时间复杂度为 O(1)。
  • 第二种方法实现了相反的逻辑：push操作非常简单（如在堆栈和队列实现中，它只是在末尾或开头存储新数据），代价是更复杂的pop操作。 由于数据现在未排序存储unsorted，pop 操作必须检查整个数据结构以识别具有最高priority的元素。 在这种方法中，push 操作的时间复杂度为 O(1)，而 pop 操作的时间复杂度为 O(n)。

请注意，这两个选项都会导致时间复杂度不对称。 一个操作的时间复杂度为 O(1)，而另一个操作的时间复杂度为 O(n)。 使用简单的底层数据结构作为Vector 和 Linked List，平衡和高效的解决方案是不可能的。

稍后我们将回到这个问题并开发一个更有效和更平衡的解决方案。 在本章中，我们将首先实现一个基于链表的实现，它使数据在 push 方法中排序。

Sorted Linked Lists

为了有一个存储排序的数据的Priority Queue，我们将首先开发一个sorted的链表。 这是添加了新操作的现有链表：addSorted(object value)。 如果链表已排序，即所有元素从小到大排序，则addSorted方法将在链表的正确位置添加一个新元素，以确保链表保持排序。

addSorted方法的思想是从左到右遍历链表，直到找到比新元素大的元素。 在这一点上，我们现在新数据应该出现在链表的当前元素之前。 由于在给定元素之后添加元素更容易，因此将遍历traverse链表，直到当前元素的下一个指针大于新元素。 如果链表为空，或者链表只有一个元素，这种方法可能不可行。 因此，我们必须识别所有这些情况并分别处理。

// THIS CODE IS NOT CORRECT JAVA CODE
// BUT IT ILLUSTRATES THE LOGIC OF THE ALGORITHM

public void addSorted ( Object o )
{
    // an empty list , add element in front
    if( head == null ) head = new ListElement (o , null );
    else if( o < head.first ())
    {
        // we have to add the element in front
        head = new ListElement (o , head );
    }
    else
    {
        // we have to find the first element which is bigger
        ListElement d = head ;
        while (( d . rest () != null )&&( o > d . rest (). first ()))
        {

        }

虽然上述代码的算法方法是正确的，但它不是有效的 JAVA 代码。 这是因为 < 和 > 运算符正在处理上述代码中“Object”类型的对象。 这在 JAVA 中是不允许的。 这很容易理解：链表可以包含各种数据，包括类的实例。 因此，假设我们将对象“Car”的实例存储在链表中。 JAVA 怎么知道如何比较和订购汽车？ 需要的是我们告诉 JAVA 我们只会通过addSorted方法向LinkedList添加元素，我们可以保证它们可以进行比较。

从技术上讲，这是通过将对象传递给实现 Comparable 接口的addSorted方法来表达的。 这些类型的对象应该实现一个 CompareTo方法，该方法实现比较。

因此，与其说是void addSorted(Object data)，不如说是void addSorted(Comparable data)。 但这只有在存储在链表中的元素实现 Comparable 接口时才有可能。 因此，一连串的变化从这里开始。

在迄今为止开发的所有代码中，需要将所有出现的“Object”替换为“Comparable”，以便能够保持所有数据结构的接口和用法一致。

一旦完成，下面的代码就是addSorted方法在语法上正确的 JAVA 代码。

public void addSorted ( Comparable o )
{
    // an empty list , add element in front
    if( head == null ) head = new ListElement (o , null );
    else if( head . first (). compareTo ( o ) > 0)
    {
        // we have to add the element in front
        head = new ListElement (o , head );
    }
    else
    {
        // we have to find the first element which is bigger
        ListElement d = head ;
        while (( d . rest () != null )&&
               ( d . rest (). first (). compareTo ( o ) < 0))
        {
            d = d . rest ();
        }
        ListElement next = d . rest ();
        d . setRest (new ListElement (o , next ));
    }
    count ++;
}
