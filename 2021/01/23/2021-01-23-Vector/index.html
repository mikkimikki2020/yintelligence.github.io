Vector

VUB 4013759ENR Algorithms and Data Structures

Video + Slide 35 ~ 45

Vector是最简单的数据结构之一。 在最简单的形式中，它只不过是一个array。 但是，在本章中，我们将使用各种有用的方法扩展array的functionality，从而生成vector。

Vector将支持向其添加元素、访问元素、删除所有元素、删除特定元素、检查check向量是否为空、检索retrieve其中元素的数量、验证verify特定元素是否是vector的一部分等功能。 所以，显然不仅仅是简单的array。 为了实现所有这些方法，我们将从array开始并为其添加功能。

The basic operations on a vector

Functionality

  • getFirst: returns the first element of the vector.
  • getLast: returns the last element of the vector.
  • addFirst: adds an element to the front.
  • addLast: adds an element to the end.
  • get: gets an element at a given position.
  • size: returns the number of elements in the vector.
  • contains: verifies whether the vector contains a given element or not.
  • removeFirst: removes the first element.
  • removeLast: removes the last element.

Creating a vector

当我们想要创建一个vector时，我们需要指定vector的大小。 这是必要的，因为我们将在顶部构建的array需要具有fixed size。

Vector 类有一个构造函数constructor，它接受一个argument，即向量的capacity。 在constructor中，array被分配并存储在数据变量中。 变量count将存储vector中的元素数量，在constructor中，它被设置为0。

public class Vector{
    protected Object data[];
    protected int count;
    
    public Vector(int capacity){
        data = new Object[capacity];
        count = 0;
    }
}

Accessing an element in a vector 访问元素

Vector 类支持 get 方法来retrieve给定位置的元素。 此处显示的实现假定该位置有效：它不应为负数且不应大于或等于列表中的元素总数。

public Object get(int index){
    return data[index];
}

Adding an element to a vector 添加元素

addLast方法将在vector的末尾添加一个新元素。 此处显示的实现假设在向量的末尾有一个可用的空位置，并且只是将新元素存储在正确的位置，而不进行任何检查。

public void addLast(Object o){
    data[count] = o;count ++;
}

public void addFirst(int item){
    for(int i=count;i>0;i−−){
        data[i]= data[i−1];
    }
    data[0]=item;
    count++;
}

Overwriting an element in a vector 覆盖元素

public void set(int index, Object obj){
    data[index] = obj;
}

Getting the size of a vector, checking whether a vector is empty 检索数量, 检查是否为空

public int size(){
    return count;
}

public boolean isEmpty(){
    return size() == 0;
}

如果有一个方法size可以返回vector中元素的总数，那么实现方法isEmpty很容易：调用size方法并检查它是否返回 0。

请注意，在上面的代码中，没有 if 语句需要完成此操作。

Checking whether the vector contains a given element

public boolean contains(Object obj){
    for(int i=0;i<count;i++){
        if(data[i]. equals(obj))
            return true;
    }
    return false;
}

Other operations on vectors

Binary Search 二分搜索法

A divide and Conquer approach

当vector中的元素sorted时，可以构建更有效的方法来查找vector中的元素。在二分搜索法Binary Search中，vector在中间被分成两部分，并决定元素应该在vector的前半部分还是后半部分。然后，仅在vector的一半中继续搜索。同样，这个sub-vector在中间被分割，并决定元素应该在sub-vector的两半中的哪一个。这个过程在较小的sub-vector上一遍又一遍地重复，直到vector只有一个元素。

  • First check whether the element is supposed to be in the first or the second half of the vector.
  • This way only half of it needs to be searched for.
  • In the right half of the vector, check whether the element should be in the first or the second half.

上述过程如图所示。在该示例中，检查元素15是否出现在vector [1 5 9 14 15 16 22 37]中。在第一步中，将vector中间的元素（在本例中为 14）与搜索关键字（在本例中为 15）进行比较。由于搜索关键字大于中间元素，因此该元素应位于vector的右半部分。因此，在第二步中，搜索继续在vector的右半部分，即 [15 16 22 37]。这个较小vector的中间元素是 16。由于搜索关键字小于中间元素，因此该元素必须位于vector的左半部分。因此，在第三步中，继续搜索[15 16]。中间的元素是15，这是我们的关键。所以，我们找到了元素。

在下面给出的实现中，vector的元素实际上没有被复制到更小的sub-vector中。使用变量start和变量stop来标记较大vector中sub-vector的开头和结尾更有效。

public boolean binarySearch(Object key){
    int start = 0;
    int end = count - 1;
    while(start <= end){
        int middle = (start + end + 1) / 2;
        if(key < data[middle ]) 
            end = middle -1;
        else if(key > data[middle ])
            start = middle + 1;
        else return  true;
    }
    return  false;
}

Some Remarks

  • Simple, yet useful. Vector will be the base for building more complex structures.
  • Implementation is limited to a vector containing integers.
  • We need generic data structures in order to build a reusable tool set.
  • Using Object, Comparable, Generics, ...
  • 简单但有用。 Vector将是构建更复杂结构的基础。
  • 实现仅限于包含integer的vector。
  • 我们需要通用数据结构来构建reusable的工具集。
  • 使Object、Comparable、Generics泛型、...

  6. 添加方法reverse, 它颠倒reverse了vector元素的顺序。 这种方法不应该创建一个新的vector，而是真正改变当前的vector。

    public void reverse() {
    Object x;
    int n = count;
    for (int i=0;i<(count/2);i++){
        x=data[i];

  7. 实现方法“Vector repeat()”，它返回一个重复vector元素的新vector 。 假设您有一个名为v1的向量，其中包含元素 [1 2 3]。 v1.repeat() 的结果应该是一个包含元素[1 1 2 2 3 3]的新vector ，而 v1 保持不变。 实现重复方法。 给出你的方法的时间复杂度并激励它。

    public Vector Repeat() {
    Vector vx = new Vector(2*count);
    for (int i=0;i<count;i++) {
        vx.addLast(data[i]);
        vx.addLast(data[i]);
    }
    return vx;
}

  8. 实现方法“Vector interleave(Vector v2)”，它交错两个vector。 假设您有一个名为 v1 的vector，其中包含元素 [1 2 3]，另一个名为 v2 的vector包含元素 [4 5 6]。 v1.interleave(v2) 的结果应该是一个包含元素 [1 4 2 5 3 6] 的新vector，而 v1 和 v2 保持不变。 我们确实假设两个vector具有相同的长度！ （你真的可以扩展这个方法来处理不同长度的vector吗？）给出你的方法的时间复杂度并激励它。

    public Vector interleave(Vector v2) {
    int n=count+v2.size();
    Vector v3=new Vector(n);
    for (int i=0;i<count;i++) {
        v3.addLast(data[i]);
        v3.addLast(v2.data[i]);
    }
    return v3;
}

  9. Vector 类的缺点之一是它有一个固定的capacity：向量不能包含比由capacity参数定义的元素多，在构造函数中指定。 向vector类添加一个方法“private void extendCapacity()”，它使vector的capacity加倍。 这分 3 个步骤完成：

    1. 创建一个名为data2的新数组，它可以包含“2*capacity”元素。
    2. 将data中的元素复制到data2中。
    3. 将data设置为 data2。

    更改vector类的 addLast 方法，使其检查vector是否已满。 如果vector未满，可以像现在一样添加数据。 当向量已满时，您应该在实际添加元素之前首先调用extendCapacity。

    private void extendCapacity() {
    int n = data.length;
    Object data2[]= new Object[2*n];
    for (int i=0;i<count;i++) {
         data2[i]=data[i];
    }
    data=data2;
}

2011-2012 September:

Gnome Sort is an algorithm based on a technique used by gnomes to sort flowers by their height. Assume a row of n flowers where each flower has a position between 0 and n-1. A gnome at position i can see the flower at his current position, the previous flower (i-1) and the next flower (i+1).

Step by step, the gnome will move from position 0 to position n-1. At each step the gnome looks at the current flower and the next flower. If both flowers are in the right order, he moves on to the next flower. In case the flowers are not in the right order, he will switch both flowers and move back to the previous flower.

There are two exceptions: when the gnome cannot move to the previous flower, because he is in the beginning of the row, he will move on to the next position; if he cannot move to a next flower, because he is at the end of the row, his job is finalized.

  1. Implement a method gnomeSort that sorts a Vector according to the given methodology.
  2. Implement a method gnomeSort that sorts a Double Linked List according to the given methodology.
  3. What is the time complexity of both methods? Motivate your answer.
public void gnomeSort()
{
    int temp = 0;
    if (data[0] > data[1])
    {
        temp = data[1];
        data[1] = data[0];
        data[0] = temp;
    }
    int i = 1;
    while (i < count-1)
    {
        if (data[i] > data[i+1])
        {
            temp = data[i+1];
            data[i+1] = data[i];
            data[i] = temp;
            i--;
        }
        else
            i++;
    }
}

For the vector [7 1 7 74 21 7 9 26 10] the result might for instance be [74 10 26 17 7 21 9 7] (other solutions possible).

2012-2013 September: A mathematical puzzle

A well-known mathematical puzzle is the following one: suppose you have a row of 1000 coins. All coins are the same, they have two sides: up and down. Let us number the coins from 1 to 1000 to identify them. All coins are facing up. Now, you first flip every second coin, so this is coin 2,4,6, ..., 1000. In a second step, you flip every third coin, so this is coin 3, 6, 9, ... This way, you continue flipping coins, until in the very last step, only the last coin is flipped. The challenge in this puzzle is then to tell what coins are facing up after this procedure.

In this question, you will make a computer program that executes all steps form the puzzle. We will store coins in a Vector. Each coin is simply represented by a Boolean.

  1. Make a method “initialize” that takes no arguments and returns a vector of 1000 coins, all facing up.
  2. Make a method “solve” that takes the initialized vector as an argument, and solves the puzzle by simulating all steps of the puzzle.
  3. (extra credit) Which coins are facing up at the end?
