Introduction

  • The problem :
    • Complexity of networked systems
  • The solution :
    • Decomposition into independent subsystems

电信系统可能非常复杂。为了控制这种复杂性，有必要将系统细分subdivide为一定数量的子系统subsystem，这些子系统应尽可能彼此独立。

从IBM在70年代初宣布的SNA开始，已经将计算机网络系统地设计为一组独立的功能层functional layers。

现在更名为“国际电信联盟 International Telecommunications Union”和“国际标准化组织 International Standardization Organization”的“法国电信委员会 Comité Consultatif pour la Télégraphie et la Téléphonie”已在70年代后期试图开发一种分层系统的正式框架，称为“Seven layers OSI (Open Systems Interconnection) model”（ITU recommendation X200, ISO standard 7498)。

他们提出的七层框架 seven layers framework的概念和broad lines被广泛用于描述网络，但是现在实际协议已被互联网标准Internet protocols所取代，而这些协议是独立于这些标准化机构开发的。

Layered network model

  • Introduced by IBM ( SNA , 1971 )
  • Standardized by ITU ( X200 ) and Iso ( 7498 )
  • Method universally adopted
  • Standards not really obeyed in practice

Example: a dialomatic negotiation

首先将通过一个假想的例子来解释分层layered分解decomposition的原理。

为了理解layered model的基础思想，将以与网络问题相同的方式来处理与虚构的国际谈判有关的技术问题。

  • X和Y的两位总统分别身在不同的城镇Tx和Ty，只讲不同的语言Lx和Ly，他们想讨论一些重要的问题。

    这可以通过在此进行总结来说明，他们（隐含地）同意了他们遵循的一些“外交规则”，以交换政治论点。

  • 但是，为了进行谈判，他们需要一些支持服务，这将使他们能够克服由于语言差异和距离而引起的困难。

  • 另外，他们希望这些服务的提供方式应使谈判的内容保密。换句话说，他们应该需要某种安全的电话服务，还提供翻译。

Principles of layered model 原理

在前面的示例中，按照此图中示意性表示的一些一般原理，将复杂的通信问题分解为4个独立的层。

  • 每一层解决一个特定的问题。
  • 第n层到第n + 1级到第n级服务访问点Service Access Points提供服务。
  • 从第n层的角度来看，第n + 1层的不同实体由其级别n地址address标识。
  • 级别n的实体根据专门针对该层达成的一些规则（语言Lc，加密密钥Kc和频率Fc）相互通信。 这种“水平”协议称为“ protocols of level n（n级协议）”。

Example: The telephone Service

可以使用传统电话网络作为示例来阐明引入的概念：

  • 电话服务在两个电话接入点（电话机）之间传送声音。
  • 电话网络具有两个以上的服务访问点（实际上超过109个），并且所有这些访问点均通过其地址（电话号码）进行标识。

Protocol Data Unit 协议数据单元 (PDU)

重要的是要注意，级别为n的实体之间相互通信，而没有它们之间的直接通信通道，它们通过虚拟通道virtual channel进行通信，该virtual channel由基础层实现。

每一层直接都可以认为与同层进行了虚拟的通道传播（Virtual Level n Communication Path)，虚拟的通道里传输的东西可以理解为，两个层直接商量好的数据单元（Protocol Data Unit)。

  • 从一个第n级实体发送到另一实体（在OSI行话中称为“level n Protocol Data Unit”或nPDU）的消息实际上将被委托给n-1层，该层将其封装在（n-1）PDU，依次将其委托给n-2层等。
  • 最终，最终以物理方式传输的消息可能比在级别n生成的消息要长得多，因为所有中间层都将自己的服务信息添加到其中。

为了进一步阐明所涉及的机制，可以在这张图中考虑n + 1级的两个实体，即两个人A和B，A希望通过快递服务向B发送一封信。

  • 快递服务是通过n级实体访问的：员工将A的信件放入特定的信封中，其中包含快递服务正确地将信件送到B所需的所有服务信息。
  • 然后，将这封信的信封中的内容运送到B镇的快递服务办公室。此运输由一级（n-1）级服务提供商完成。
  • 在B的城镇中，快递服务的另一名员工将信封带到B的办公室，从信封中取出信件，丢弃信封，然后将信件交给B。

如上图所示，甚至有可能一个（n + 1）PDU导致多个nPDU。

  • 例如，当级别n协议要求在可以传输任何数据之前建立连接并且连接的打开和关闭确实需要交换许多消息时，就是这种情况。

Approaches to communications 通信方法

存在两种根本不同的通信方法。

  1. Connected oriented networks
    • 传统的电话系统完全面向连接
  2. Connectionless networks
    • 邮政系统以无连接方式运行

重要的是要牢记面向连接connected oriented或无连接connectionless是特定协议的属性，适用于分层系统中的单个层。

It is perfectly possible to use connectionless protocol at layer n while layers n+1 and n-1 are connection oriented.

当第n + 1层和第n-1层面向连接时，完全可以在第n层使用无连接协议。

在特定的电信环境中，几乎所有组合都是有意义且有用的。

Connection oriented communication 面向连接通讯

通过电话进行的通信可以分解为三个明显不同的阶段：

  1. Establish connection
    • using the signalling protocol
  2. Use connection
    • using an informal protocol
    • being billed for duration of usage
  3. Terminate connection
    • Terminate connection

在面向连接的协议中，两个地址之间的virtual link被建立，使用和删除。

  1. 根据用户的请求，网络管理在两个地址a和b之间建立连接。
    • 用户必须在其请求中指定地址a和b以及所请求连接的特征
    • 网络管理根据可用资源选择路由route，或者，如果没有足够的资源可用，则拒绝请求
  2. 建立连接后，用户可以在a和b之间传输数据，而不必再赘述这些地址
    • 通常，网络管理会连续监视所有连接以记帐它们的使用情况并检测并纠正可能的故障。
  3. 最后，用户请求关闭连接

在开始进行数据传输之前，必须完成许多工作，但是一旦建立连接，数据传输就非常简单和高效。

Main points 关键点

  • Network has a state : the list of active connections
  • Network is fragile : state can be lost
  • Application domain
    • Very efficient for transferring high volumes of data
    • Mandatory if stringent Quality of Service request
    • Poor initial response times due to connection state
    • Inapplicable if nodes have high failure rate.

在面向连接connection oriented的协议中，由于发送方和接收方具有状态变量，因此可以对连续的消息进行编号，以允许接收方向发送方发送ack（确认）或否定确认（通知）。然后，后者可以保留已传输的消息，直到它们被确认为止，然后重新传输已被否定确认的消息。

  • Guarantee Quality of Service
    • Reservation of transmission capacity of connection establishment
    • Monitoring of data transfer and correction of transmission errors by retransmission (sliding window protocols)
Transmission errors detection 传输错误检测

通过将冗余redundant信息添加到每个消息（例如，消息的所有字节之和sum of all bytes of the message），可以检测传输错误transmission errors。

在无连接协议connectionless protocol中，对错误消息的唯一可能处理是忽略它们，而将丢失消息的处理留给更高层。

Sliding window protocols 滑动窗口协议

某些协议一一发送消息，等待ack后再发送下一条消息。其他协议（用于长途往返延迟的链接）在等待ack的同时继续发送消息。这种协议称为“滑动窗口协议sliding window protocols”，窗口的大小是在收到确认之前可以发送的消息数。

Connectionless Communication 无连接通讯

在邮政服务中，无需进行任何初步设置即可将普通信件从A点发送到B点。将信件及其目的地地址和服务付款证明放在A的信箱中即可。邮局将信件收集在信箱中，按目的地分类，将其运输到目的地，然后将其放入所需的邮箱中。

  • 没有建立任何连接，没有保留有关各个字母的记录。
  • 由于放置在不同信箱中的字母的数量差异很大，因此无法保证对所有字母进行分类和交付需要多长时间。

这些服务非常简单：用户将数据包传输到服务提供商，后者试图在包中包含的地址尽快传递这些包。

  • 建立连接不需要初始延迟initial delay
  • 但是必须为每个数据包独立地确定路由，即使它们都具有相同的目的地
  • 并且由于只能在统计的基础上保留资源，因此严重的延迟甚至数据包丢失都可以由拥塞congestion问题引起。
  • 由于每个数据包都是分开处理的，因此不保留任何连接列表，并且通常不保留任何已发送数据包的记录。

当必须将少量数据small volumes of data发送到各个目的地并且响应时间很短short response times时，无连接通信connectionless communications似乎就足够了。

Main points 关键点

  • The network is stateless.
  • Network is robust
    • No state stored in any node
    • No error correction by retransmission
  • Application domain
    • whenever single message response times are critical
    • whenever nodes are unreliable
  • No guaranteed Quality of Service
    • No reservation of transmission capacity
    • No error correction by retransmission

在无连接协议connectionless protocol中，对错误消息的唯一可能处理是忽略它们，而将丢失消息的处理留给更高层。

No connection establishment and each packet decided independently even if they have same destination.

没有连接建立，每个数据包独立，即使它们具有相同的目的地。

Sometimes there are severe delays and packet loss caused by congestion problems.

有时会出现拥塞问题导致的严重延迟和丢包。

Layered models for communication networks

Two layer model

仅基于两层的第一个模型在“连接性connectivity和互操作性interoperability”之间建立了区别。

“普通的旧电话服务”示例清楚地表明了区别：由于采用了全球范围的编号方案，并且所有国家电话网络都实现了互连，因此在世界上任意两个点之间建立电话连接非常容易，因为在这两点之间传送声音。这是全球连接。

但是，如果人们希望通过电话进行互操作interoperate（做有意义的事情），则他们不仅应该能够交换声音，而且还应该能够理解彼此的声音，即在对方讲话时进行聆听，反之亦然，以便使用通用的语言和谈论他们都很了解的主题。

在此模型中，称为“网络层networks layer”的下层提供连接服务，而称为“应用层application layer”的上层处理使连接有用的其他问题。

  • 即使在单个连接网络中，也可能需要使用不同的通信技术，因为距离可能相差很大，并且某些用户的位置固定，而另一些用户移动。不同的网络实体必须共存。
  • 由于单个网络服务可用于广泛的应用程序，因此需要将几个不同的应用程序实体与不同的网络实体接口interface。
    • 实际上，如果存在n个不同的网络实体和m个不同的应用程序实体，则需要使n × m个接口interface可用。

Three layer model

为了减少网络实体和应用程序实体之间的interface数量，通常在网络和应用程序层之间插入一个称为“ Internet and Transport layer”的附加层。这样，仅需要n + m而不是n × m接口。

此外，中间层可用于将某些网络提供的服务增强到某些应用程序所需的服务级别。

  • 例如，某些应用程序需要面向连接的连接服务，而可用的网络仅提供无连接服务。 在这种情况下，中间层可以将连接添加到基础服务。

Example: WWW

例如，可以考虑使用万维网。

在应用程序级别，可以找到http协议以及作为客户端应用程序实体的浏览器（例如Explorer或FireFox）和服务器端的Web服务器软件（例如Apache）。

中间层实现TCP和IP协议。 基础网络是由各种Internet服务提供商（ISP）提供的网络，例如比利时，Proximus，Telenet或Belnet。

在这些ISP的网络之间的互连点上，可以找到IP实体，但找不到TCP实体，因为只能与应用程序实体一起找到传输实体。

The seven layers of OSI model

“开放系统互连Open System Interconnection”模型使用7层而不是3层。

3层模型的应用程序层可分为三层：

  1. Application Layers - makes the connectivity useful
    1. Application层提供了网络软件和特定应用程序之间的“应用程序接口Application Programmers Interface”（API）。
    2. Presentation层（6）处理有关数据语法data syntax和语义semantic的问题
    3. Session层控制应用程序同步synchronization（一个应用程序应在另一个发送时监听）
  2. Internet and Transport layers - TCP/IP
  3. Networks Layer - provide the connectivity service
    1. 由三层模型的Internet and Network layers处理的所有寻址addressing和路由routing问题都被分组为一个称为“Network层”的第三OSI层。
    2. Data link层将数据打包在compatible with the physical transmission的帧frame中，并控制这些帧的传输，执行传输错误检测，并在需要时通过重新传输错误的帧进行纠错。
    3. Physical层的底层负责实际数据传输的物理细节，例如通过电波形对位进行编码encoding bits by means of electrical waveforms

试想一种情况，你想和特朗普对话并给他播放B站的鬼畜视频

你需要：

  1. 找到特朗普的电话，拨打给他，于是你们建立了连接（Network Layer)
  2. 找来他的翻译，跟他对话，并给他播放鬼畜视频（Application Layer)

此时你会发现，电话连接并不能给他播放鬼畜视频，于是，你可能需要与特先生建立其他的连接（Network Layer)，并且需要在第一层和第二层之间加一个新的接口（interface），能让特朗普看到你给他播放的鬼畜视频。

所以，每多一种application layer，每多一种network layer，接口的数量都会增加

Interfaces = Applications × Networks

这可不行！你想到你还想给特先生跳支舞，也想特先生给你唱个歌，那得需要更多的接口，更多的层，于是你想到，你可以雇佣闪电侠，让闪电侠把你的意思快速传给特先生。新的一层就创造出来了（Internet&Transport Layer)，这纯粹是因为你太懒，不想搭这么多接口。

Assigning addresses 分配地址

分配地址时，可以使用两种不同的方法：该地址包含有关所有者的一些信息，或者不包含任何信息。

Hierarchical addresses 分层地址

当地址包含某些信息时，可以将地址细分为不同的层次结构字段，每个字段传达一些有关地址的本地化或所有者的信息。这些地址称为“分层地址 hierarchical addresses”。

  • Addresses composed of meaningful fields
  • Address = hierarchical list of domain names
  • A domain is an organizational and / or geographic entity regrouping lower level domains or hosts.
  • Addresses belonging to a given domain are maintained by the authority of that domain .
  • Example : classical telephone numbers
    • 32 2 629 2905 . 32475819327
    • hierarchy jeopardized by number portability !

例如，经典电话号码包含许多有关其本地化的信息：从电话号码32 2 629 2905可以通过查看该号码在比利时的前两个数字（32）并通过查看下一个数字来得出（2），它位于布鲁塞尔。接下来的三位数（629）标识Proximus交易所（埃尔森的Borrensstreet），以及下一位数字（1、2或3）代表VUB的私人交易所。后三位数字标识连接到VUB专用交换机的特定线路。

使用这种分级编号系统，每个国家/地区都可以维护其用户数据库，编号的前几位表示哪个数据库包含相关信息。

比利时的电话号码已失去其部分等级特性，因为电话用户在电话提供商之间进行切换时可以保留原始号码。如果VUB不再是Proximus客户，它将不再实际连接到Borrensstreet中的交易所。它将具有属于另一个运算符的另一个（不可见）数字。当呼叫VUB时，将有一个端口号数据库

Flat addresses 单位地址

由于组织方案不再必须与物理实现physical implementations相匹配，因此保持分层地址的一致性变得越来越困难。

  • Organizational and geographic hierarchy do not necessarily match
  • Address values contain no useful information
  • Example : Internet network ID'S
    • VUB = 134.184
    • ULB = 164.15
  • Using large set of flat addresses user-unfriendly
  • Solution
    • Assign a flat and an hierarchical address to every user and maintain a database linking both

因此，通常会决定物理地址不应传达任何上下文信息（可能是分配日期，因为大多数地址随后会按顺序分配），但是，由于单一数据库应包含所有地址，因此这种平面寻址方案给目录数据库带来了沉重负担。

通过为用户分配物理单位地址physical flat addresses和逻辑分层地址logical hierarchical addresses可以解决此问题。

基于分层逻辑地址，可以在映射逻辑和物理地址的本地维护数据库的分层结构中分解目录。

Domain Name System域名系统（DNS）为Internet用户实现了这种分布式目录。

  • 因为地址可能会发生改变，需要一个不体现物理信息的地址
  • 随机分配的地址，所有地址都是平行的
  • 还是想要看归属信息？--数据库

Domain Name Servers 域名服务器 (DNS)

DNS (Domain Name Servers, 域名服务器) 是一种分布式目录distributed directory，它允许具有稳定地址stable address的任何用户以用户友好的方式访问互联网。

  • In Each domain , there is at least one DNS

    在每一个节点的区域下存在的数据库服务，存着Flat Addresses以及Hierarchical Addresses，可以进行转换

    • Name DNS Address of all subdomains
    • Name address of all hosts in the domain
    • DNS address of , at least , root of DNS system
    • Names DNS addresses of frequently used domains

  • Each host must know the address of one DNS

  • Address resolution can be done recursively or by successive calls to different DNS

  • Local DNS is locally maintained

  • Caching greatly improves performance of DNS by keeping the translations of most recently accessed domains and hosts

Internet Domains

互联网上的每个主机都在其物理互联网地址physical internet address（当前为32位二进制数，逐渐由128位数字代替）旁边，具有一个适合域层次结构hierarchy of domains的逻辑地址logical address 。

每个domain可以包含其他domain和本地主机。 在每个域domain中，一个称为“域名服务器Domain Name Server”（DNS）的本地数据库将保留

  • 子域列表 list of subdomains
  • 本地主机列表 list of the local hosts
  • 相应的物理地址 corresponding physical addresses
    • 子域的物理地址是所在DNS服务器的地址

Session Initiation Protocol 会话发起协议 (SIP)

SIP为由于例如移动性而具有快速更改地址的用户补充了DNS。

  • The SIP server can handle both diversity in terminals and mobility of terminals

为了管理移动性mobility，这意味着在不中断进出移动设备的数据流flow of data的情况下更改设备的地址，每个移动用户都需要有一个稳定的参考地址reference address。

当移动用户处于活动状态时，他会将其移动设备（例如智能手机）记录在其SIP服务器中。 SIP服务器维护一个已登录用户logged- in users的数据库，其中包含其各自的地址和其设备的属性。

当用户的地址由于移动性而更改时，SIP服务器将通过类似于移动电话切换的过程自动更新其数据库。

Redirection mode 重定向模式

当某人想与SIP用户建立语音连接时，首先与该用户的SIP服务器建立连接，该连接会将主叫方的当前地址通知主叫方。

  • Most efficient data transfer
  • Caller has access to actual called address

每当被叫方更改其地址时，SIP服务器都会向呼叫者通知要使用的新地址。

使用重定向模式Redirection mode的SIP服务器会严重损害被呼叫者的隐私，因为任何呼叫者都可以通过被呼叫者的地址确定其物理位置。

Proxy mode 代理模式

SIP服务器通常扮演被呼叫者的代理角色，将呼叫者与被呼叫者之间的实际数据传输转发给自己。

  • Longer transmission delays
  • SIP server can be bottleneck
  • Caller has no access to the actually called address

当然，当许多用户共享同一台SIP服务器时，这样的SIP proxy server可能成为瓶颈。

World Wide Web

最初旨在访问分布式超文本的WWW已成为任何计算机或智能手机与最终用户在其设备（应用程序）上运行的交互式应用程序之间的标准化接口。

万维网是Tim Berners Lee和Robert Caillau最初在“欧洲核子研究中心”（CERN）开发的应用程序，它使物理学家可以与统一的用户进行协商，以访问位于互联网上任何地方的各种数据库。随着网络很快成为营销和信息传播的强大工具，WWW应用程序已成为迄今为止互联网上最常用的应用程序。

  • Uniform , hypertext based , user friendly interface for distributed databases
  • Inexpensive , high quality , browsers available for almost all computers
  • Well over 109 pages available worldwide , mainly for public relations , publicity and electronic commerce
  • As executable programs can be part of hypertext, web browsers are now a universal run-time environment for end-user applications ( apps )

大多数防火墙都阻止可执行程序，以避免病毒入侵，但由于每个人都希望它们而不能阻止网页。 由于超文本可以包含隐藏在网页中的可执行程序，因此，Web浏览器现在是执行通过Internet（“apps”）加载的程序的标准环境，因此，需要在每个Web浏览器中都包含复杂的安全性。

What is the Web?

Basically, the web has three key ingredients:

The Web = Internet + HTML + URL + HTTP

  1. The Hypertext Mark-up Language (HTML)
  2. The Universal Resource Locators (URL)
  3. The Hypertext Transfer Protocol (HTTP)

HTML (Hypertext Mark-up Language)

HTML是一种功能强大的语言，用于描述包含文本，图片，声音甚至可执行程序的页面

HTML页面也可以包含超链接，实际上是特定页面的URL，单击与超链接关联的图标时，将提取超链接引用的页面。 某些超链接与图标不关联，并且每当显示包含超链接的页面时，都会提取相应的页面。 此类“自动”超链接用于，例如，将广告添加到网页

  • Multimedia document
    • normal text
    • graphics and images (stored in separate files)
    • sound (stored in separate files)
    • executable programs
  • Anchors - references to other hypertext documents
    • clickable normal text or image (icon)
    • address (URL) where the corresponding document can be found

URL (Universal Resource Locators)

URL实际上是扩展的Internet地址，不仅提供了主机的地址，而且还提供了所需页面的主机上的标识，用于传输该页面的协议以及该页面上的应用程序级别实体的标识。 能够处理该协议的主机。

Protocol used to communicate with resource

  • each protocol further defines URL format

Internet Resource address includes:

  • Domain name
    • Internet Protocol address of machine ( 134 . 121 023 . 001 )
    • Domain Server Name of machine(www.vub.ac.be)
  • Port number on machine
    • Optional : Each protocol has a default port number

Resource details (protocol specific)

  • i.e. directory and filename of an HTML page

HTTP (Hypertext Transfer Protocol)

Web应用程序显然是“一对多many to one”信息流的一个示例，因为来自许多不同网站的数据可以在客户端的浏览器中合并。

“超文本传输协议HTTP”是网络上最常用的应用程序级别协议。在Web服务器上，与网站的所有访问者保持连接在应用程序级别上将是一项繁重且无用的任务，尤其是因为这将意味着关闭所有忘记关闭其连接的偶然访问者的连接。 因此，http协议是无连接协议。称为HTTPS的面向连接的加密替代协议通常用于需要连接和/或高度安全性的应用程序。

  • HTTP Client : a Web Browser
    • Presenting information encoded in HTML
    • Translating user interactions Into the HTTP.
  • HTTP Server a process interpreting client requests
    • Sending pieces of hypertext stored in HTML
    • Translating information available in other formats
    • acquiring information from outside the Web
  • Stateless ( connectionless ) communication protocol
  • Uses a connection oriented transport protocol

重要的是要牢记面向连接或无连接是特定协议的属性，适用于分层系统中的单个层。 当第n-1层是面向连接的时，完全可以在第n层上使用无连接协议，反之亦然。 在特定的电信环境中，几乎所有组合都是有意义且有用的。

已经提到HTTP是无连接的应用程序层协议。 但是，当访问站点时，需要将页面从服务器传输到客户端，并且这种传输应该是无错误的，这几乎意味着，如后所述，这是面向连接connection oriented的协议。

因此，HTTP使用TCP，一种面向连接connection oriented的传输协议。 每当从Web服务器请求页面时，在传输整个页面（可能与多媒体附件一起）后，就会打开和关闭TCP连接。

  • Benefit: performance of server
  • Penalty: Server can not keep information about user

由于HTTP是无连接的，因此按定义它也是无状态的，换句话说，应用程序实体没有内存。

这对于诸如家庭银行业务之类的应用程序显然是不可接受的，在该应用程序中，服务器将要求用户对他/她自己进行身份验证，并记住整个银行业务会话的身份验证。通常，此问题可以通过使用其他面向连接的应用程序级别协议HTTPS解决，但是对于不太重要的应用程序，http可以提供一些技巧。

  • HTTP is a connectionless protocol to avoid waste resources.
  • HTTPS is a connection oriented and encrypted alternative protocol for requirinf connection or high level of security.

Cookie

在无状态协议中模拟状态变量的技巧在于来回发送一条特定的，通常不可见的消息（称为“ Cookie”），该消息包含由于无连接协议而无法存储在应用程序实体中的状态变量。

例如，在银行业务中，服务器可以在成功进行身份验证后发回一个密码密钥，用户可以通过该密钥发送请求以查看他/她的银行帐户状态。 出于明显的安全原因，此类密钥应为一次性使用且难以猜测。

Web security

  • Most attacks start with an executable program being installed remotely in a computer.

    大多数攻击始于远程安装在计算机中的可执行程序。

  • Initially , web browsers provided a "sand bag" to run remote programs, prohibiting them to access local resources such as disks

    最初，网络浏览器提供了一个"sand bag"来运行远程程序，禁止它们访问磁盘等本地资源。

  • Making web browsers the run-time environment for a multitude of application programs made " sandbags " too restrictive !

    使网络浏览器成为众多应用程序的运行时环境，使"sand bag"变得过于严格！

  • Only trusted apps should be allowed !!!

    应该只允许受信任的应用程序！！！
