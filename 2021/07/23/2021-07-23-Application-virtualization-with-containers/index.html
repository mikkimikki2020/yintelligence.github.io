1. Containers 容器

Containers virtualize the OS environment for an application, similar to how hypervisors (type 2) virtualize the hardware

容器为应用程序虚拟化操作系统环境，类似于hypervisors (type 2) 如何虚拟化硬件

  • Containers wrap up an application in a lightweight image that contains all essential binaries and libraries to operate.

    容器将应用程序封装在一个轻量级映像中，该映像包含所有必需的二进制文件和要操作的库

    类似集装箱

  • Containers operate on top of the (Linux) kernel in combination with the container engine

    容器结合容器引擎container engine在（Linux）内核之上运行

1.1 Features

  • Standardized packaging for software and dependencies

    软件和依赖项的标准化打包

    • Easy to distribute and deploy

      易于分发和部署

  • Isolate applications from each other

    相互隔离应用程序

    • Eliminates issues with incompatible dependencies

      消除了不兼容的依赖关系的问题

  • Share the same underlying OS kernel

    共享相同的底层操作系统内核

  • Support for all major Linux distributions

    支持所有主要的 Linux 发行版

    • VMs, Physical hosts, cloud instances

    • Ability to move around between machines without any modification

      无需任何修改即可在机器之间移动的能力

    • Ability of containers to work together

      容器协同工作的能力

      Docker is the most popular container environment today.

  • Containers also native to Windows Server 2016

    容器也是 Windows Server 2016 的原生容器

1.2 Containers vs. VMS

Containers ≠ VM

  • Easy connection to make 易于连接
  • Fundamentally different architectures 完全不同的架构
  • Fundamentally different benefits 根本不同的好处

1.2.1 Lightweight 轻量级

VMs

Every app, every copy of an app, and every slight modification of the app requires a new virtual machine.

每个应用程序、应用程序的每个副本以及应用程序的每个细微修改都需要一个新的虚拟机。

Containers

  • Original App

    • No OS to take up resources & start up

  • Copies of App

    • Can share storage of app and bins/libs via Overlay file system

      可以通过 Overlay 文件系统共享应用程序和 bins/libs 的存储

    • Shared volumes 共享卷

  • Modified App

    • Copy on write capabilities allow to only save the diffs Between container A and container A’

      写入时复制功能只允许保存容器 A 和容器 A' 之间的差异

1.2.2 Compatibility

Support for all major Linux distributions

支持所有主要的 Linux 发行版

  • VMs, Physical hosts, cloud instances

  • Ability to move around between machines without any modification

    无需任何修改即可在机器之间移动的能力

  • Ability of containers to work together

    容器协同工作的能力

1.3 Isolation

Containers share portions of the host kernel

容器共享主机内核的一部分

Containers use:

  • Namespaces
    • per-process isolation of OS resources (filesystem, network and user ids)
  • Cgroups
    • resource management and accounting per process

1.4 Images and containers

2. Docker

Docker is a software platform that allows you to build, test, and deploy applications quickly, packaging software into standardized units called containers.

Docker 是一个软件平台，可让您快速构建、测试和部署应用程序，将软件打包到称为容器的标准化单元中。

  • Lightweight, open, secure platform

  • Simplify building, shipping, running apps

    简化构建、传送、运行应用程序

  • Runs natively on Linux or Windows Server

  • Also runs on Windows 10 or Mac development machines

    • Via Hyper-V (Windows 10) or via a Linux virtual machine

2.0 Terminology

  • Docker Image

    • The basis of a Docker container
    • Represents a full application 代表一个完整的应用程序

  • Docker Container

    • The standard unit in which the application service resides and executes

      应用服务驻留和执行的标准单元

  • Docker Engine

    • Creates, ships and runs Docker containers deployable on a physical or virtual, host locally, in a data center or cloud service provider

      创建、运送和运行可部署在物理或虚拟主机、本地主机、数据中心或云服务提供商的 Docker 容器

  • Registry Service

    Cloud or server based storage and distribution service for your container images

    容器镜像的基于云或服务器的存储和分发服务

    • Docker Hub (Public)
    • Docker Trusted Registry (Private)

  • Docker hub

    • public docker images

2.2.0 Get public docker images

Via Docker hub.

# Create image based on the official Node 6 image from dockerhub
FROM node: latest
# Create a directory where our app will be placed
RUN mkdir-p /usr/src/app

# Change directory so that our commands run inside this new directory
HORKDIR /usr/src/app

# Copy dependency definitions
COPY package.json /usr/src/app

# Install dependecies
RUN npm install

# Get all the code needed to run the app
COPY. /usr/src/app

# Expose the port the app runs in
EXPOSE 4200

# Serve the app
CMD ["npm","start"]

2.2.1 Build docker images

Build Docker images by processing the instructions from a Dockerfile

  • Instructions on how to build a Docker image

  • Very similar to “native” Linux commands

  • Optimize the Dockerfile according to changes

    根据变化优化Dockerfile

    • Intermediate image built for each incremental step

      为每个增量步骤构建的中间映像

      允许加速构建完整图像

2.3 Docker compose

2.3.1 Manage multiple containers

  • Build & run multi-container Docker applications reading instructions from a docker-compose.yml file

    构建和运行多容器 Docker 应用程序，从 docker-compose.yml 文件中读取指令

  • Supports specifying environment variables, volume mount points, network configuration, etc.

    支持指定环境变量、卷挂载点、网络配置等

  • Creates user-defined network bridge for all containers specified in the dockercompose.yml

    为 dockercompose.yml 中指定的所有容器创建用户定义的网桥

version : 2
services :
	my-application:
		build: ./
		ports:
			- "8000:8000'
		environment:
			- CONFIG FILE
	db:
		image: postgres
	redis:
		image: redis
		command: redis-server --save " " --appendonly no ports: 
			-"6379"
