XEN

VUB 4011247EER Operating Systems and Security

1. Description 描述

The goal of the Cambridge group - design a VMM/hypervisor capable of scaling to about 100 VMs running standard applications and services without any modifications to the Application Binary Interface (ABI). (2003, Computing Laboratory, Cambridge University)

Cambridge 小组的目标是设计一个 VMM/hypervisor，能够扩展到大约 100 个运行标准应用程序和服务的 VM，而无需对应用程序二进制接口 (ABI) 进行任何修改。 （2003，剑桥大学计算实验室）

  • Xen is a type-1 hypervisor with a very low footprint

    Xen 是一种占用空间很小的type-1 hypervisor:

    • the ARM port amounts to less than 90K lines of code.

      ARM 端口的代码行数不到 9 万行

  • Linux, Minix, NetBSD, FreeBSD and others can operate as para-virtualized Xen guest OS running on x86, x86-64, Itanium, and ARM architectures.

  • Applications make system calls using hypercalls processed by Xen;

    privileged instructions issued by a guest OS are paravirtualized and must be validated by Xen

    应用程序使用 Xen 处理的hypercalls进行系统调用

    来宾操作系统发出的特权指令是半虚拟化的，必须由 Xen 验证

2. Components 组分

2.1 Xen domain

  • ensemble of address spaces hosting a guest OS and applications running under the guest OS.

    托管来宾操作系统和在来宾操作系统下运行的应用程序的地址空间集合。

  • Runs on a virtual CPU.

    在虚拟 CPU 上运行。

  • Dom0

    • dedicated to execution of Xen control functions and privileged instructions.

      专用于执行 Xen 控制功能和特权指令

  • DomU

    • unprivileged guests O.S.

2.2 XenStore – a Dom0 process.

XenStore is a database of configuration information shared between domains

XenStore 是域之间共享的配置信息数据库

  • Supports a system-wide registry and naming service

    支持系统范围的注册表和命名服务

  • XenStore is often used as a mechanism for controlling devices in guest domains

    XenStore 通常用作控制来宾域中设备的机制

  • A watch function informs listeners of changes of the key in storage they have subscribed to.

    watch 函数通知听众他们订阅的存储中的键的变化

  • Communicates with guest VMs via shared memory using Dom0 privileges

    使用Dom0权限通过共享内存与来宾 VM 通信

2.3 Toolstack

Responsible for creating, destroying, and managing the resources and privileges of VMs.

负责创建、销毁和管理 VM 的资源和权限。

  • To create a new VM, a user provides a configuration file describing memory and CPU allocations and device configurations.

    要创建新 VM，用户需要提供描述内存和 CPU 分配以及设备配置的配置文件。

  • Toolstack parses this file and writes this information in XenStore.

    Toolstack 解析此文件并将此信息写入 XenStore。

  • Takes advantage of Dom0 privileges to map guest memory, to load a kernel and virtual BIOS and to set up initial communication channels with XenStore and with the virtual console when a new VM is created.

    利用Dom0权限映射来宾内存、加载内核和虚拟 BIOS 以及在创建新 VM 时设置与 XenStore 和虚拟控制台的初始通信通道。

2.4 Get access to virtual devices

Dom0 also runs a set of drivers called paravirtualized backends (PV) to give access to

Dom0 还运行一组称为半虚拟化后端 (PV) 的驱动程序来访问

  • Disk
  • Network
  • … the other unprivileged virtual machines

DomU run the corresponding paravirtualized frontends to get access to a set of generic virtual devices.

DomU 运行相应的半虚拟化前端以访问一组通用虚拟设备。

2.5 Driver domains

The Xen architecture allows driver domains:

  • unprivileged virtual machines (DomU) with the only purpose of running the driver and the paravirtualized backend for one class of devices.

    无特权的虚拟机 (DomU)，其唯一目的是为一类设备运行驱动程序和半虚拟化后端。

  • Running in DomU makes the system more secure because they allow large pieces of code, such as the entire network stack, to run unprivileged.

    在 DomU 中运行使系统更加安全，因为它们允许大段代码（例如整个网络堆栈）在无特权的情况下运行。

  • Driver domains also improve isolation and resilience

    驱动程序域还提高了隔离性和弹性

3. Abstraction for networking and I/O

Split drivers have a front-end in the DomU and the back-end in Dom0; the two communicate via a ring in shared memory.

拆分驱动程序在DomU中有前端，在Dom0中有后端； 两者通过共享内存中的环进行通信。

3.1 Ring

A circular queue of descriptors allocated by a domain and accessible within Xen. Descriptors do not contain data, the data buffers are allocated off-band by the guest OS.

由域分配并在 Xen 内可访问的描述符的循环队列。 描述符不包含数据，数据缓冲区由客户操作系统在带外分配。

3.2 Transmit a packet 发送数据包

Two rings of buffer descriptors, one for packet sending and one for packet receiving, are supported

支持两环缓冲区描述符，一环发送，一环接收

见 3.3 第二张图

To transmit a packet:

  1. A guest OS enqueues a buffer descriptor to the send ring

    来宾操作系统将缓冲区描述符排入发送环

  2. Xen copies the descriptor and checks safety

    Xen 复制描述符并检查安全性

  3. Copy only the packet header, not the payload

    只复制包头，不复制负载

  4. Execute the matching rules

    执行匹配规则。

3.3 Example: Xen I/O

Xen data transfer using I/O rings.

  • The communication between a guest domain and the driver domain over an I/O and an event channel

    来宾域和驱动程序域之间通过 I/O 和事件通道event channel的通信

  • NIC is the Network Interface Controller

    NIC 是网络接口控制器

  • The circular ring of buffers 缓冲区的圆环。

4. Strategies

4.1 Virtual memory management

  • Paging

    • A domain may be allocated discontinuous pages.

      一个域可以分配不连续的页面。

    • A guest OS for efficiency : page table updates are batched for performance and validated by Xen for safety direct access to page tables and handles pages faults directly for efficiency

      为提高效率的客户操作系统：页表更新为性能进行批处理并由 Xen 验证以确保安全 直接访问页表并直接处理页错误以提高效率

    • Page table updates are batched for performance and validated by Xen for safety

      页表更新为性能进行批处理并由 Xen 进行验证以确保安全

  • Memory

    • Memory is statically partitioned between domains to provide strong isolation

      内存在域之间静态分区以提供强隔离

    • XenoLinux implements a balloon driver to adjust domain memory

      XenoLinux 实现了一个气球驱动程序来调整域内存

  • Protection

    • A guest OS runs at a lower priority level , in ring 1 , while Xen runs in ring 0.

      来宾操作系统在环 1 中以较低的优先级运行，而 Xen 在环 0 中运行。

  • Exceptions

    • A guest OS must register with Xen a description table with the addresses of exception handlers previously validated

      客户操作系统必须向 Xen 注册一个描述表，其中包含先前验证过的异常处理程序的地址

    • exception handlers other than the page fault handler are identical with r86 native exception handlers

      除页面错误处理程序之外的异常处理程序与 r86 本地异常处理程序相同

  • System calls

    • To increase efficiency , a guest OS must install a "fast" handler to allow system calls from an application to the guest OS and avoid indirection through Xen

      为了提高效率，客户操作系统必须安装一个“快速”处理程序，以允许从应用程序到客户操作系统的系统调用，并避免通过 Xen 进行间接调用。

  • Interrupts

    • A lightweight event system replaces hardware interrupts , synchronous system calls from a domain to Xen use hypercalls and notifications are delivered using the asynchronous event system

      轻量级事件系统取代了硬件中断、从域到 Xen 的同步系统调用使用超级调用和使用异步事件系统传递的通知。

4.2 CPU Multiplexing

  • Multiplexing

    • A guest OS may run multiple applications

  • Time

    • Each guest OS has a timer interface and is aware of "real" and "virtual" time

      每个来宾操作系统都有一个计时器接口，并且知道“真实”和 “虚拟”时间

4.3 IO devices

  • Network and I/O devices

    • Data is transferred using asynchronous I/O rings

      使用异步 I/O 环传输数据

    • a ring is a circular queue of descriptors allocated by a domain and accessible within Xen

      环是由域分配并在 Xen 中可访问的描述符的循环队列

  • Disk access

    • Only Domo has direct access to IDE and SCSI disks

      只有 Domo 可以直接访问 IDE 和 SCSI 磁盘

    • all other domains access persistent storage through the Virtual Block Device (VBD) abstraction

      所有其他域通过虚拟块设备 (VBD) 抽象访问持久存储

5. Xen in cloud

The previous description is great for a single machine, but …

前面的描述非常适合单机，但是……

Today’s private or public clouds often include hundreds or thousands of machines.

今天的私有或公共云通常包括数百或数千台机器。

How to manage the cloud effectively and efficiently becomes a central issue in cloud computing.

如何有效且高效地管理云成为云计算的核心问题。

For instance, Amazon EC2 currently utilizes a highly customized version of the Xen hypervisor, taking advantage of paravirtualization (in the case of Linux guests).

例如，Amazon EC2 目前使用高度定制版本的 Xen 管理程序，利用半虚拟化（在 Linux 来宾的情况下）。
