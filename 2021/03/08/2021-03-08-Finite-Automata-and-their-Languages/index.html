Finite Automata and their Languages

Introduction

  • Recognition is a fundamental computational problem for formal languages:
    • For string x and language L: does x belong to L?
  • The problem is solved by designing an automaton for L:
    • Complete computational model of the recognition process for L
  • The complexity of language classes can be studied by comparing the complexity of the corresponding classes of automata

Automata: The General Idea

  • An automaton is a specification for a symbol processing machine
  • Different from a real machine:
    • It is a specification
    • Operates on symbols
    • Relatively simple internal structure to study
  • An automaton can recognize whether a string is part of the language that is specifies

States

  • As an automaton executes:
    • Changes in its internal condition
    • Each possible internal condition is called a state
  • The states form the internal memory of the automaton
  • A Finite Automaton (FA) has a finite number of states and no other form of memory

Diagrams and recognition

State transition diagram

  • A State Transition Diagram graphically represents an automaton
  • It consists of:
    • States (encircled letters)
    • Start State (state with short arrow pointing to it)
    • Transitions (arrows from one state to another)
    • Accepting State (double encircled letters)
  • A string is accepted if it leads from the start state to an accepting state
  • The set of strings that an automaton accepts is called its language

Example

  • Start State: A
  • Accepting State: C
  • Transitions: A → C, A → B, B → C

  • The language of this automaton is {a, bc}
  • E.g. aa passes through an accepting state, but is not accepted itself

Skip/trap sate

  • State D is a sink state (no way out)
  • A non-accepting sink state is called a trap state
  • Trap states clutter diagrams and are usually not drawn

Trap-free diagram

  • Equivalent, trap-free diagram
  • Missing transitions are assumed to lead to the trap state

Proof: L = {A, BC}

  • Prove that: L = {a, bc}
    • Prove: {a, bc} ⊆ L
    • Prove: L ⊆ {a, bc}

Proof: {A, BC} ⊆ L

  • Prove that: {a, bc} ⊆ L
  • Prove that the strings a and bc are part of the language specified by the automaton
  • Recognize both strings using the automaton

Proof: L ⊆ {A, BC}

Prove that: L ⊆ {a, bc}

  • x ∈ L → x ∈ {a, bc}
  • x ∉ {a, bc} → x ∉ L (contrapositive)
  • Prove that no other strings than a and bc are part of the language described by the automaton
    • String has to start by a or b, otherwise it is not accepted
    • If string starts by a, then it cannot be followed by any other symbol, as there is no transition from state C
    • If string starts by b, then it cannot be followed by a or b, if the second symbol is a c, then there cannot be a next symbol

Formal notation for finite automata

Definition: Finite Automata

A finite automaton M is a quintuple (Q, Σ, q0, δ, A), where:

  • Q is a finite set of states
  • Σ is a finite set of symbols
  • q0 ∈ Q, where q0 is the start state
  • A ⊆ Q, where A is the set of accepting states
  • δ : Q × Σ → Q

δ is the transition function, with as domain the set of all ordered pairs (q, σ)

Example

Specify a machine M = (Q, Σ, q0, δ, A) for the language {a} by giving the values of Q, Σ, δ and A*, and also by drawing the corresponding diagram.

Q = {q0},

Σ = {a},

A = {q0} ,

δ(q0, A) = q0.

  • Draw the transition diagram
  • Justify the claim that M accepts all strings over the alphabet beginning with a and no others

Solution

  • f the first symbol is a, the automaton will always stay in the accepting state q1, no matter what other symbols follow
  • If the first symbol is d, it will be trapped in the state q2, no matter what other symbols follow

Closure of the transition function

Definition:

  • M = (Q, Σ, q0, δ, A)

  • δ∗ : Q × Σ∗ → Q

  • For any q ∈ Q, δ∗ (q,Λ) = q

  • For any q ∈ Q, any σ ∈ Σ and any x ∈ Σ∗ , δ∗ (q, xσ) = δ(δ∗ (q, x), σ)

    Note that the definition is recursive!

Theorem : δ∗ for strings of length 1

  • Theorem: ∀q ∈ Q : ∀σ ∈ Σ for which |σ| = 1 : δ∗ (q, σ) = δ(q, σ)

  • Proof:

    • δ∗ (q, σ) = δ∗ (q,Λσ) Substitution

      ​ = δ(δ∗ (q,Λ), σ) Closure of δ (recursive case)

      ​ = δ(q, σ) Closure of δ (base case)

Example

  • M = ({q0, q1, q2}, {a, d}, q0, δ, {q1})

  • Proof that ad takes M to state q1

  • Proof:

    • δ∗(q0, ad) = δ(δ∗(q0, a), d) Closure of δ (recursive case)

      ​ = δ(δ(q0, a), d) Theorem δ∗ for strings of length 1

      ​ = δ(q1, d) Evaluation

      ​ = q1 Evaluation

Recognized language

Definition: Language recognized by a FA

  • The language L(M) recognized by a finite automaton M = (Q, Σ, q0, δ, A) is the set of all strings x ∈ Σ∗ such that δ∗ (q0, x) ∈ A
  • M accepts the strings of L(M)
  • M rejects the strings of Σ∗  L(M)

Algorithm: Recognizing the language of a FA

  • q: current state, x: input string, σ: current symbol, ⇐: assignment
  • dequeue: remove an element from the front and return it
  • Algorithm returns TRUE (accept) or FALSE (reject)

Nondeterministic finite automata

General idea

In contrast to Deterministic Finite Automata (DFAs), Nondeterministic Finite Automata (NFAs) can have more than 1 transition for a single symbol out of a state

Example

For the finite language L = {ab, ac}, draw an NFA state transition diagram, giving each string of L its own states and transitions, sharing only the start state

Characteristics

  • An NFA makes a series of choices that lead to acceptance of the input whenever this is possible
  • Codomain of δ:
    • DFA: Q : Set of states
    • DFA: 2Q : Powerset (all subsets) of the set of states

Definition: nondeterministic FA

  • M = (Q, Σ, q0, δ, A)
  • δ : Q × Σ → 2Q

Definition: nondeterministic generalized transition function

  • M = (Q, Σ, q0, δ, A)

  • δ : Q × Σ → 2Q

  • For any q ∈ Q, δ∗ (q,Λ) = {q}

  • For any q ∈ Q, any σ ∈ Σ, and any x ∈ Σ∗ \[ δ^∗(q,xσ)=∪_{p∈δ^∗(q,x)}δ(p, σ) \]

Definition: Language recognized by a NFA

  • The language L(M) recognized by a NFA M = (Q, Σ, q0, δ, A) is the set of all strings x ∈ Σ∗ such that δ∗(q0, x) ∩ A ≠ ∅
  • The NFA accepts the strings of L(M)
  • The NFA rejects the strings of Σ∗  L(M)

Example

Consider the language L = {x | x ∈ {a, b}} and x ends with ab}. This language contains, for example, the string bbab, but not abba. Draw a state transition diagram for both a DFA and an NFA to recognize L*. Make each as simple as possible.

Removing nondeterminism

Simulating NFAs

  • How to turn NFAs into real computational devices?

    • backtracking
    • parallel computing
    • simulate it using a DFA

  • Simulation also proofs that NFAs are not more powerful than DFAs: they recognize the same class of language

  • After consuming a, there is no way to know whether the NFA is in q0 or q1
  • We tolerate this uncertainty: our state of knowledge about the current state is {q0, q1}
  • The states of knowledge become the states of a new, deterministic automaton

Subset construction

Definition

The subset construction, S, is a function, S : N → D, that applies to any NFA, M = (Q, Σ, q0, δ, A), to yield a DFA M′ = S(M) = (Q′ , Σ′ , q′0 , δ′ , A′), where the following relationships hold between the parts of M and M′ :

  • Σ′ = Σ

  • q′0 = {q0}

  • Q′ ⊆ 2Q

  • δ′ : Q′ × Σ → Q′ , where \[ ∀q′∈ Q : ∀σ ∈ Σ : δ ′ (q ′ , σ) = ∪_{q ∈ q′} δ(q, σ) \]

  • A′ = {q′ ∈ Q′ |q′ ∩ A ≠ ∅}

Λ-transitions

General idea

  • Just like nondeterminism, Λ-transitions allow simpler solutions

  • They can also be converted into a deterministic form by simulation

  • An NFA that allows Λ-transitions is called a Λ-NFA

  • Λ-NFA allow Λ as a label on transitions

    • Such a transition consumes no symbol

Example

Λ-NFA formal definition

  • M = (Q, Σ, q0, δ, λ, A)

  • λ : Q → 2 Q

  • For any state qi and input σ: the next state qj ∈ δ(qi , σ) ∪ λ(qi)

    • Using σ if choice was based on δ
    • Not using σ if choice was based on λ
    • Using or not using σ if both sets or non-empty

λ-closure

Yields the set of all states reachable by zero or more Λ-transitions

Definition:

  • n = |Q|

Λ-Elimination

  • M = (Q′ , Σ, q0, δ′ , A′ )

    1. For each q ∈ Q, compute and record λ∗ (q) (needed for the next two steps)

    2. For each q ∈ Q and σ ∈ Σ, compute δ′ , knowing that \[ δ′ (q, σ) = ∪_{p∈λ^∗(q)}δ(p, σ) \]

  1. For each q ∈ Q, if λ∗(q) ∩ A ≠ ∅, then make q a member of A′

Regular languages

Definition: Regular Languages

A language L is a regular language if (and only if) there exists an FA M such that L = L(M)

Closure properties

  • The set of regular languages is closed for:
    • Concatenation
    • Closure
    • Union
    • Intersection
    • Set Difference
    • Symmetric difference
    • Complementation
  • This means that the result of applying these operations to regular languages L1 and L2 always is a regular language itself
  • Closure can be proven by constructing an FA for the result

Closure of L1L2

  • Theorem: If L1 and L2 are regular languages, then their concatenation L = L1L2 is a regular language

  • Proof:

    • L1 = L(M1) where M1 = (Q1, Σ1, q0,1, δ1, A1)
    • L2 = L(M2) where M2 = (Q2, Σ2, q0,2, δ2, A2)
    • New machine L = L(M) where M = (Q1 ∪ Q2, Σ1 ∪ Σ2, q0,1, δ, A2)
    • New transition function contains all transitions from δ1 and δ2 and Λ-transitions from each state in A1 to q0,2
    • Finish proof by showing that L = L(M) = L1L2

Closure OF L1*

  • Theorem: If L1 is a regular language, then L = L1* is a regular language.

  • Proof:

    • New machine L = L(M) where M = (Q1 ∪ {q0}, q0, δ, {q0})
    • New transition function contains all transitions from δ1 and Λ-transitions from q0 to q{0, 1} and from each state in A1 back to to q0
    • Finish proof by showing that L = L(M) = L1*

Closure of L1 ∪ L2

  • Theorem: If L1 and L2 are regular languages, then L = L1 ∪ L2 is a regular language.

  • Proof:

    • New machine L = L(M) where M = (Q1 × Q2, Σ,(q0,1, q0,2) , δ, A)
    • New transition function: δ((q1, q2), σ) = (δ1(q1, σ), δ2(q2, σ)) for every q1 ∈ Q1, q2 ∈ Q2 and σ ∈ Σ
    • New accepting states: A = {(q1, q2)|q1 ∈ A1 or q2 ∈ A2}

Closure of L1 ∩ L2

  • Theorem: If L1 and L2 are regular languages, then L = L1 ∩ L2 is a regular language.

  • Proof:

    • New machine L = L(M) where M = (Q1 × Q2, Σ,(q0,1, q0,2) , δ, A)
    • New transition function: δ((q1, q2), σ) = (δ1(q1, σ), δ2(q2, σ)) for every q1 ∈ Q1, q2 ∈ Q2 and σ ∈ Σ
    • New accepting states: A = {(q1, q2)|q1 ∈ A1 and q2 ∈ A2}
