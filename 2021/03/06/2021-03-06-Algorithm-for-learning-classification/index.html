Classification

Goal: given a set of examples described by several attributes (or features) and a set of classes, label each example with a class

Applications:

  • biometry: identify a person from her voice, face, gait, ...

    生物测定：通过声音、面部、步态等识别一个人

  • category recognition （object, speech, musical instrument,...）

    类别识别（物体、语音、乐器等）

  • data mining: predict the user’s behaviour depending on his past history

    数据挖掘：根据用户过去的历史预测用户的行为

  • medical diagnosis: knowing the symptoms and eventual test results, determine whether a patient is sick

    医学诊断：了解症状和最终检测结果，确定患者是否生病

Why is it hard?

  • The classes themselves are not observable 类本身是不可观察的

    • the classes represent high-level information （identity, behaviour,...） but only low-level attributes （pixel values, numbers of clicks on a web page,...） are observed

      这些类表示高级信息（身份、行为等），但只观察到低级属性（像素值、网页点击次数等）

    • the human brain is good at making sense of data, but we still don’t know how

  • There are many attributes 有很多属性

    • each attribute individually has little correlation with the classes 每个属性单独与类几乎没有相关性

      • so we need a lot of attributes

        所以我们需要很多属性

    • the correlation between the attributes and the classes is not known beforehand 属性和类之间的相关性是事先未知的

      • it has to be learnt

        它必须被学习

  • There are many examples

    • if not, it could be done by hand

      如果没有，可以手工完成

    • learning algorithms are costly

      学习算法成本高昂

Machine learning for classification 机器学习分类

  • Classifier: a function that takes features and returns the class of an example with those features

    分类器：获取特征并返回具有这些特征的示例的类的函数

    • a classifier can label any possible example, not only the problem set

      分类器可以标记任何可能的示例，而不仅仅是问题集

  • Machine learning approach 机器学习方法

    • learn the classifier from a training set, then apply it to the problem examples

      从训练集中学习分类器，然后将其应用于问题示例

      • supervised learning: the classes of the training examples are known

        监督学习：训练样本的类别是已知的

      • the training set is much smaller than the problem set

        训练集远小于问题集

  • The learning step is costly but performed only once

    学习步骤成本高，但只执行一次

  • How can we ensure that the classifier will work on the problem data?

    我们如何确保分类器能够处理问题数据？

    • we cannot for sure 我们不能确定:

    • Occam’s razor: the simplest solution is often the right one

      奥卡姆剃刀：最简单的解决方案往往是正确的

Geometrical view 几何视图

  • The examples lie in the attribute space 示例位于属性空间

    • each attribute is a dimension 每个属性是一个维度

    • each example is a point 每个示例都是一个点

    • classes are represented by colours 类用颜色表示

  • A classifier embodies a partition of the attribute space

    分类器体现了属性空间的划分

  • Learning a classifier is partitioning the training set into homogeneous subsets 学习分类器是将训练集划分为同构子集

    • each subspace contains training examples of exactly 1 class

      每个子空间包含恰好 1 个类的训练示例

  • Applying a classifier is therefore just checking in which subspace a point lies 因此，应用分类器只是检查点位于哪个子空间

Structure of the classifier 分类器的结构

  • The subspaces of the classifier do not have to be connected

    分类器的子空间不必连接

  • The border can be much more complex than a single line

    边框可以比单线复杂得多

  • Generally, the types of borders allowed are set by the algorithm

    一般允许的边框类型由算法设置

    • need expert knowledge of the problem

      需要对问题的专业知识

    • there is no single algorithm that solves every problem

      没有一种算法可以解决所有问题

Attribute selection 属性选择

  • Learning is hard. Learning algorithms in general are both costly and suboptimal. 学习很难。学习算法通常既昂贵又次优。

  • Learning is easier if one starts with “good” attributes

    如果从“好”属性开始，学习会更容易

    • fewer attributes decrease the cost of learning

      更少的属性降低了学习成本

    • a good initial guess helps finding the best classifier

      一个好的初始猜测有助于找到最好的分类器

  • What is a “good” attribute? 什么是“好”属性？

    • easiest case: the classes themselves as an attribute

      最简单的情况：类本身作为属性

    • hardest case （insoluble）: no attributes at all

      最难的情况（不可解决）：根本没有属性

    • an attribute that carries a lot of “information”

      一个携带大量“信息”的属性

    • an attribute whose “information” is related to the class

      其“信息”与类相关的属性

Symbolic and numeric attributes 符号和数字属性

  • Symbolic attributes 符号属性

    • finite, discrete valued: e.g. gender, status,...

      有限的、离散的值：例如性别、地位、...

    • no notion of order or distance between values, only boolean comparison

      没有顺序或值之间的距离的概念，只有布尔比较

    • can be enumerated 可以列举

    • must use logic to combine different attributes after testing them

      测试后必须使用逻辑来组合不同的属性

  • Numeric attributes 数字属性

    • real or integer valued: e.g. grades, temperature, pixels, samples,...

      实数或整数值：例如等级、温度、像素、样本...

    • can be measured 可以测量

    • can use algebra to combine different attributes before testing them 可以在测试之前使用代数组合不同的属性

      • can create new attributes 可以创建新属性

Entropy 熵

Information theory （Shannon 1948）

  • Communication problem: how to compress a signal, i.e. a stream of independent random variables X ?

    通信问题：如何压缩信号，即独立随机变量 X 的流？

    • e.g.: signal = succession of coin tosses

      例如：信号 = 连续抛硬币

  • Quantity of information in the signal = minimum bit rate needed to transmit it

    信号中的信息量 = 传输它所需的最小比特率

    • i.e. the average number of bits per symbol.

      即每个符号的平均位数。

  • It is a function of the probability distribution of the signal 它是信号概率分布的函数

    • the distribution represents what is known before the actual transmission, i.e. what does not depend on the number of samples

      分布代表在实际传输之前已知的内容，即不依赖于样本数量的内容

    • the bit rate should be chosen before transmission

      传输前应选择比特率

  • The more uncertain the distribution, the more information in the content 分布越不确定，内容中的信息就越多

  • if the coin is double-headed, then there is nothing to transmit

    如果硬币是双头的，那么就没有什么可传输的

  • Probabilities are multiplicative, information is additive 概率是可乘的，信息是可加的

    • if ones tosses a coin n times, there are 2 n possible outcomes, but they can all be written using n bits only

      如果一个人掷硬币 n 次，则有 2 n 种可能的结果，但它们都只能使用 n 位写入

    • information content depends on the logarithm of the probability distribution 信息内容取决于概率分布的对数

  • Impossible events should not matter 不可能的事件应该无关紧要

    • adding “the coin can rest on its side with probability 0” should not change the amount of information

      添加“硬币可以以 0 的概率侧放”不应改变信息量

    • weight the logarithms by the probability of each event

      通过每个事件的概率对对数进行加权

  • Entropy: \[ H(x) = - \sum_{X = x}p_x\log_2p_x \]

Example: coin toss 例子：抛硬币

  • Two values (e.g. Heads and Tails)
    • respective frequencies p and 1-p

What does it mean?

  • How can one coin-toss hold less than 1 bit of information when there are still 2 possible outcomes?

    当仍然有 2 种可能的结果时，一次抛硬币怎么能包含少于 1 位的信息？

    • The result is probabilistic: for any bit rate b>H(X) and any arbitrarily low error rate e>0, there is an encoder with bit rate b and error rate e.

      结果是概率性的：对于任何比特率 b>H(X) 和任何任意低的错误率 e>0，都有一个比特率 b 和错误率 e 的编码器。

  • What does an non-integer quantity of information mean? 非整数信息量是什么意思？

    • It is a rate for a stream of independent samples 它是独立样本流的速率
    • The stream is encoded by grouping samples together into words 通过将样本分组为单词来编码流
    • H(X) = 0.4 means that words of 10 samples can be encoded using 4 bits only. H(X) = 0.4 意味着 10 个样本的字只能使用 4 位进行编码。

  • How does it work?

    • If H(X) is less than maximal, then there are rare events

      如果 H(X) 小于最大值，则存在罕见事件

    • Conjunctions of rare events become rarer and rarer as word length increases

      随着字长的增加，罕见事件的连词变得越来越罕见

    • At some point they become negligible and do not need to be encoded anymore

      在某些时候，它们变得可以忽略不计，不再需要编码

Mutual information 相互信息

Objective: measure how much of the information in two random variables is shared

目标：衡量两个随机变量中有多少信息是共享的

  • if the variables are independent 如果变量是独立的

  • maximal if one variable is a function of the other

    如果一个变量是另一个变量的函数，则为最大值

  • Mutual information: I(X;Y) = H(X) + H(Y) - H(X,Y)

Computing the entropy numerically 数值计算熵

  • Don’t forget that log（0+） tends to -∞...

    • ... but 0 +log0+） converges to 0 收敛到 0
    • test if the probability is 0, do not compute explicitly in that case 测试概率是否为 0，在这种情况下不要显式计算

  • How to compute log2:

    • many languages do have a log2 function （MATLAB, C, C++,...）

    • ... but some do not （Java,...）

    • for any bases a and b, \[ \log_bx = \frac{\log_ax}{\log_ab} \]

  • in our case b=2

  • a is usually e, maybe 10

Back to attribute selection 返回属性选择

  • Finding attributes with high entropy寻找具有高熵的属性:

    • start with a compressing transform （e.g. time/frequency for sounds, wavelet or DCT for images, movement vectors for video,...）

      从压缩变换开始（例如声音的时间/频率，图像的小波或 DCT，视频的运动矢量，......）

    • for numeric attributes, variance can often be used as a good surrogate for entropy, and is easier to compute

      对于数值属性，方差通常可以用作熵的一个很好的替代品，并且更容易计算

  • Finding features with high mutual information with the classes 查找具有高互信息的类的特征:

    • expert knowledge may be available （e.g., pitch is key to music transcription）

      可能有专业知识（例如，音高是音乐转录的关键）

    • Common sense works 常识有效!

      • e.g., don’t classify text documents based on frequency of “a”, “the”, “and”, ...

        例如，不要根据“a”、“the”、“and”、...的频率对文本文档进行分类

Some common classifiers 一些常见的分类器

Bayesian classifiers 贝叶斯分类器

  • Maximum Likelihood learning: for each class C, learn a probability distribution p(.|C) that maximizes ∏p(m|C) for all the m in the training data.

    最大似然学习：对于每个类 C，学习一个概率分布 p(.|C)，它使训练数据中所有m 的∏p(m|C) 最大化。

  • classification: for a point m, find the class C that maximises p(C|m)

    分类：对于点m，找到最大化 p(C|m) 的类 C

    • using Bayes’ rule: \[ p(C|m) = \frac{p(C)p(m|C)}{p(m)} \]

  • scales linearly with the number of classes 与类数成线性比例

Support Vector Machines支持向量机 （SVM）

  • linear classifiers: boundaries are hyperplanes

    线性分类器：边界是超平面

  • Maximal Margin learning: learn the hyperplane that is the furthest away from making a mistake

    最大边际学习：学习离犯错最远的超平面

    • maximize the distance to the closest （=worst） points of each class （the support vectors）

      最大化到每个类（支持向量）的最近（=最差）点的距离

  • mainly for 2 classes

Decision trees 决策树

  • A tool to represent decision making

    代表决策的工具

  • Decompose a complex problem into a sequence of simple questions

    将复杂问题分解为一系列简单问题

  • Which question to ask next depends on the previous answers

    接下来问哪个问题取决于之前的答案

    • can be represented as a tree 可以表示为一棵树

  • Tree structure 树结构:

    • leaves叶: decisions 决策
    • nodes节点: questions 问题
    • branches分支: possible answers to the parent question 父问题的可能答案

Decision trees and classification 决策树和分类

  • Classification can be seen as a decision problem

    分类可以看作是一个决策问题

    • the decision to take: which label apply to a data point?

      采取的决定：哪个标签适用于数据点？

  • Classifiers can be represented as a decision tree

    分类器可以表示为决策树

    • a decision tree also partitions the attribute space

      决策树也划分了属性空间

    • question: on which side of a given border is the point we’re interested in?

      问题：在给定边界的哪一侧是我们感兴趣的点？

    • several trees can represent the same classifier

      几棵树可以代表同一个分类器

Learning decision trees 学习决策树

  • Learning problem 学习问题

  • which questions to ask?要问哪些问题？

  • If one asks enough questions, one will eventually classify the training set 如果问足够多的问题，最终会对训练集进行分类

    • 1 question = 1 new split of the space

      1 个问题 = 1 个新的空间分割

    • with enough splits, one can eventually separate each training point from all others

      通过足够的分割，最终可以将每个训练点与所有其他训练点分开

    but would that classifier generalise well to data it hasn’t seen before?

    但是这个分类器能很好地泛化到它以前没有见过的数据吗？

  • Goals of the learning 学习目标:

    • the classifier should be general 分类器应该是通用的

    • it should be fast to apply 申请应该很快

      • build a classifier that needs to ask few questions before taking a decision （a shallow, bushy tree）

        建立一个分类器，在做出决定之前需要问几个问题（一棵浅而浓密的树）

ID3 algorithm （Quinlan 1986）

ID3 takes a training set and builds a decision tree such that each leaf is homogeneous D3 接受一个训练集并构建一个决策树，使得每个叶子都是同质的

  • builds the tree from the root to the leaves 从根到叶构建树

  • greedy algorithm: no backtracking

    贪心算法：无回溯

ID3(Training_Set) =
	if all the points in Training_Set have the same class C
	then return Leaf(C);
	else
		Question = Find_Best_Question(Training_Set);
		[Set_1,...,Set_n] = Split(Training_set, Question);
		return Tree(Question, ID3(Set_1),...,ID3(Set_n));

How does `Find_Best_Question work?

  • which questions are allowed 哪些问题是允许的?
  • how to select the best one 如何选择最好的?

Finding the next question to ask 寻找下一个要问的问题

  • ID3 restricts itself to questions that test only one attribute

    ID3 将自身限制为仅测试一个属性的问题

    • fewer questions to choose from 可供选择的问题更少

    • geometrically, all the borders are orthogonal to one axis

      在几何上，所有边界都与一个轴正交

    • for symbolic attributes mainly 主要用于符号属性

      • symbolic attributes cannot be combined 符号属性不能组合
      • symbolic attributes can be enumerated 可以枚举符号属性

  • Each question is chosen so that the resulting subsets are as classified as possible 选择每个问题，以便尽可能对结果子集进行分类

    • the entropy of the class variable within a set measures how classified it is

      集合中类变量的熵衡量它的分类程度

\[ H(s) = - \sum_{C = c}p_c\log_2p_c \]

Finding the best question 找到最好的问题

  • The entropy of a partition is the sum of the entropies of the subsets weighted by their size 分区的熵是按其大小加权的子集的熵之和

  • Entropy also provides a stopping criterion: a leaf is a node with entropy 0

    熵还提供了一个停止标准：叶子是熵为 0 的节点

  • The information gain is defined as the difference between the total entropies before and after partitioning

    信息增益定义为分区前后总熵之差

    • It is also the mutual information between the attribute and the class, conditioned to the previous splits

      信息增益定义为分区前后总熵之差 \[ G(S,Q) = H(S)-\sum_{S_i∈Split(S,Q)}\frac{|S_i|}{|S|}H(S_i) \]

  • Compute G for all possible questions and select the largest one 为所有可能的问题计算 G 并选择最大的一个

Which questions to ask? 要问哪些问题？

  • 符号属性Symbolic attributes（性别，头发颜色，...）:

    • only a finite number of possible values for all data

      所有数据只有有限数量的可能值

    • one branch for each different value

      每个不同值的一个分支

  • 数值属性Numerical attributes（温度、像素值等）:

    • infinite range even if only a finite number of values are present in the training set

      即使训练集中仅存在有限数量的值，也具有无限范围

    • choose a threshold t and ask the question: value < t?

      选择一个阈值 t 并提出问题：value < t?

  • Threshold choice 阈值选择:

    • the information gain criterion still holds

      信息增益准则仍然成立

    • the best threshold must be between two training examples of different classes 最佳阈值必须在不同类别的两个训练样本之间

    • try all possible thresholds 尝试所有可能的阈值

Strong and weak points of ID3 ID3的优缺点

  • Each question only checks one feature 每个问题只检查一个特征:

    • works well with attributes of different types （ex: size and gender）

      适用于不同类型的属性（例如：大小和性别）

    • designs rectangular regions only, even if it does not fit the data

      仅设计矩形区域，即使它不适合数据

  • Generates the shortest possible tree 生成最短的树:
    • fast classification after learning 学习后快速分类
    • helps to generalise the model 有助于泛化模型
    • helps to understand the generated rules 有助于理解生成的规则
  • Training is expensive:

    • try all possible questions at each node

      在每个节点尝试所有可能的问题

    • for each question, compute the answer with each data point of the subset

      对于每个问题，使用子集的每个数据点计算答案

    • infinite range attributes generate lots of different questions

      对于每个问题，使用子集的每个数据点计算答案

A word on overfitting 关于过拟合的一句话

  • Noise blurs the boundaries between classes

    噪音模糊了类之间的界限

  • Sampling can alter the perception of the problem

    抽样可以改变对问题的看法

    • selection bias 选择偏差
    • rare events 罕见事件

  • General solution: more data! (not always possible)

    通用解决方案：更多数据！（并非总是可能）

  • Overfitting过拟合

    • learning a tree that models properties specific to the training set 学习一棵树来模拟训练集特有的属性

      • the tree works on the training set but not on the problem data

        该树适用于训练集，但不适用于问题数据

      • perfect classification of the training set is most often too much

        训练集的完美分类通常太多了

Pruning 修剪

Reduced-error pruning 减少错误修剪

  • Split the training set in two 将训练集一分为二:

    • one for learning 一个学习
    • one for pruning 一种用于修剪

  • Goal: learn a tree that classifies the pruning set well although it was not learnt on it

    目标：学习一棵树，它可以很好地对剪枝集进行分类，尽管没有在上面学习

  • First build a tree with ID3 on the training set 首先在训练集上建一个ID3的树

  • Then greedily prune nodes to improve the classification of the pruning set 然后贪婪地修剪节点以提高修剪集的分类

  • What if we don’t have enough data to make a pruning set? 如果我们没有足够的数据来制作剪枝集怎么办？

Rule pruning 规则修剪

The tree can be written as a set of logical rules 树可以写成一组逻辑规则

  • each path is a single conjunction of tests 每条路径都是测试的单一结合
  • there is a disjunction between different paths 不同路径之间存在分离

Rule post-pruning 规则后修剪

  • First convert the tree into rules

    首先将树转换为规则

  • Then greedily prune the rules by removing preconditions

    然后通过删除先决条件来贪婪地修剪规则

  • Only prune if the precision of the rule does not decrease

    仅在规则的精度不降低时进行修剪

    • Precision = fraction of points of the correct class among the points picked by the rule

      精度 = 规则选取的点中正确类别的点的分数

  • The pruning is done on the training set directly

    剪枝直接在训练集上完成

    • no need to split the set in two 无需将集合一分为二
    • more used in practice 在实践中更多地使用

Summary

  • Information as a measurable quantity

    作为可测量数量的信息

    • information as structure in data

      作为数据结构的信息

  • ID3: a method of using information to decide on structure

    ID3：一种使用信息来决定结构的方法

  • Overfitting 过拟合

    • pruning to re-generalise

      修剪以重新概括
