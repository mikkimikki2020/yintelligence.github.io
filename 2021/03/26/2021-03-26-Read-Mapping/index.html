Related resources: Mapping, CAM: Mapping strategies for sequence reads, Colostate: Mapping Reads to a Reference

Introduction

Sequencing produces a collection of sequences without genomic context. We do not know to which part of the genome the sequences correspond to.

Mapping the reads of an experiment to a reference genome is a key step in modern genomic data analysis.

What is read mapping? 1:49 - 4:19

Map reads on a reference genome. Read mapping is the process to align the reads on a reference genomes.

A mapper takes as input a reference genome and a set of reads.

  • Input: Many reads, length l strings R1,...,Rm and approximate reference genome string G
  • Output: positions in G where reads match

With the mapping the reads are assigned to a specific location in the genome and insights like the expression level of genes can be gained.

Read mapping: input 1:02:26

  • Set of reads

  • Reference genome(s)

    A reference genome (also known as a reference assembly) is a digital nucleic acid sequence database, assembled by scientists as a representative example of the set of genes in one idealized individual organism of a species.

Read mapping: output (mapped reads) -1:03:05

  • List of reads with positions

Unique / multi reads 58:27

One read can map to one unique position: uni-read or to multiple positions: multi-read.

Complications

  • Millions of reads to analyse
  • Repetitive regions: non-unique mapping
  • Mutations/errors: imperfect match, incorrect mapping
  • Paired end reads
  • Numbers: ~100M reads, |G| ~ 3×109, read length: 100-200bp

Strategies for mapping multi-reads 1:00:03 - 1:01:55

The multi read resolution strategies are:

  • Discarded
  • All mapping locations are used (not recommended)
  • The mapping location is randomly chosen among the possibilities

Mapping strategy 1: seed and extend 1:03:05 - 1:12:39

Seed-and-extend strategy is based on the observation that a good alignment should contain exact or inexact short matches between two sequences.

Principle

  1. Find an exact match using a seed of size k.
  2. Extend the seed allowing mismatches.

A general view of seed-and-extend strategy that contains four steps:

  1. seed generation
    • Seeds are the shorter sequences extracted from a read and can be generated using different methods.
  2. seed mapping
    • The seeds that exactly match a reference sequence can be identified through a mapping process facilitated by an index method.
  3. extending each matched seed
  4. alignment of the read to the reference sequence.

Mapping strategy 2: suffix tree and suffix array

Suffix tree

  • Build suffix tree for G

  • For each read Ri, find matches of Ri to G by tree traversal

  • Time complexity: O(lm+|G|) for exact matching

  • Can store human genome text in 750M bytes (6G bits)

    but need ~64G bytes for the tree

  • Large constants, hard to implement

Suffix tree: construction

Suffix tree: compaction (d = 2)

  • Limit to prefix of length 2.
  • Leafs display the prefix starting position.

Compacted suffix tree

Solution: AT → (3, 1).

Search complexity in O(r) where r is the read length.

Suffix array

Based on the same principle as a suffix tree, a suffix array is an efficient and compact index of the genome.

Summary 1:13:04 - 1:16:00

  • Naive search is not practicable (high number of reads, large genome). -
  • Seed and extend is a simple and efficient alignment method. The seed size is a key parameter.
  • Suffix trees (and array) are efficient ways to index the genome.
