Secondary structure for unknown sequence

Zpred

  • Based on GOR
    • Uses homologous sequence information
      • Averaging of prediction per residue
      • Calculates conservation value
  • Some amino acids are more similar to each other than others…
    • Related to physical characteristics (of the side chain), not DNA sequence mutation variation
  • Amino acid characteristics
  • Conservation value
    • Zvelebil conservation number (Cn)
      • 1.0 for identical residue
      • 0.9 for residues with same characteristics (e.g. I and L)
      • Subtract 0.1 for every different characteristic
        • R positive, polar
        • V aliphatic, hydrophobic, small
        • 5 differences; conservation 0.9 – (0.1 x 5) = 0.4
      • Gaps get all characteristics – Results in low conservation value!
  • The results…
    • Q3 ranges from 44.9% (allβ) to 76.9% (allα)
    • Results for all β are worse than GOR IV…

Nearest-neighbour method

  • Proteins similar sequence share same fold
    • Assumption: also true for short stretches
  • Examine query sequence with same window
    • Identify best-matching fragments
    • Look at most prevalent secondary structure for middle residue

PREDATOR

  • Identifies potentially hydrogen-bonded residues
    • α helices, parallel and antiparallel βsheet
  • Nearest-neighbor for secondary structure
    • α helices, βsheet and coil
  • Propensities
    • Turns
  • Combines results
  • Can use homologous sequences
    • Pairwise alignments!
  • The results without homologous…
    • Q3 ranges from 49.0% (α+β) to 66.4% (α/β)
    • Similar to GOR IV
  • The results with homologous…
    • Q3 ranges from 54.2% (α+β) to 80.5% (α/β)
    • Again homologous sequences help…
  • Results

Hidden Markov model

  • Discrete Markov process
  • Transition probabilities expressed in transition matrix

Discrete Markov process

  • Probability for Sun-cloud-rain-rain-sun?

  • First have to define starting state (sun), then: \[ P = P(sun |sun)× P(cloud |sun)×P(rain | cloud)× P(rain |rain)× P(sun |rain) P = 0.50 × 0.30 ×0.35× 0.40 × 0.10 = 0.0021 \]

  • Can get probability for sequence of related events

    • Dependence only on previous state(s)!
    • Markov chain

Hidden Markov Model (HMM)

  • Define:

    • Hidden states (N=3) \[ S = \{S_1, S_2,..., S_N\} \]

    • State of model at step n \[ q_n \]

    • Observations (M=20) \[ V = \{v_1, v_2,..., v_m \} \]

    • State transition matrix \[ A = \{p_{i|j}\}\\ p_{i|j} = P(q_{n+1} = i | q_{n} = j),1≤ i, j ≤ N \]

    • Emission distribution \[ E = \{e_j(k)\}\\e_j(k) = P(V_kat n | q_n = S_{j});1≤ j ≤ N,1≤ k ≤ M \]

    • Initial state distribution \[ π= π_{i}\\ π_{i} = P(q_1 = S_j),1≤ j ≤ N \]

  • 3-state secondary structure HMM

    • Produce observation sequence
      • Choose an initial state Sj according to π; step q1
      • Choose an emission observation vk based on state
      • Proceed to step qn+1, pick state Si using transition probabilities from matrix A
      • Again emission, proceed or stop, …

  • Have to ‘recover’ the hidden state probabilities from a known observed sequence

    • Calculate all state sequences of sequence length T?
      • 2TNT calculations; not efficient
    • Forward algorithm
      • Start with first amino acid in sequence
        • Calculate probabilities for observing next amino acid in sequence for each secondary structure combination
      • N2T calculations

  • Hidden states model can be made more complex to take more variables into account

    • Hydrophobicity, charge bias, helix length, …
    • Incorporate homologous sequence

  • For example HHPred

Machine learning

  • Neural networks
  • Random forests
  • Support vector machine
  • …

Neural networks

  • Layers with connected nodes or units

  • Many architectures now possible for neural networks
    • Is a world in itself
  • Always crucial is the network training
    • Training set
      • With as little bias as possible
      • Is used to make the neural network ‘learn’
  • Complexity of network to fit problem
    • Want predictions as ‘general’ as possible!
Network training
  • Test set
    • To validate results and avoid overfitting

PHD

  • Uses two neural networks
    • First sequence-to-structure
      • Predict secondary structure
    • Second structure-to-structure
      • Takes output of first network
      • Correlate conformations for neighbouring residues
  • The results…
    • Q3 ranges from 73.5% (allβ) to 87.9% (allα)
    • Much better…

Example:

PROF

  • Predict secondary structure with different approaches
    • GOR-type, multiple sequence alignment, …
  • Use neural network to choose final prediction
  • The results…
    • Q3 ranges from 58.15 (allβ) to 90.7% (allα)

PSIPRED

  • Similar to PHD
    • But with first PSI-BLAST multiple alignment step
  • The results…
    • Q3 ranges from 65.4% (allβ) to 94.4% (allα)

Hurray for neural networks

  • Give the best results overall
  • Black box
  • Does not contribute to our understanding of how sequence encodes secondary structure…
  • Lots of effort going towards ‘understandable AI’
    • But not so easy

  • Combine results from different approaches
  • Increases confidence in positives

Which data in the end?

Sequence to secondary structure

  • Validation
    • Community testing efforts (CASP, CAFASP)
    • Many approaches already tried before…
    • Always worth checking out!
  • What does the input data mean?
    • Understand the results
    • Know what can be used as input in the first place
  • What does the output of each method mean?
    • What are you using it for?
    • Accuracy, confidence, …
  • Why again?
    • Enormous amount of sequence-only data – Identify structured regions:
      • Can give insights in possible function
      • Can help in deciding where to mutate sequence
  • But!
    • Sequences can adopt multiple secondary structure in different circumstances...
      • Absolute truths are rare in biology!
    • Alpha-helix easier than beta-sheet…
  • Can probably only get up to ~90% accuracy
