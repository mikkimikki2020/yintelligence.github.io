CSS: Cascade Style Sheets

Slide + Question + Files + Solution

Video + Background

What is CSS?

  • Styling of HTML DOM objects
    • Specified by their class, id, html tag or relation to another DOM object
  • Styling is processed by a layout-engine (e.g. WebKit)
    • Layout-engine decides how to render your HTML objects
    • W3 standards to assure that all browsers are the same
    • Not all features are support by all layout engines unfortunately

How is CSS added?

  • External CSS

    <link rel="stylesheet" type="text/css" href="mystyle.css">

  • Internal CSS

    <style>
...
</style>

  • Inline CSS

    <h1 style="color: red;">Hello World</h1>

    Applying Styles

Styling by tag

body {
    background-color: #ffffff;
}

h1 {
    color: red;
}

Styling by ID

<h1 id="myElement">This is a test</h1>
#myElement {
    color: brown;
}

Styling by class

<h1 class="redTitle">This is a test</h1>
<h2 class="redTitle">This is a second test<h2>
.redTitle {
    color: red;
}

SCSS/SASS: Syntactically Awesome Style Sheets

  • Add variables, includes, conditions, ...
  • E.g. variable ”primary-color” for consistent colouring

Helpful Links

  • CSS Tutorial
  • Build fast, responsive sites with Bootstrap

Exercise

CSS (Cascading Style Sheets) is the most common way to define the styles of an HTML document. In this exercise you will have to create CSS rules to improve the layout of the provided HTML document.

CSS rules are defined by specifying which element you want to style (using element type, id,class,...) and then we define some rules that set parameters such as the position, font, size or colour.

#mydiv {
    color: maroon;
    margin-left: 40px;
}

Task 2

There are CSS frameworks that will help you create layouts or provide nice uniform styling for common components. Some may even offer automatic adaptation to different devices and screen sizes. One of the most popular CSS frameworks is Bootstrap.

Take a look at the framework and the examples. Next, apply some of the features to your webpage from task 1. Use the manual to get you started.

You should at least use the grid system to define your layout. You can optionally use some of the other components to make your site prettier. You can modify the HTML if needed. Make sure your old CSS rules from task 1 do not conflict with what you are trying to achieve with Bootstrap.
