Exercise

Ever since its release in 1993, the HTML standard has continued to evolve together with the web. Nowadays, it is capable of much more than simply showing text and images. In this exercise session we focus on some of the latest additions that were introduced by the HTML5 standard that was finalized in 2014.

Valid HTML5 Documents

An HTML document is just a file that ends with a *.html extension and contains valid HTML code. Usually the first line in an HTML document will be a document type description that tells the browser which version of HTML you will be using. In the past, you had to use complex DocTypes but this changed in HTML5 where the following line is enough to indicate that you are using HTML5:

<!DOCTYPE html>

As seen in the lecture, your browser is very forgiving when the HTML is invalid meaning it will still try to render as much as possible. If you want to know if your HTML code is valid or not, you can use an online validation tool such as: Markup Validation Service.

Task 1: Form Validation

During previous exercises we have already done server-side validation with servlets and client-side validation with JavaScript. Developers quickly realized that in a lot of use-cases the things that are being checked are always the same (e.g. the field can not be empty, minimum password length,email validity, date formatting, ...).

To prevent developers from writing the same code over and over again, a solution was introduced in HTML5. Developers can now provide some restrictions directly in the HTML code (not JavaScript!) and the browser will automatically make sure that the rules are followed. Some example here.

Create an HTML form with client-side validation for the following fields. DO NOT use JavaScript. No example is provided; try to use your current HTML knowledge in combination with Google to build the form from scratch. None of the fields are allowed to be empty, you should use placeholders to describe the field and on top of that the following rules apply:

  • Username: minimum 6 characters
  • Password: minimum 6 characters
  • Email: needs to be a valid email address
  • Birthday: needs to be a valid date
  • Postal code: In Belgium postal codes are between 1000 and 9999

Task 2: Canvas Paint App

In this task we will experiment with the ”canvas” element introduced in HTML5. The canvas is used for creating dynamic graphics and games without the need of 3rd party plug-ins such as Flash. With JavaScript we can draw lines, text, shapes and pictures in a canvas element.

For more information see: HTML Canvas Graphics.

As a starting point, extract all the files in the provided ZIP file and run paint.html. A functional”pain” application is provided, allowing you to clear the canvas and switch the colour of the paintbrush. The last two buttons (invert and choose file) do not work yet.

The goal of this exercise is to add this functionality yourself. When you take a look at the HTML code, you will see that the HTML itself is very simple with most of the work being done in paint.js. Reading Files with JavaScript

he next step will be to read the data from the selected image. In order to read a file from the user’s hard disk we will use the HTML5 FileReader API.

Use this as an example: Reading Files Using The HTML5 FileReader API.

However, in our case we will use the reader's readAsDataURL() method instead of readAsText(). Add code to the handleLoad function to read t he selected file and put the content in a variable. Like last time event.target.files[0] contains the path of the selected file.

Loading Images into a Canvas

Now that we have the image data in a variable, we just have to draw it on the canvas. We can not draw the raw data directly on the canvas, for this we must first create an Image object and then make the canvas draw the image object.

The following example should give you an idea of what to do (line 18 to 24): HTML5 Canvas Load Image Data URL.

The example also shows you how to draw the created Image object to the canvas. Use the previously read file data as a source for the image. This means that you will have to put the Image creation code inside of the existing filereader’s onload function. var isDragging = false;
var color = "#0000FF"; // set the initial drawing color to blue

var canvas = null;
var context = null;

// this function will be called as soon as the document is done loading and the DOM is complete
$( document ).ready(function() {
    canvas = document.getElementById('myCanvas');
    context = canvas.getContext('2d');

    // tell the canvas which function to call when the mouse is pressed/moved/released
	$('#myCanvas').mousedown(handleMouseDown);
	$('#myCanvas').mousemove(handleMouseMove);
	$('#myCanvas').mouseup(handleMouseUp);

	// clear canvas when 'clear' button is pressed
	$('#btnClear').click(clearCanvas);
	
	// set color to red when 'red' button is pressed
	$('#btnRed').click(function(){
		setColor("#FF0000");
	});

	// set color to green when 'green' button is pressed
	$('#btnGreen').click(function(){
		setColor("#00FF00");
	});

	// set color to blue when 'blue' button is pressed
	$('#btnBlue').click(function(){
		setColor("#0000FF");
	});

});

function clearCanvas()
{
	context.clearRect(0, 0, canvas.width, canvas.height);
}

function setColor(newcolor)
{
	color = newcolor;
}

function handleMouseDown(event)
{
	var mouseX = event.pageX - this.offsetLeft;
	var mouseY = event.pageY - this.offsetTop;
	isDragging = true;
	context.beginPath();
    context.moveTo(mouseX, mouseY);
}

function handleMouseMove(event)
{
	if(isDragging){
		var mouseX = event.pageX - this.offsetLeft;
		var mouseY = event.pageY - this.offsetTop;
		context.lineTo(mouseX, mouseY);
		context.strokeStyle = color;
		context.stroke();
	}
}

function handleMouseUp(event)
{
	isDragging = false;	
}
