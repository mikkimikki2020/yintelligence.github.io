AJAX and LeafletJS

Background + Video + Question + Solution

AJAX (Asynchronous Javascript and XML) is a set of technologies used on the client side to retrieve additional data after a document is loaded, without having to reload the entire page. In the first part of this exercise session we will use AJAX to retrieve the weather for the next few days when a user clicks a button. In a second part we will learn to use LeafletJS (an open source map) and cover topics such as geolocation, markers and pop-ups.

Exercise 1: Family Example

For the first part you will need the fileajax.html, which can be found on Canvas. Open it and press the ”Get Data” button.

In this file you will find an example that uses jQuery to make an asynchronous request. In this example, JSON is returned that represents a family and its members. This JSON is converted to a JavaScript object by jQuery, making it easy to display.

The example is complete and functional but we want to make sure you understand what is happening.

  • When you press the button, the family data is retrieved from an external location, where?
  • What happens when you manually visit this external location with your browser?
  • What does the data look like and how do you use it in JavaScript?

Note: some browser do not allow asynchronous requests from local HTML files to other domains. To by pass this, we make use of JSONP. This is achieved by appending ?callback=? to the target URL,as seen in the example. For more information, see: here

Exercise 2: Weather API

The goal of this exercise is to do something similar as the family example, but with another API. We will retrieve the weather forecast for the next few days. Cloud-based API’s are very popular these days, but unfortunately for us they are also more commercialized than they were several years ago. Public API’s like Google Maps API or Weather API’s are no longer free if you are including the min public software. These companies often provide an API key that allows you to make a limited amount of requests per hour or day. One of these API’s is the website the ”Get API key and start” for the free tier.

REST API Documentation

Documentation of REST API’s usually always looks very similar

  • Description: What does the request to the API do or return?
  • HTTP Request: What URL, what parameters, what method (POST, GET, DELETE, ...),other headers (e.g. Authorization)
  • HTTP Response: What is the response code, what is the response content-type, how does the response look?
  • Examples: Often, an example URL and response is given

Exercise 3: LeaftletJS

As mentioned in exercise 2, Google Maps is no longer free for developers. Fortunately, LeafletJS is an open-source maps framework that supports everything that was possible with Google Maps and much more. It is used by companies such as Facebook, GitHub, Flickr, IGN, The Washington Post,...

Other frameworks are available such as MapBoxexist) (free till 50k users). The advantage of using LeafletJS is that you get to understand the fundamental concepts of map frameworks such as Google Maps. While the Google Maps API is very high level and does not show internal API calls, we will use these internal API calls when developing with Leaflet.

Framework Concepts

  • Map Layers: This is the main concept of Leaflet. Visually you have multiple layers that overlay information. Even the background (the actual map) is a layer that can be configured. You can also choose what layer is visible before what (e.g. do you want markers to be in front or behind your routes or drawings?)
    • Tile Layer: Every map API offers a tile layer. Google Maps has several tile layers(satellite, traffic and normal maps). The reason why they are called tiles is because in the case of satellite, the whole map comprises of square images that need to load independently. When you are zoomed out, you do not want load images that are detailed enough to see the colour of the cars - this would of course be too much data. Multiple images are generated for each zoom level. In the case of non-satellite images, many modern maps use vector graphics that are much smaller and do not become blurry when zooming in on them.
    • Overlay Layers: Overlay layers align data on top of the tile layer. The idea behind this is that you can visualise information such as the contours of a building, heat map or more.
    • Menu Layer(s): Every map contains some buttons that the user can click. This can be the zoom level buttons or a search window.
  • Remote Services: Services such as geocoding or routing require a bit of computational power. MapBox provides a public free service that we can use to perform these actions, but know that you can also self-host open-source databases. For the geocoding we use Nominatim and for the routing machine we use OSRM.
  • Plugins: LeafletJS is an open-source framework that allows you extend functionalities (layers) by adding plugins. A list of known plugins can be found here.
