Java Servlets and JavaServer Pages (JSP)

Slide + Question + Files + Solution

Video + Background

JSP: JavaServer Pages

  • JSP are HTML pages with Java programming
  • JSP adds ”Expressions”
  • JSP adds ”Scriplets”

Example: Plain HTML page

<!DOCTYPE html>
<html>
    <head>
        <title>Insert title here</title>
    </head>
    <body>
    </body>
</html>

Example: JSP Directive

<%@ page language="java" contentType="text/html;charset=ISO-8859-1" pageEncoding="ISO-8859-1"%>
    
<!DOCTYPE html>
<html>
    <head>
        <meta charset="ISO-8859-1">
        <title>Insert title here</title>
    </head>
    <body>
    </body>
</html>

The directive tells you what the result is that we are returning (this case HTML with Java)

Example: JSP Expression

!DOCTYPE html>
<html>
    <head>
        <title>Insert title here</title>
    </head>
    <body>
        <%= new java.util.Date() %>
    </body>
</html>

Expressions ”evaluate” to the result of the Java code (in this case the date)

Example: JSP Scriplets

<!DOCTYPE html><html><head><title>Insert title here</title></head><body><%int count = 0;for (int i = 0 ; i < 10 ; i++ ){count++;}%>The counter is <%= count %></body></html>
  • Scriplets allow you to run Java code. Other than expressions it does not ”evaluate” to a result shown on the page.
  • Scriplets ”evaluate” the HTML code contained. (e.g. if statements, for loops, ...)

Example: JSP Request variable

<!DOCTYPE html>
<html>
    <head>
        <title>Insert title here</title>
    </head>
    <body>The GET parameter for "name" is<%=request.getParameter("name") %>
	</body>
</html>

JSP pages have access to the ”request” object that contains information about the HTTP request.

Example: JSP Response variable

<!DOCTYPE html>
<html>
    <head>
        <title>Insert title here</title>
    </head>
    <body>
        <% response.setHeader("X-Password", "test"); %>
	</body>
</html>

JSP pages have access to the ”response” object that allows you to manipulate the HTTP response.

Example: JSP Out variable

<!DOCTYPE html>
<html>
    <head>
        <title>Insert title here</title>
    </head>
    <body>
        <% out.write("Hello World!"); %>
	</body>
</html>

JSP pages have access to the ”out” object that allows you to writeto the HTML output.

Example: JSP Session variable

<!DOCTYPE html>
<html>
    <head>
        <title>Insert title here</title>
    </head>
    <body>Logged in as <%= session.getAttribute("username") %>
	</body>
</html>

JSP pages have access to the ”session” object that contains information about the session. A session links a session ID (e.g.WISESESSION) to a dictionary of information that is hidden to the user

JSTL: Standard Tag Library

  • Adds HTML tags that perform actions
    • for loops, if-else
    • string formatting and escaping
    • SQL queryingImuch more ...
  • More convenient than using scriplets inside your HTML code

Example: JSTL Core tags

<%@ taglib prefix="c" uri="http://java.sun.com/jsp/jstl/core" %>
    
<!DOCTYPE html>
<html>
    <head>
        <title>Insert title here</title>
    </head>
    <body>
        <c:forEach var="i" begin="0" end="10">
            Item <c:out value = "${i}"/> <br/>
        </c:forEach>
    </body>
</html>

Example: JSTL Formatting tags

%@ taglib prefix="c" uri="http://java.sun.com/jsp/jstl/core" %><%@ taglib prefix="fmt" uri="http://java.sun.com/jsp/jstl/fmt" %>
    
<!DOCTYPE html>
<html>
    <head>
        <title>Insert title here</title>
    </head>
    <body>
        <fmt:setLocale value="en"/>
        <fmt:setBundle basename="edu.vub.ex4.LanguageBundle"var="lang"/>
        <fmt:message key="count.one" bundle="${lang}"/><br/>
        <fmt:message key="count.two" bundle="${lang}"/><br/>
        <fmt:message key="count.three" bundle="${lang}"/ <br/>
	</body>
</html>

Java Servlets

What are Servlets?

  • Servlets run in a J2EE web container
  • Servlets are mapped to an url (e.g. example.com/hello)
  • Servlets can perform different actions based on the type of HTTP request (GET, POST, ...)
  • Servlets return an HTTP response (or they redirect to a JSP view)

What are web containers?

  • Web containers run multiple servlets
  • Web containers map ”base” URLs to different applications(e.g. example.com/previousservlet/hello)
  • Web containers handle security, concurrency and deployment
  • Web containers can be created by different software: WildFly,Apache Tomcat, Jetty, ...

Example: Servlet

@WebServlet("/addUser.do")
public class MainServlet extends HttpServlet {
    private static final long serialVersionUID = 1L;
    
    public MainServlet() {
        super();
    }
    
    protected void doGet(HttpServletRequest request, HttpServletResponse response) throws ServletException, IOException {
        HttpSession session = request.getSession(true);
        if (session.getAttribute("username") != null &&session.getAttribute("username").equals("admin")) {
            request.getRequestDispatcher("/WEB-INF/secret.jsp").forward(request,response);
        }
        else {
            response.sendRedirect("login.jsp");
        }
    }
    ...
}

Exercise

Java Servlets

The goal of this exercise is to learn about server-side development for dynamic web pages.

In this exercise we will use simple Java Servlets to build a user registration form. We will use these servlets to validate the input on the server side (e.g. Is everything filled in? Is the password strong enough?).

We will use Eclipse IDE and Apache Tomcat for this exercise. However, as a student you have access to IntelliJ IDEA Ultimate through the GitHub Student Pack. Alternatives for Tomcat such as WildFly, Jetty, Oracle WebLogic, ... also exist.

Step 1: Installing & Configuring workspace

  1. Download Apache Tomcat 9.0 from tomcat.apache.org(Download the .zip or .tar.gz file).

  2. Extract the downloaded archive. This extracted directory contains the server that will be used to run your servlets. You do not need to start this server, we will configure Eclipse to automatically start Tomcat when you compile your project.

  3. Download and install JDK (Java Development Kit) 8 or higher from oracle.com

  4. Download the latest Eclipse IDE from eclipse.org

  5. Start the installation for Eclipse . You will be asked to specify the type of projects you need Eclipse for. While we will use it for JavaEE (Enterprise), it is advised to select the normal "Eclipse for Java Developers" option.

  6. Once installed, create a new project

  7. If you do not see a category called ”Web” (what normally is the case when you have installed the regular ”Eclipse For Java Developers”), you will have to enable web projects.

    1. Go to ”Help”>”Install New Software” in the top menu.
    2. In the top drop-down box, select the URL that matches the downloaded Eclipse version.
    3. Find and select ”Web, XML, Java EE and OSGi Enterprise Development”. Finally click”Next” to continue the installation.
    4. Wait for the installation to complete and restart Eclipse IDE if asked.
    5. Go back to the previous step and start a new project.

  8. Create a new ”Dynamic Web Project”.

  9. Provide a name for your project and click the ”New Runtime...” button in ’Target Runtime’ window.

  10. Select ”Apache Tomcat 9.0” as the server runtime environment, select ”Create a new local server” and click ’Next’.

  11. Specify the directory where you extracted Apache Tomcat in step 2.

  12. Finally, finish the creation of the dynamic web project. You will receive a message asking to switch to the Java EE perspective. It is not really important but you can click ”Yes”.

  13. It is advised to test the Apache Tomcat configuration.

    1. Create a new JSP file in the ”WebContent” folder of your newly created project (you can call it NewFile.jsp).

    2. Add the following Java code to the body of the HTML page. This is will display the time and date of the server every time the page is loaded.

      <%= new java.util.Date() %>

    3. After creating the file, start the JSP file by clicking the ”Play” button. If everything goes well, you will see a new browser window showing the current date.

Step 2: Complete the HTML form

Create a new JSP file in the ”WebContent” folder called ”addUser.jsp”. In the ZIP file that comes with this exercise you will find an incomplete version of addUser.jsp that you can use as a starting point. Copy its content to your addUser .jsp.

In order to have a nice layout, the example uses a table with 3 columns. As you can see in the HTML code, a <tr> tags opens a new row in the table (tr = table row). Every <td> opens a new cell within the row:

We will use the first column for displaying the title of the field. The second column contains a text input where the user can fill in his details. The last row is reserved for displaying an error message when the user does not provide valid input.

The goal of step 2 is to complete the HTML code until the form looks like the screenshot in the beginning of the document. A user should fill in:

  • A username
  • A password (hide the input like the screenshot)
  • The password again (to make sure he did not make a mistake)
  • First name
  • Last name
  • Email address
  • Gender (male or female)
  • Birthday

Step 3: Make the form submit the information to the server

Add a target and method to the form

First of all we need to complete the <form> tag to make it work. Take a look at the documentation. Make sure the form submits to the target ”addUser.do” using a POST request.

Add a servlet on the server side to handle the request

Next we will add a servlet component that will handle the logic on the server side. In Eclipse,right-click the project and select New ¿ Servlet. Call it MainServlet and click next.

In the next window, make sure to change the URL mappings. If you remember, we told the HTML form to post the data to ”addUser.do”. By setting the URL mapping to ”/addUser.do” we tell the server that MainServlet will handle any requests sent to that address.

On the final page when creating a servlet you can specify the methods that should be added by default. For now you do not need to change anything here. The default configuration will generate both a ”doGet” and ”doPost” method to handle GET and POST requests respectively.

After finishing the servlet creation you will see the Java class file in your project. We are going to modify our ”doPost” method to check if we can detect the POST requests to our servlet. Since our form is configured to use POST we will only use this method for handling the validation. You can try out your servlet by making it output some text to the console:

protected void doPost(...) throws ServletException, IOException {
    System.out.println("test");
}

Try this by selecting the ”addUser.jsp” file again, run it and press the ”submit” button. You should see something like this:

Step 4: Add server validation to the servlet

In this step we will check if all the input fields were entered correctly. In the ZIP file that comes with this exercise you will find an incomplete version of MainServlet.java that you can use as a starting point. The comments inside of the code will explain what is going on. First start by simply making sure all mandatory fields have been filled in (not empty). Continue with the next step and make sure it works. If everything works you should show errors for the fields that were left empty. If everything is successful the servlet will redirect the user to ”addSuccess.jsp”. Create this addSuccess.jsp and make it display ”The user has been successfully added” in the HTML code.

When that works you can extend your validation with the following rules:

  • username: minimum of 6 characters
  • password: minimum of 8 characters
  • password2: must be equal to the previous password field
  • email: must be a valid email address (use the included isEmailValid helper function)

Step 5: Update the JSP page to show errors if the servlet sends us back

In the ZIP file that comes with this exercise you will find an updated version of addUser.jsp(addUserstep5.jsp) that you can use as a starting point. Update your version that it also includes the Java code in the example above. The example shows you how to display the errors (if any) for the Username and First Name field. Apply this for all fields in your own version.
