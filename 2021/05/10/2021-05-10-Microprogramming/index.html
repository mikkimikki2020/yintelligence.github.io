Microprogramming

Video + Slide

The control unit

控制单元是控制所有活动的计算机的核心元件。 它从程序存储器program memory中读取instruction并将这些instruction转换为电信号，指示计算机的每个组件在正确的时间执行特定的任务。

Activities of Control unit includes:

  • Reading instructions from program memory;
  • Translating these instructions into electrical signals;

CU是一个顺序布尔电路sequential Boolean circuit，输入：

  • 从程序存储器中fetch并在执行期间存储在指令寄存器 instruction register中的指令
  • （如果存在）与 ALU 关联的条件代码位 condition code bit
  • 中断线 interrupt line
  • 来自内存系统的内存就绪线 memory ready line
  • 来自直接内存访问控制器Direct Memory Access controller的总线请求线 bus request line

基于这些输入并通过internal clock计时，CU生成用于控制计算机所有部分的digital sequence。

执行CU的一组布尔函数包括，对于每个command信号，instruction register中包含的位bit、各种输入信号（例如interrupt line）和计时计数器timing counter的内容的组合布尔函数。

Microprogramming

为了实现大量输入的大量布尔函数，最简单的解决方案包括将这些布尔函数中的每一个的连续successive值存储在memory的successive words中，CU的所有输入，由内容增加计数器augmented by the contents of a counter counting对执行特定指令所需的连续时钟周期successive clock cycle进行计数，该指令用于寻址该存储器的字。

The control unit is a set if Boolean functions and includes:

  • a combinational Boolean function of the bits from the instruction register;
  • various input signals;
  • timing counter;

这样的存储器，其内容决定了计算机的entire operation，通常称为“微存储microstore”，微存储的内容称为“微程序microprogram”。

微型存储器可以是只读存储器 (ROM)，其内容在制造过程中是固定的，也可以是允许动态修改微程序的可写存储器。可写微存储允许在计算机的生命周期内修改计算机的指令集。这对于纠正微程序中的错误、添加可以加速特定程序的特定指令或什至构建可以本地使用不同指令集的计算机非常有用。由于可写存储器通常比 ROM 使用更多的芯片空间和能量，几乎所有现代处理器都有 ROM 微型存储器。

Micro programmed Control Unit

微程序控制单元microprogrammed control unit的全局组织可以概括如下：

  • Instruction register的内容用作mapping rom 中的地址，其中包含与instruction register中包含的指令的执行相对应的序列的主微存储main microstore中的开始地址。
  • 对于 microstore 包含的每条指令，在连续字successive word中，每个clock cycle读取一个，每个控制线control line为执行指令需要采取的状态序列。
  • 除了processor的控制信号control signal外，microstore还包含一些附加位additional bits，这些位与来自 ALU 和中断线interrupt line的条件代码相结合，控制微指令的排序sequencing：
  • 当一条指令完成时，通过mapping rom从instruction register中获取下一条microinstruction的地址。
  • 当中断发生时，当前指令完成，但不是启动下一条指令，而是执行用于激活中断处理程序的microstore。
  • 在所有其他情况下，获取fetch下一条微指令microinstruction。

由于一条指令的执行是从 microstore 中获取许多microinstruction的结果，因此后者和相关的电子设备需要比微程序计算机microprogrammed computer执行指令的节奏快一个数量级。

Horizontal & vertical microprogramming

在高级advanced微程序处理器microprogrammed processor的简单设计中，每条微指令中的位数number of bits可能非常大，因为需要控制许多设备。 数百位是正常的。 这种方法被称为“横向微编程horizontal microprogramming”

然而，许多控制功能是相互排斥的（例如，不允许两个不同的register同时将它们的内容放在同一条总线上）并且可以编码以减少microinstruction的大小，这会导致更短的microinstruction，但它减少了并行parallel执行不同操作的机会，通常会导致更长的微程序。 这被称为“垂直微编程vertical microprogramming”。

Risc vs Cisc

在设计计算机体系结构时，必须在一组简单但快速的指令 (RISC) 或一组较慢但功能强大的 (CISC) 指令之间进行选择。

当每个芯片的晶体管transistor数量是一个关键的设计问题时，选择 CISC 架构的架构师需要一个大型微型存储microstore，因此无法负担大型片上缓存cache。

RISC 架构的推动者正确地声称，cache中的transistor比microstore中的transistor对整体性能的贡献更大。

  • RISC: a small set of simple but fast instructions;
  • CISC: a large set of slower but powerful instructions;

对于汇编级程序员来说，CISC 方法很有吸引力，因为它减少了他们的程序长度，因此许多 60 年代和 70 年代出生的处理器系列都有一个复杂的指令集。出于向上兼容的动机，这些指令集一直存在到现在。

另一方面，assembly language大多被高级语言取代，为简单指令集编写编译器比为复杂指令集编写编译器更容易，这就解释了为什么八九十年代设计的指令集大多属于RISC变种。

在九十年代后期，每个芯片的晶体管数量变得不那么重要，并且由于专门为 x86 指令集优化的软件的大量可用性，带有 x86 指令集微程序解释器的 RISC 处理器变得非常普遍。

History of mircoprogramming

1951 年，剑桥大学的Maurice Vincent Wilkes发明了计算机control unit的微编程microprogramming，但由于缺乏良好的只读存储器read-only memories和读写存储器read-write memories 的成本，该概念在工业用途中不切实际。

ROM (Read Only Memory) is a kind of microstore. Its contents can be fixed during the process;

Writeable memory that can be modified dynamically by microprogram.

IBM 于 1963 年为其非常成功的 360 系列大型计算机采用了微程序架构。

在 70 年代，非常快速的 RAM 和 ROM 存储器的可用性使小型计算机制造商能够为特定应用程序提供专门的指令集，例如用于加速数字信号处理的快速傅立叶变换指令。为了允许在一台机器上执行针对不同大型机的目标代码，一家公司 NanoData 公司甚至提出了一种大型机，其中每个程序都可以加载它想要使用的指令集，但这并没有取得商业上的成功。

消除（部分）经典指令集并允许编译器直接生成微代码有时被处理器设计人员认为是一种增加并行度parallelism的方法，而无需大量电路来防止产生结果的指令和使用它们的指令之间的竞争条件操作数。通过这种方式，应用程序程序员和编译器设计人员可以在应用程序允许的情况下引入并行性parallelism。

遵循这种方法的计算机体系结构称为“超大指令字体系结构very large instruction word architectures”。然而，惠普和英特尔在 2010 年左右沿着这个思路联合开发的Itanium处理器家族似乎并没有取得巨大的商业成功，可能是因为编译器设计人员和应用程序开发人员似乎没有接受挑战.

Evolution of processor chips

1965 年至 2005 年间，处理器的进步是基于技术进步：在一个芯片上集成更多更快的晶体管。

Gordon Moore's law:

  • doubling number of transistors every two years
  • Smaller transistors = higher clock speeds
  • Processor performance doubles every 18 months

BUT

  • What can be done with more transistors ? ?
  • Clock frequency > 5GHz problematic on large chips
  • Put 2 or more processors on the same chip
  • Paradigm shift: Technology parallel processing

到 2005 年，指数级的进步不得不停止：额外的晶体管没有任何有意义的用途，更高的时钟频率clock frequency导致了主要的设计问题。

由于晶体管的数量仍有可能增加，与其试图将更强大的处理器集成到一个芯片上，不如将上一代的两个、四个甚至八个处理器放在一个芯片上。这一举措将计算性能问题从处理器设计转向了高效可靠的并行处理算法parallel processing algorithm的设计。

Cloud Computing

高速通信，首先是在组织内部，后来是通过互联网在全球范围内，迅速导致了客户端服务器架构的普遍化：最终用户拥有一台针对用户友好性进行了优化的个人计算机，它调用专用服务器，首先用于存储，然后用于繁重的计算。 由于数据的安全存储需要在不同位置复制数据，因此中小型组织发现将其数据存储外包给专业公司很有吸引力。

计算服务器性能的提高，以及在不同系统上模拟任何计算机系统的便利性也迅速导致了外包处理。

事实上，我们又回到了集中式大型机时代，它具有所有优点（成本较低）和缺点（您的数据处理完全由外部组织控制）。

  • High-speed Local Area Networks + Optimization of resources ( hardware, software and personnel ) = Generalization of client-server architectures.
  • Reliable data-storage @ multiple sites + high-speed internet outsourcing of storage
  • Mastering of parallel processing techniques + more and more powerful processors oversized servers
  • Available processing power allows emulation of most computers on any other processor .
  • Processing is also being outsourced !
