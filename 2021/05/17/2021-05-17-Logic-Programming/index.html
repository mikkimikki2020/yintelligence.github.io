Logic Programming

Slide 62 ~ 78

给定一个已知事实数据库和一个规则数据库，系统能否自动回答查询？

Prolog

  • Logic programming language
  • programming on logic
  • Early seventies
  • Initially an interpreter, later on also compilers.
  • Automatic reasoning on rules, queries and facts.

Facts

male(john).
female(mary).
male(marc)
parent(john,marc).
parent(mary,marc).
  • Facts是始终正确的断言。
  • Facts存储在数据库中：事实库factbase
  • 没有大写！
  • 以点结束！

Rules

boy ( john ) :−male ( john ) .
father(john,mary) :- parent(john,mary),male(john).

如果 john 是男性，则 john 是男孩。如果 X 是 Y 的父母且AND X 是男性，则 X 是 Y 的父亲。

  • 如果规则体的所有子句都成立，则规则头结束。
  • 规则存储在数据库中 : rulebase.
  • Rules 可以有变量，它们以大写开头！
  • 以点结束！

Queries

check against facts and rules;

male(X).

parent(X, marc).

father(X,Y).
  • Who is male?
    • →[[X=john],[X=marc]]
  • Who are the parents of marc?
    • →[[X=john],[X=mary]]
  • Find all father - child relations
    • →[[X=john, Y=marc]]

Answering Queries

  • Query是一种事实（现在可能包含变量），用于检查事实和规则。
  • Prolog 将尝试将变量绑定到factbase中出现的值。
  • Prolog 根据factbase中的事实应用规则库中的规则并尝试解决查询。
  • 不那么明显......

Get Prolog

  • Install a free PROLOG version (e.g. YAP Yet Another Prolog).
  • Make a text file, put facts and rules in the file, save it as .pl
  • Run prolog and type [’testfile.pl’]. This loads the facts and the rules.
  • A prompt appears ? put your query there and press ¡enter¿

Our objectives

  • Prolog 可以做的更多，我们不在乎。我们只想展示逻辑编程logic programming的力量。
  • 我们定义了 ToyProlog，有事实、简单的规则和查询。
  • 我们研究 ToyProlog 如何工作。
  • 我们研究可以用 Toyprolog 构建的强大推理系统的例子。
  • 我们将实现 ToyProlog。

ToyProlog can only do three things

  1. ToyProlog 可以存储新的事实facts。 我们假设事实存储在简单的数据库中，例如 一个列表。
  2. ToyProlog 可以存储新规则rules。 我们假设规则存储在简单的数据库中，例如 一个列表。
  3. 我们专注于 ToyProlog 如何在rulebase和factbase上操作以回答查询。

The fundamental principle: Resolution

  • 允许我们用是或否回答一些简单的查询。
  • 如果 A 成立并且 A implies B，那么我们可以得出结论 B.
  • A 是事实，A implies B 是规则，B 是新事实。

From the fact and the rule:

male(john).
boy(john) :- male(john) //get new fact boy(john).

We can conclude the new fact:

boy(john).
  • 规则和查询可以有变量。
  • 事实没有变量。
  • 将事实与查询相匹配称为Pattern Matching。
  • 将规则子句clause与查询相匹配称为Unification。

Pattern matching

将fact与query相匹配称为Pattern Matching。

  • 为了知道是否可以为某个 X 得出规则的头部，我们需要将规则的主体与事实库中的事实进行匹配。
  • 如果谓词predicate相同并且变量 X 可以绑定到 john，那么这种匹配是可能的。

Using pattern matching and unification to answer queries

Assume again the facts and the rule:

male(john).
female(mary).
male(marc).
parent(john,marc).
parent(mary,marc).
father(X,Y) :− parent (X,Y),male (X).

We would like to answer the query:

father(A,B)

  1. Match all facts in the factbase with the query

    ⇒ No matches in this case.

  2. For each rule in the rulebase:

    1. Unify the conclusions of the rule with the query.
    2. Upon successful unification of the conclusion, try to answer the body of the rule.

    ⇒ Unification of the conclusion of the rule (father(X,Y)) with the query father(A,B) results in the frame {X=A,Y=B}. So, we need to answer the query parent(X,Y), male(X) given this frame (= recursion step).

Implementing an interpreter for ToyProlog

ToyProlog at a glance

class Interpreter:
    def __init__(self):
        self._facts = DBase()
        self._rules = DBase()
        self._parser = Parser()

	def addFact(self, someFact):
        pattern = self.parser.parse(someFact)
        self._facts.store(pattern)
        
	def addRule(self, someRule):
        pattern = self._parser.parse(someRule)
        self._rules.store (pattern)
        
	def getFactBase(self):
        return self._facts
    
    def getRuleBase(self):
        return self.rules
    
    def _qeval(self, query):
        frameStream = []
        frameStream.append(Frame())
        return query.qeval(frameStream, self._facts, self ._rules)
    
    def repl(self, string):
        o = self._parser.parse(string)
        for frame in self._qeval(o):
            print(frame)

class DBase:
    def __init__(self):

Frame

A Frame is a dictionary of associations between variables and values.

class Frame:
    def __init__(self):
        self._bindings = {}

	def addBindingToFrame(self, name, value):
        self._bindings[name] = value
        
	def containsBinding(self, name):
        return name in self._bindings
    
    def getBinding(self, name):
        return self._bindings[name]
    
    def __str__(self):
        result = []
        for key, value in self._bindings.items():
            result.append(str(key) + ”:” + str(value))
        return ','. join (result)

The queries, facts and rules are composed of patterns.

class Pattern:
    def __init__(self, exps):
        self._expressions = exps
        
	def __len__(self):
        return len(self._expressions)
    
    def __getitem__(self,item):
        return self._expressions[item]
    
    def __str__(self):
        expstrings = [str (x) for x in self._expressions ]
        return '[' + ''.join(expstrings) + ']'
    
    def rename(self, prefix):
        for exp in self._expressions:
            exp.rename (prefix)
            
	def dependsOn(self, variable, frame):
        any ([x. dependsOn (variable, frame) for x in self ._expressions])

class Pattern:

        result = [] 
        for rule in ruleBase:
            result += self.applyRule(rule, frame, factBase, ruleBase)
        return result
    
    def _applyRule(self, rule, frame , factBase, ruleBase) :
        newRule = rule.rename ()
        resFrame = Matcher.unifyMatch(self,newRule.getConclusion (), frame)
        if resFrame is None: 
            return []
        else:
            frames = [resFrame]
            ruleBody = newRule.getBody()
            if len(ruleBody) == 0:
                return frames
            else:
                return ruleBody.qeval(frames, factBase, ruleBase)

Pattern matching

class Matcher ():
    def patternMatch (pattern1, pattern2, frame):
        if frame is None:
            return frame
        elif pattern1 == pattern2:
            return frame
        elif isinstance (pattern1, toyProlog.Variable.Variable):
            return Matcher._extendIfConsistent(pattern1, pattern2, frame)
        elif isinstance (pattern1, toyProlog. Pattern. Pattern) 
        and isinstance (pattern2, toyProlog. Pattern. Pattern):
        if len(pattern1) != len(pattern2):
            return None
        extendedFrame = frame
        for exp1 , exp2 in zip (pattern1, pattern2) :
            extendedFrame = Matcher.patternMatch (exp1, exp2, extendedFrame)
            return extendedFrame
        else:
            return None
