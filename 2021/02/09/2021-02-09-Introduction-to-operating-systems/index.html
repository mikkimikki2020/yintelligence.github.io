Introduction to Operating Systems

What is an operating system? 什么是操作系统？

Lots of hardware !

  • One or more processors
  • Main memory
  • Disks
  • Printers
  • Various input/output devices

Managing all hardware requires a layer of software – the operating system.

  • Provide consistent abstractions to applications, even on different hardware

    为应用程序提供一致的abstractions，即使在不同的硬件上

  • Manage sharing of resources among multiple applications

    管理多个应用程序之间的资源共享

  • Protected access to shared resources

    对共享资源的保护访问

  • Security, authentication, isolation, fault tolerance

    安全、认证、隔离、容错

  • Communication among logical entities

    逻辑实体之间的通信

  • Referee (sharing CPU resources), illusionist (isolation and protection), glue (providing services)

    裁判（共享CPU资源）、幻术师（隔离保护）、胶水（提供服务）

The one program running at all times on the computer is the kernel – Everything else is either a system program (ships with the operating system) or an application program

始终在计算机上运行的一个程序是内核kernel——其他一切要么是系统程序（随操作系统一起提供）要么是应用程序

OS Abstracts Underlying Hardware to help Tame Complexity 操作系统抽象底层硬件以帮助驯服复杂性

  • Processor → Thread

    处理器 → 线程

  • Memory → Address Space

    内存 → 地址空间

  • Disks, SSDs, … → Files

    磁盘、固态硬盘…… → 文件

  • Networks → Sockets

    网络 → 网络套接字Sockets

  • Machines → Processes

    机器 → 流程Processes

  • What hardware interface to handle? (physical reality)

    要处理什么硬件接口？（物理现实）

  • What’s software interface to provide? (nicer abstraction)

    提供什么软件界面？（更好的抽象）

Where is the software? 软件在哪里？

Two modes: User mode and Kernel mode

两种模式：用户模式和内核模式

运行 Windows 的计算机中的processor有两个不同模式：User模式 和Kernel模式 。 根据processor上运行的代码的类型，处理器在两个模式之间切换。

App在User模式下运行，核心操作系统组件Core operating system components在Kernel模式下运行。 虽然许多驱动程序以Kernel模式运行，但某些驱动程序可能以User模式运行。

  • 启动User模式应用程序时，OS会为该App创建进程 。 process为应用程序提供专用private的“虚拟地址空间virtual address space” 和private的“句柄表handle table” 。

    由于App的虚拟地址空间为private空间，因此一个App无法更改属于其他App的数据。

    • 每个App都隔离运行，如果一个App发生故障，则故障仅局限于该应用程序。 其他App和OS不会受该故障的影响。

    除了专用private之外，User模式应用程序的virtual address space也受到限制。

    在User模式下运行的processor无法访问为OS保留的虚拟地址。 限制User模式应用程序的virtual address space可防止App更改以及可能损坏关键的OS数据。

  • 在Kernel模式下运行的所有代码都共享share单个虚拟地址空间virtual address space。

    这意味着Kernel模式驱动程序不会与其他App和OS隔离。

    如果Kernel模式驱动程序意外写入错误的虚拟地址，则属于OS或其他App的数据可能会受到安全威胁。

    • 如果Kernel模式驱动程序发生故障，整个操作系统就会发生故障。

此图说明了User模式组件与Kernel模式组件之间的通信。

Protecting processes 保护进程

Two features offered by hardware:

  • dual mode operation 双模操作

    • processes run in user or kernel mode (some operations prohibited in user mode)

      进程在user模式或kernel模式下运行（某些操作在用户模式下被禁止）

  • address translation 地址转换

    • processes have distinct/isolated address spaces 't R/W memory of another program or the OS)

      进程具有不同/隔绝的地址空间（不是另一个程序或操作系统的 R/W 内存）

      • each process进程 = (an address space地址空间) + (>=1 threads of control控制线程)

    • HW translates from virtual to physical addresses

      硬件将虚拟地址转换为物理地址

Why do you need an operating system? 为什么需要操作系统？

  • Allow multiple programs to run at the same time

    允许多个程序同时运行

  • Manage and protect memory, I/O devices, and other resources

    管理和保护内存、I/O 设备和其他资源

  • Multiplexes (shares) resources in two different ways

    以两种不同的方式复用（共享）资源

    • In time
    • In space

Operating system

This software layer provides low-level services to a program (application).

该软件层为程序（应用程序）提供低级服务。

  • Platform-specific hardware abstraction layers

    特定于平台的硬件抽象层

  • File management, disk access

    文件管理、磁盘访问

  • Keyboard/display interfacing

    键盘/显示器接口

  • Integrated power-management systems

    集成电源管理系统

  • Device drivers for common peripherals

    常见外围设备的设备驱动程序

  • Scheduling multiple programs for execution

    调度多个程序执行

    • Or even just multiple threads from one program

      甚至只是来自一个程序的多个线程

  • Program makes system calls to the OS

    程序对操作系统进行系统调用

The OS ZOO

  • Mainframe operating systems
    • Big-thousands of disks....
    • Lots of jobs with lots of I/O
  • Services-batch (payroll) transactions (airline reservations, timesharing (query database)
    • Elderly-Unix, Linux replacing them
    • Server operating systems
    • Workstations
    • File, print, web servers
    • BSD, Linux, Windows
  • Multiprocessor operating systems
    • Use multiple cores
  • PC operating systems
    • Linux, Mac, Windows
  • Smart phone operating systems
    • Android, iPhone, Blackberry (past: Palm, Symbian popular OS’s)
  • Embedded operating systems (e.g. TV sets, cars, MP3s)
    • Everything is in ROM (no apps can run on it)
    • QNx, Vxworks
  • Real-time operating systems
    • Hard (e.g. factory) deadline
    • Soft (e.g. multi-media) deadline
  • Smart card OS (e.g. border crossing cards)•Java in ROM

Computer hardware

CPU

  1. three-stage pipeline.
  2. A superscalar CPU.

  1. Intel style: A quad-core chip with a shared L2 cache.
  2. AMD style: A quad-core chip with separate L2 caches.

Memory hierarchy

Caches

Main memory is divided into cache lines (64 bytes)

  • 0-63 in line 1, 64-127 in line 2

    When program reads a word-cache hardware checks to see if in cache.

  • If so, then have a cache hit (2 cycles).

  • Otherwise, make request of main memory over the bus (expensive)

Cache is expensive and is therefore limited in size

Can have cache hierarchies

Cache other things, like URL addresses

Main memory

  • RAM: For processing data storage
  • ROM-can’t be changed. Fast, cheap
    • eg.-keeps bootstrap OS loader
  • EEPROM (Electrically Erasable PROM) Can be re-written, but slowly
    • E.g.-serves as films in digital cameras, as disk in portable music players

Disks

  • Tracks are divided into sectors (512 bytes)
  • Multiple tracks form a cylinder

I/O devices

  • Controller runs a device-accepts commands from the OS and executes them
  • Complicated business
    • Eg. Gets command to read sector x on disk y. Must convert to (cylinder, sector, head) address, move arm to correct cylinder, wait for sector to rotate under the head, read and store bits coming off the drive, compute checksum, store bits as words in memory
  • Controller contains a computer to run its device

The Bus hierarchy

The structure of a large Pentium system

  • In the beginning there was one bus-couldn’t handle the traffic when cpu and memories got faster and bigger
  • Create a hierarchy of faster buses (PCI, PCIe) and specialized buses (SCSI, USB)
