Local Search Algorithms

Good for optimization problems

适合优化问题

  • no goal test

    没有目标测试

  • the path to the goal is irrelevant

    通往目标的道路无关紧要

  • the goal state itself is the solution

    目标状态本身就是解决方案

    • integrated circuit design 集成电路设计
    • factory-floor layout 厂房布局
    • automatic programming 自动编程
    • telecommunications network optimization 电信网络优化

Find configuration satisfying constraints

找到满足约束的配置

  • e.g. n-queens (we return to this later)

State

  • keep a single "current" state, try to improve it
  • cf. keeping the entire path and trying to optimise that

Advantages

  • Use little memory (usually constant amount)

    保持单一的“当前”状态，尝试改进它

  • Can find reasonable solutions in large or infinite (continuous) problems

    保留整个路径并尝试优化它

Example: the n-Queens puzzle

Put n queens on an n × n chess board such that no queen can attack another

  • ie. no two queens on the same row, column, or diagonal

State space landscape

A state has

  • Location 地点

    • where it is in the state space

      它在状态空间中的位置

  • Elevation 海拔

    • its heuristic cost or objective value

      它的启发式成本或目标值

Solutions (two ways of thinking about it, but equivalent)

  • Heuristic cost 启发式成本 (find minimum cost 最小成本) : global minimum 全局最小值
  • Objective function 目标函数 (find best state 最佳状态 or maximal cost 最大成本): global maximum 全局最大值

“Local” because we don’t care about the path through the search space

“Local”，因为我们不关心通过搜索空间的路径

A local search algorithm is

  • complete: if it always finds a goal if one exists
  • optimal: if it finds the global minimum/maximum

Problem: depending on initial state, search can get stuck in local maxima/ minima

问题：根据初始状态，搜索可能会陷入局部最大值/最小值

Hill-climbing search

Like agenda-based search, but the agenda contains only one item

类似于基于议程的搜索，但议程仅包含一项

  • so we take the best option at each stage, and throw away all the others

    所以我们在每个阶段都采取最好的选择，然后扔掉所有其他的

Local search: use complete states 使用完整状态

  • always 8 queens on board 棋盘上总是有 8 位皇后

  • the numbers in the example count attacks if a piece moves vertically

    如果棋子垂直移动，示例中的数字会计算攻击次数

Heuristic 启发式

  • h = number of pairs of queens that are attacking each other, either directly or indirectly

    h = 直接或间接互相攻击的皇后对数

  • h = 17 for the example

  • h = 0 is a solution state (or global minimum)

Successors?

  • The set of states generated by moving one queen

    移动一个皇后产生的一组状态

8-queens problem

  • A local minimum with h = 1
  • What is the set of successor states?
  • Why is this a local minimum?
  • What is the next step of hill-climbing?

Greedy local search

贪心算法，又名贪婪法，是寻找最优解问题的常用方法，这种方法模式一般将求解过程分成若干个步骤，但每个步骤都应用贪心原则，选取当前状态下最好/最优的选择（局部最有利的选择），并以此希望最后堆叠出的结果也是最好/最优的解。

这就好像一个贪婪的人，他事事都想要眼前看到最好的那个，看不到长远的东西，也不为最终的结果和将来着想，贪图眼前局部的利益最大化，有点走一步看一步的感觉。

  • grabs best neighbour without “thinking ahead”

Rapid improvement in heuristic value 启发式值的快速提升

  • But at a fatal cost 但付出了致命的代价

  • Local Maxima/Minima will stop search altogether

    本地最大值/最小值将完全停止搜索

    • Ridges: a sequence of local maxima that make it difficult to escape.

      山脊：一系列局部最大值，使其难以逃脱。

    • Plateaux: heuristic values uninformative

      高原：启发式值无信息

    • Foothills: local maxima that are not global maxima

      山麓：不是全局最大值的局部最大值

How bad is the problem?

Algorithm reaches a point where no progress is made 算法达到了没有任何进展的地步

To test on the 8-queens example:

  • run many searches starting from random configurations of 8-queens

    • too many to do it exhaustively

  • result

    • 86% of starting configurations get stuck
    • 14% of starting configurations succeed

  • the algorithms works quickly, usually only evaluating 3-4 steps before either getting stuck or finding a solution

    算法运行迅速，通常只评估 3-4 个步骤，然后就会陷入困境或找到解决方案

How can we solve the problem?

Allow different moves 允许不同的动作

  • e.g., sideways

  • must account for local maxima (foothills) otherwise could get stuck in an infinite loop

    必须考虑局部最大值（山麓），否则可能陷入无限循环

    • e.g., cap number of sideways moves

      例如，限制横向移动的次数

    • but we risk losing generality with such ad hoc solutions

      但我们可能会因这种临时解决方案而失去一般性

Result:

  • 6% stuck
  • 94% solution

Works much slower 工作慢得多

  • evaluating 21-64 steps before either getting stuck or finding a solution

Variants of hill-climbing

  • Stochastic hill-climbing 随机爬山

    • choose randomly between available uphill moves

      在可用的上坡动作之间随机选择

      • choose from uniform or non-uniform distribution

        选择均匀分布或非均匀分布

    • converges more slowly but sometimes finds better solutions

      收敛速度较慢，但有时会找到更好的解决方案

      • optimality still not guaranteed in all cases

        在所有情况下仍不能保证最优性

  • First-choice hill-climbing 首选爬山
    • Good for big problems 适合大问题

      • Uses Stochastic hill climbing but randomly generates successors and picks first larger one

        使用随机爬山但随机生成后继并选择第一个较大的

  • Random-restart hill-climbing 随机重启爬山

    • If at first you don't succeed, try again, starting from a different place

      如果一开始你没有成功，再试一次，从不同的地方开始

      • requires that non-optimality can be recognised

        要求可以识别非最优性

Simulated Annealing search

Hill-climbing

  • only improves on the current solution

    只改进了当前的解决方案

  • not complete

    • may get stuck in local minima/maxima

      可能会陷入局部最小值/最大值

Random walk

  • moves from state to state randomly

    随机地从一个状态移动到另一个状态

  • complete (given infinite time) but very inefficient

    完成（给定无限时间）但效率非常低

Simulated Annealing combines these two to give a compromise between search complexity and completeness

Simulated Annealing结合了这两者以在搜索复杂性和完整性之间做出折衷

Escape local maxima by allowing some bad moves

通过允许一些不好的动作来逃避局部最大值

Gradually decrease frequency of allowed bad moves as search proceeds

随着搜索的进行逐渐减少允许的错误动作的频率

By analogy with a process of hardening in steel production 以钢铁生产中的硬化过程为例

  • want crystal structure of metal to be all lined up

    想要金属的晶体结构都排列整齐

  • heat the metal to make the crystals vibrate and “shake out” irregularities

    加热金属使晶体振动并“抖出”不规则性

  • let it cool slowly so that the more ordered form stays ordered

    让它慢慢冷却，以便更有序的形式保持有序

So in the algorithm, we have an imaginary notion of “temperature”, which allows random movement outside the hill-climb

所以在算法中，我们有一个假想的“温度”概念，它允许爬山外的随机运动

  • as the “temperature” drops, less random movement is allowed

    随着“温度”的下降，减少随机运动的频率

Like getting a ping-pong ball into the deepest crevice of a bumpy surface

就像将乒乓球放入凹凸不平的表面最深的缝隙中

  • Left alone by itself, ball will roll into a local minimum

    单独放置，球将滚入局部最小值

  • If we shake the surface, we can bounce the ball out of a local minimum

    如果我们摇动表面，我们可以将球反弹出局部最小值

  • The trick is to shake hard enough to get it out of local minimum, but not hard enough to dislodge it from global one

    诀窍是用力摇晃以使其脱离局部最小值，但又不足以将其从全局最小值中移出

  • We start by shaking hard and then gradually reduce the intensity of shaking

    我们从用力摇晃开始，然后逐渐降低摇晃的强度

If the “temperature” decreases slowly enough, simulated annealing search finds a global optimum with probability approaching 1.0

如果“温度”下降得足够慢，模拟退火搜索会找到一个概率接近 1.0 的全局最优值

Widely used in VLSI layout, airlines scheduling, etc.

广泛应用于超大规模集成电路布局、航空公司调度等。

Local Beam Search

Keep track of k states rather than just one

跟踪k个状态，而不仅仅是一个

Start with k randomly generated states

从k 个随机生成的状态开始

At each iteration

  • generate all successors of all k states

    生成所有k个状态的 所有后继

    • NB: not the same as an agenda k long!

  • if any one is a goal state, stop

    如果任何一个是目标状态，停止

  • else select the k best successors from the complete list and repeat

    否则从完整列表中选择 k 个最佳后继者并重复

So this is like a kind of selective breadth first search

所以这就像一种选择性广度优先搜索

Local beam search looks like running k hill-climbing algorithms in parallel, but it is not

局部波束搜索看起来像并行运行 k 个爬山算法，但它不是

  • the results of all k states influence each other

    所有k个状态的结果相互影响

  • if one state generates several good successors; they all end up in the next iteration

    如果一个状态产生了几个好的继承者；他们都在下一次迭代中结束

  • states generating bad successors are weeded out

    产生不良继任者的状态被淘汰

This is both a strength and a weakness:

  • unfruitful searches are quickly abandoned and searches making the most progress are intensified

    没有结果的搜索很快被放弃，取得最大进展的搜索得到加强

  • can lead to a lack of diversity: concentration in a small region of the search space

    可能导致缺乏多样性：集中在搜索空间的一个小区域

  • remedy: choose k successors randomly, biasing choice towards good ones

    补救措施：随机选择k个后继者，偏向于好的选择

Genetic Algorithms

遗传算法通常实现方式为一种计算机模拟。对于一个最优化问题，一定数量的候选解（称为个体）可抽象表示为染色体，使种群向更好的解进化。

传统上，解用二进制表示（即0和1的串），但也可以用其他表示方法。进化从完全随机个体的种群开始，之后一代一代发生。在每一代中评价整个种群的适应度，从当前种群中随机地选择多个个体（基于它们的适应度），通过自然选择和突变产生新的生命种群，该种群在算法的下一次迭代中成为当前种群。

Genetic Algorithms (GAs) are a variant of local beam search

遗传算法 (GA) 是Local Beam Search的一种变体

  • A successor state is generated by combining two parent states

    通过组合两个父状态生成后继状态

  • Start with a large number of randomly generated states (a population)

    从大量随机生成的状态（一个群体）开始

  • A state is represented as a binary string

    状态表示为二进制字符串

  • An evaluation function or fitness function assesses the quality of a state

    评估函数或适应度函数评估状态的质量

    • higher values for better states.

Produce the next generation of states by

  • selection 选择
  • crossover 交叉
  • mutation 突变

Problem encoding for GAs

The way that you choose to encode the problem often has implications for the solution

您选择对问题进行编码的方式通常会对解决方案产生影响

Example: 8 queens, 8 × 8 board

  • 64 bits, one for each square? (= 8x8 matrix)
  • 8 octal digits? (one 3-bit number for each column/row)

21641300 (octal)

010001 110010001011000000 (binary)

Fitness function: number of nonattacking pairs of queens (min = 0, max = 28) ‣

适应度函数：非攻击皇后对的数量（最小 = 0，最大 = 28）

  • Fitness here = 6 + 5 + 4 + 3 + 3 + 2 + 0 = 23

GA Selection

After encoding states and calculating fitness, select pairs for reproduction

编码状态和计算适应度后，选择对进行繁殖

  • many methods can be used

Simplest: randomly choose pairs of states with non-uniform probability

最简单：随机选择具有非均匀概率的状态对

Selection for Reproduction

Cross-over

Pairs are selected by one of a range of methods:

  • Roulette-wheel (as in our example)
  • Tournament

Cross-over point for each pair is randomly selected

每对的交叉点是随机选择的

  • Resulting new chromosomes represent new states

    产生的新染色体代表新状态

Mutation

Cross-over is not enough

  • if the population does not contain examples that have each bit of the chromosome at both possible values parts of the search space are inaccessible

    如果总体不包含染色体的每一位都处于两个可能值的示例，则搜索空间的部分不可访问

So introduce mutation

  • low probability of flipping a random bit at each cross-over step

    在每个交叉步骤中翻转随机位的低概率

The whole process

  • Initial Population 初始人口

    • Random bit strings – make sure every bit means something

      随机位串——确保每一位都有意义

  • Selection

    • Select according to output of fitness function

      根据适应度函数的输出选择

    • Various methods (look them up!)

  • Cross-over

    • Random point in chromosome 染色体中的随机点

    • Various methods, always involving death of unfit chomosomes

      各种方法，总是涉及不合适的染色体的死亡

  • Mutation

    • To introduce possibilities not currently present in the population

      引入目前染色体中不存在的可能性

  • Repeat from “selection” until fitness levels are as desired

    从“选择”开始重复，直到达到所需的健康水平

Adversarial Search and Games

We have assumed so far that the world we are searching doesn't change

到目前为止，我们假设我们正在寻找的世界不会改变

  • This is often not the case

    情况往往并非如此

Consider Games

  • Two or more players in competition 两名或两名以上选手参加比赛

  • There are many kinds of game, but we will only deal with zero-sum games of perfect information

    博弈有很多种，但我们只处理完全信息的零和博弈

    • 2 players: player one wins → 1
    • ​ player two loses → -1
    • ​ TOTAL 0

Why study games?

Games are a form of multi-agent environment

游戏是多智能体环境的一种形式

  • What do other agents do and how do they affect our success?

    其他agent做什么以及他们如何影响我们的成功？

  • Cooperative vs. competitive multi-agent environments

    合作与竞争的多代理环境

  • Competitive multi-agent environments give rise to adversarial search = games

    竞争性多智能体环境导致对抗性搜索 = 游戏

  • Interesting subject of AI study because they are hard for humans

    人工智能研究的有趣主题，因为它们对人类来说很难

  • Easy to represent and agents restricted to small number of actions

    易于表示且代理仅限于少量操作

Adversarial vs. non-adversarial search

Search – no adversary 搜索 – 没有对手

  • Solution is (heuristic) method for finding goal

    解决方案是寻找目标的（启发式）方法

  • Heuristics and constraint satisfaction techniques can find optimal solution

    启发式和约束满足技术可以找到最优解

  • Evaluation function: estimate of cost from start to goal through given node

    评价功能：评价比赛位置的好坏

  • Examples: path planning, scheduling

    示例：路径规划、调度

Games – adversary 游戏—有对手

  • Solution is strategy (strategy specifies move for every possible opponent reply)

    解决方案是策略（策略为每个可能的对手回复指定移动

  • Sometimes time limits force an approximate solution

    有时时间限制迫使近似解

  • Evaluation function: evaluate quality of game position

    评价功能：评价比赛位置的好坏

  • Examples: chess, checkers, Othello, war games, simulations of competition for limited resources

    示例：国际象棋、跳棋、黑白棋、战争游戏、有限资源竞争模拟

Game setup

  • Two players: MAX and MIN

  • Initial state: e.g. board configuration of chess

  • Each player uses a search tree to determine next move

    每个玩家使用搜索树来决定下一步行动

MINIMAX algorithm

MINIMAX without αβ - pruning

Definition of optimal play for MAX assumes MIN plays optimally

MAX 最佳播放的定义假设 MIN 播放最佳

  • maximizes worst-case outcome for MAX

    最大化 MAX 的最坏情况结果

But if MIN plays worse than optimally

但是如果 MIN 的表现比最佳状态差

  • MAX will do even better

    MAX会做得更好

Complexity:

  • Time: O(bm) - exponential in depth of tree 树的深度指数
  • Space: O(m) - linear in depth of tree 树的深度线性

Can improve effective complexity by Alpha-Beta (α-β) Pruning

可以通过 Alpha-Beta (α-β) 剪枝提高有效复杂度

Alpha-beta (α-β) pruning in MINIMAX

Use the same algorithm as before, but consider ranges instead of values

使用与以前相同的算法，但考虑范围而不是值

  • range is expressed as a pair, [lowest,highest] or [α,β]

Alpha-Beta Algorithm

General alpha-beta pruning

Consider a node n somewhere in the tree 考虑树中某处的节点 n

If player has a better choice at 如果玩家有更好的选择

  • Parent node of n

    n 的父节点

  • Or any choice point further up

    或进一步向上的任何选择点

n will never be reached in actual play

在实际比赛中永远不会达到n

Hence when enough is known about n, it can be pruned.

因此，当对 n 了解足够多时，就可以对其进行修剪。

Properties of Alpha-Beta pruning

  • Pruning does not affect final results

    修剪不影响最终结果

  • Entire subtrees can be pruned

    可以修剪整个子树

  • Good move ordering improves effectiveness of pruning

    良好的移动排序可以提高修剪的效率

  • With “perfect ordering”, time complexity is O(bm/2)=O((b1/2)m)

    • Branching factor of √b

    • Best case Alpha-beta pruning can look twice as far ahead as plain MINIMAX in a given amount of time

      在给定的时间内，最佳情况下 Alpha-beta 剪枝的效果是普通 MINIMAX 的两倍

  • Repeated states are again possible 重复状态再次成为可能

    • Store them in memory; make large gain in memory from pruning

      将它们存储在内存中；从修剪中获得大量内存

Games of imperfect information

Minimax and alpha-beta pruning require too much leaf-node evaluations

Minimax 和 alpha-beta 剪枝需要太多的叶节点评估

May be impractical within a reasonable amount of time

在合理的时间内可能不切实际

Shannon (1950):

  • Cut off search earlier (replace TERMINAL-TEST by CUTOFF- TEST)

    提前切断搜索（用 CUTOFF-TEST 替换 TERMINAL-TEST）

  • Apply heuristic evaluation function EVAL (replacing UTILITY function of alpha-beta)

    应用启发式评估函数 EVAL（替换 alpha-beta 的 UTILITY 函数）

Cutting off search

Change the termination condition of MINIMAX:

改变 MINIMAX 的终止条件：

  • if TERMINAL-TEST(state) then return UTILITY(state)

into

  • if CUTOFF-TEST(state,depth) then return EVAL(state)

Can introduce a fixed or dynamic depth limit

可以引入固定或动态深度限制

  • selected so (e.g.) time will not exceed what the rules of the game allow

    选择这样（例如）时间不会超过游戏规则允许的

When cutoff occurs, and we are not at a terminal state, heuristic evaluation is performed

当截止发生时，我们未处于终止状态，则执行启发式评估

  • in other words, if you don’t know, then take your best guess

    换句话说，如果你不知道，那么你选择最好的猜测

Heuristic Evaluation

Idea:

  • produce an estimate of the expected utility of the game from a given position

    从给定位置产生对游戏的预期效用的估计

Performance depends on quality of EVAL 性能取决于EVAL的质量

Requirements

  • EVAL should value terminal-nodes in the same order as UTILITY

    EVAL 应该按照与 UTILITY 相同的顺序来评估终端节点

  • Computation may not take too long

    计算时间可能不会太长

  • For non-terminal states the EVAL should be strongly correlated with the actual chance of winning

    对于非终端状态，EVAL 应与实际获胜机会密切相关

  • Only useful for quiescent states (=no wild swings in value in near future)

    仅对静止状态有用（=在不久的将来价值不会出现剧烈波动)

Games that include chance

In backgammon, you roll 2 dice, and then use both numbers in either order

在双陆棋中，您掷 2 个骰子，然后按任一顺序使用这两个数字

  • so a double, [1,1], has probability 1/36, all other moves have probability 1/18

    以双数 [1,1] 的概率为 1/36，所有其他移动的概率为 1/1

In this tree, we calculate the expected value of MINIMAX

Expected MINIMAX value

Given a game tree, the optimal strategy can be determined by using the minimax value of each node, n:

给定一棵博弈树，可以通过使用每个节点的极大极小值 n 来确定最优策略

  • if n is a terminal node,

    • EXPECTED-MINIMAX-VALUE(n) = UTILITY(n)

  • if n is a MAX node,

    • EXPECTED-MINIMAX-VALUE(n) = maxs∈successors(n) EXPECTED-MINIMAX-VALUE(s)

  • if n is a MIN node,

    • EXPECTED-MINIMAX-VALUE(n) = mins∈successors(n) EXPECTED-MINIMAX-VALUE(s)

  • if n is a CHANCE node,

    如果 n 是一个 CHANCE 节点

    • EXPECTED-MINIMAX-VALUE(n) = ∑s∈successors(n) P(s)EXPECTED-MINIMAX-VALUE(s)

On left, A1 wins; on right, A2 wins

  • outcome of evaluation may change if values are not scaled linearly

    如果值不是线性缩放的，评估结果可能会改变

If you need to scale values, use a positive linear transformation

如果需要缩放值，请使用正线性变换

Summary

Games are fun, useful and potentially distracting

游戏有趣、有用且可能会分散注意力

They illustrate many important points about AI

  • Perfection is unattainable, so approximation is necessary

    完美是无法达到的，所以近似是必要的

  • We need a good idea of strategy to win

    我们需要一个好的策略来获胜

    • and MINIMAX allows us to express that in terms of final states - relatively easy

      而 MINIMAX 允许我们用最终状态来表达—相对容易

  • Uncertainty constrains the assignment of values to states

    不确定性限制了对状态的赋值

Games are to AI as grand prix racing is to car design.

游戏之于人工智能，就像大奖赛之于汽车设计一样。
