Discussion 3: Relational Algebra

Video

Example: Employees

Employees(employeeNr:integer, firstName:string, lastName:string)

employeeNr firstName lastName
1 Mickey Mouse
2 Donald Duck
3 Goofy Dog
4 Pluto Dog

Projection (π)

”Select the first names from all employees”

  • πfirstName(Employees)

This will return “Mickey”, “Donald”, “Goofy”, “Pluto”

Example: Drinks

Drinks(drinkName:string, amounts:integer)

drinkName amounts
Coca cola 4
Sprite 4
Fanta 6
Pepsi 3
7up 5

Selection (σ)

“Select the drink amounts that are 3”

  • πdrinkName(σamounts=3(Drinks))

This will return "Pepsi".

Example: Employees and Students

Employees(employeeNr:integer, firstName:string, lastName:string, empType: string)

employeeNr firstName lastName empType
1 Mickey Mouse BAAP
2 Donald Duck BAAP
3 Goofy Dog BAAP
4 Pluto Dog ZAP

Students(studentNr:integer, firstName:string, lastName:string, empType:string)

employeeNr firstName lastName
1 Mickey Mouse
2 Donald Duck
3 Minnie Mouse
4 Daisy Duck

Union (∪)

  • “Select the first and last name from all employees and students” πfirstName,lastName(Employees) ∪ πfirstName,lastName(Students)

This will return all first and last names

Disjoint (∩)

“Select the first and last name from all employees who are students”

  • πfirstName,lastName(Employees) ∩ πfirstName,lastName(Students)

This will return Mickey and Donald

Set Difference (−)

“Select the first and last name from all employees who are not students”

  • πfirstName,lastName(Employees) − πfirstName,lastName(Students)

This will return Goofy and Pluto

Example: Course enrollments, assistants

CourseEnrollment(studentNr:integer, courseName:string)

studentNr courseName
3 Databases
3 Web Technologies
4 Databases
3 Advanced Topics In Information Systems
4 Master Thesis

CourseAssistants(employeeNr:integer, courseName:string)

employeeNr courseName
1 Databases
1 Web Technologies
1 Advanced Topics In Information Systems
2 Databases

Natural Join (⋈)

“Get all courses and their assistants”

πcourseName,firstName,lastName(CourseAssistants ⋈ Employees)

Division (÷)

“Select the assistants who give all 3 credit courses”

πcourseName,firstName,lastName(CourseAssistants ÷ σcredits=3(Courses)) ⋈ Employees)

Assignment (←)

Select the first and last name from all students who are not employees”

StudentNames ← πfirstName,lastName(Students)

EmployeeNames ← πfirstName,lastName(Employees)

Result ← StudentNames − EmployeeNames

Delete (X ← X − ...)

"Delete Daisy Duck from the students schema.”

Students ← Students − σfirstName=“Daissy”∧lastName=“Duck”(Students)

Insert (X ← X ∪ ...)

“Add Daisy Duuuck to the students schema.”

Students ← Students ∪ {(4, “Daisy”, “ Duuuck”)}

Update

“Change the name Daisy Duuuck to ‘Daisy Duck’ in the students schema.”

Students ← πstudentNr,firstName,”Duck”(σstudentNr=4(Students)) ∪ πstudentNr,firstName,lastName(σstudentNr6=4(Students))

TIP: We only need to change the last name

Rename (ρ)

“Rename the schema of Students”

πfirstName2,lastName2(ρStudentsCopy(studentNr2,firstName2,lastName2)(Students))

Cartesian Product (×)

“Give all pairs of course names where the first course has more credits than the second.”

πcourseName,courseName2(σcredits>credits2(Courses × ρCourses2(courseName2,credits2)(Courses)))

Question 1

Given relationship R and S, assume a person is only allowed to teach the department Mathematics, minor in Computer Science if he or she holds the following degrees: Highschool, MSc Computer Science and MSc Mathematics. Give a query in relational algebra that returns the names of all persons that are allowed to teach in that department.

R =

Name Degree
Anne MSc Computer Science
Anne Highschool
Anne MSc Mathematics
Katja MSc Computer Science
Katja MSc Mathematics
Katja Highschool
Katja MSc Physics
William Highschool
William MSc Computer Science
Christophe Highschool
Christophe MSc Computer Science

S =

Degree
MSc Computer Science
Highschool
MSc Mathematics
MSc Physics

Solution 1

πName(R ÷ σDegree≠“MSc Physics”(S))

  1. We start by specifying the attribute that we want to find (πName).
  2. Next we need to specify the schema that we are going to use to get this name (R).
  3. We use R, but we are not interested in all names in that entity but only those who are allowed to teach in the department.
  4. To do so, we do a division of R with all ”Degrees” that are not MSc Physics (Highschool, MSc Mathematics, MSc Computer Science). A division. This division (slide 33 in lecture 3) will perform a select on the “Degree” column in R, selecting only those names that have Highschool, MSc Mathematics and MSc Computer Science.

Solution 2

πName(σDegree=“Highschool”(R)) ∩ πName(σDegree=“MSc Mathematics”(R)) ∩ πName(σDegree=“MSc Computer Science”(R))

The biggest difference between the two solutions is that with the previous solution, we took the complete set in the relation R, and removed all instances that did not have the required degrees.

For this solution:

  1. Take the complete set of names from everyone with a Highschool degree (in this exercise everyone)
  2. Next, take the disjunction of that set with everyone with a MSc Mathematics (everyone without this MSc in Mathematics is removed)
  3. Finally, do the same for the MSc in Computer Science (the only remaining names are Anne and Katja)

Question 2

Calculate R ÷ S step by step (where Sch(X) returns the schema of a relation X)

  • T1 ← πSch(R)−Sch(S)(R)
  • T2' ← (S × T1) − R
  • T2 ← πSch(R)−Sch(S)(T2')
  • Result ← T1 − T2

Note: Check whether the obtained result corresponds to the result of R ÷ S

Solution

  • T1

    Name
    Anne
    Katja
    William
    Christophe

    R contains names and multiple degrees that name may have. Our projection takes the schema of R minus the schema of S giving us a projection that selects names only. We are left with all names in R without their degrees.

  • T2'

    Name Degree
    Anne MSc Physics
    William MSc Physics
    William MSc Mathematics
    Christophe MSc Physics
    Christophe MSc Mathematics

    Here we multiply S with our previously obtained T1. This gives us an intermediate result where each of the 4 names has all degrees. Next, we subtract it with R giving us a result of all names and the degrees that they did not get yet.

  • T2

    Name
    Anne
    William
    Christophe

    Similar to T1 we only project the names of our column T2'

  • Result

    Name
    Katja

    In this final result we subtract T2 from T1 giving us only Katja left.

Question 3

Given the following schemas, express the provided queries in relational algebra.

  • Suppliers(sid:integer, sname:string, address:string)
  • Parts(pid:integer, pname:string, color :string)
  • Catalog(sid:integer, pid:integer, cost:real)
  1. Give names of all suppliers supplying red parts.
  2. Give sids of all suppliers supplying a red or green part.
  3. Give names of all suppliers supplying both red and green parts.
  4. Give names of all suppliers supplying every part.
  5. Return pairs of SIDs such that the first supplier in a pair asks more for a part than the second supplier in that pair.
  6. Look for PIDs of parts that are supplied by different suppliers.

Solution

Given the following schemas, express the provided queries in relational algebra.

  1. Give names of all suppliers supplying red parts.

    πsname((σcolor =“red”(Parts) ⋈ Catalog) ⋈ Suppliers)

  2. Give sids of all suppliers supplying a red or green part.

    πsid(σcolor=”red” ∨ color=“green”(Parts) ⋈ Catalog)

  3. Give names of all suppliers supplying both red and green parts.

    Red ← πsname((σcolor=“red”(Parts) ⋈ Catalog) ⋈ Suppliers)

    Green ← πsname((σcolor=“green”(Parts) ⋈ Catalog) ⋈ Suppliers)

    Result ← Red ∩ Green

  4. Give names of all suppliers supplying every part

    πsname((πsid,pid(Catalog) ÷ πpid(Parts) ⋈ Suppliers)

  5. Return pairs of SIDs such that the first supplier in a pair asks more for a part than the second supplier in that pair.

    πsid,sid2(σcost > cost2 ∧ pid=pid2(Catalog × ρC2(sid2,pid2,cost2)(Catalog)))

  6. Look for PIDs of parts that are supplied by different suppliers.

    πpid(σsid≠sid2(Catalog ⋈ ρC2(sid2,pid,cost2)(Catalog)))

    Or

    πpid(σsid≠sid2 ∧ pid=pid2(Catalog × ρC2(sid2,pid2,cost2)(Catalog)))

Question 4 (Extra)

Given the following schemas, express the provided queries in relational algebra

  • Students(sid:integer, sname:string, address:string)
  • Courses(cid:integer, cname:string, credits:integer)
  • Employees(eid:integer, ename:string, address:string)
  • Enrolments(cid:integer, sid:integer, ayear :integer)
  • Professors(cid:integer, eid:integer)
  • Assistants(cid:integer, eid:integer)
  • Faculties(fid:integer, fname:string, abbreviation:string)
  • Programs(pid:integer, fid:integer, pname:string)
  • ProgramCourses(pid:integer, cid:integer)
  1. Give the names of all students following the course named ”Databases”.
  2. Give the names of all assistants teaching the course named ”Databases”.
  3. Give the total amount of credits in the program with name (pname) ”MSc CS”.
  4. Give the abbreviation of all faculties that have programs with the Databases course as part of their program.

Solution

Given the following schemas, express the provided queries in relational algebra

  1. Give names of all students following the course named ”Databases”

    πsname((σcname=“Databases”(Courses) ⋈ Enrollments) ⋈ Students)

  2. Give the names of all assistants teaching the course named ”Databases”.

    πsname((σcname=“Databases”(Courses) ⋈ Assistants) ⋈ Employees)

  3. Give the total amount of credits in the program with name (pname) ”MSc CS”.

    πsum(credits)((σpname=“MSc CS”(Programs) ⋈ ProgramCourses) ⋈ Courses)

  4. Give the abbreviation of all faculties that have programs with the Databases course as part of their program.

    πabbreviation(((σcname=“Databases”(Courses) ⋈ ProgramCourses) ⋈ Programs) ⋈ Faculties)
