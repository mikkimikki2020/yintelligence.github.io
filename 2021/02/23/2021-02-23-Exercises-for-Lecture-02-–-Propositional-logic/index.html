Exercises for Propositional logic

Background

Note: Some of these exercises refer to the IDP system. You can download it from here or use the web-based version at here. The latter has resource limits and can thus only be used for small problems, but should be sufficient for the current assignment.

Template files to work with IDP for the different exercises are available on Canvas.

Exercise 1: Truth Tables

A journalist stops three friends (Mr. Rossi, Mr. Bianchi and Mr. Verdi) who just left the voting office. He interviews them separately about their votes in the elections. Mr. Rossi declares: “If Bianchi voted for Berlusconi, then also Verdi did”. Mr. Bianchi declares: “If Rossi did not vote for Berlusconi, then also Verdi did not”. Finally, Verdi declares: “Rossi voted for Berlusconi while Bianchi did not”. Let r, b, v denote “Mr Rossi (resp. Mr. Bianchi, Mr. Verdi) has voted for Berlusconi”.

Formalize the declarations of the three friends and use a truth table to answer the following questions:

  • Are the declarations consistent?
  • If all three voted for Berlusconi, who is lying?
  • If all three are telling the truth, who has voted for Berlusconi and who has voted against him?

When you are finished, formalize everything in the IDP system and use it to verify your answers.

Solution

vocabulary V{
	RossiVotesBerlusconi
	BianchiVotesBerlusconi
	VerdiVotesBerlusconi
    
    
}

vocabulary V_extended{
    extern vocabulary V
    RossiTruth
    BianchiTruth
    VerdiTruth
}

theory point1 :V{
 // Use this theory to solve the question "Are the declarations consistent?"
    BianchiVotesBerlusconi => VerdiVotesBerlusconi.
    ~ RossiVotesBerlusconi => ~VerdiVotesBerlusconi.
    RossiVotesBerlusconi & ~BianchiVotesBerlusconi. 
}

theory point2:V_extended{
 //Use this to solve the second question (hint: maybe you need to extended V for this...)
  RossiTruth <=> (BianchiVotesBerlusconi => VerdiVotesBerlusconi).
  BianchiTruth <=> (  ~ RossiVotesBerlusconi => ~VerdiVotesBerlusconi).
  VerdiTruth <=> ( RossiVotesBerlusconi & ~BianchiVotesBerlusconi ). 
}


structure no_assignments:V{
    
}

structure allThreeVoteB:V{
//If needed you can put (partial) assignments here
	RossiVotesBerlusconi = true
	BianchiVotesBerlusconi = true
	VerdiVotesBerlusconi = true

}

procedure main(){
    //Check the first and third question:
    printmodels(allmodels(point1,no_assignments))
    print("We see two solutions, from this we conclude the statements are consistent")
    print("We also see that in both solutions Bianchi did not vote Berslusconi and Rossi did vote Berlusconi. We can however make no conclusion about Verdi")

    //Check the last question
    printmodels(allmodels(point2,allThreeVoteB))
    print("We see that Verdi lies in this case")
}

The statements can be formalized in logic as follows:

Rossi says: b → v
Bianchi says: ¬r → ¬¬v
Verdi says: r ∧ ¬b

When creating the truth table, you should should see that

  • There are two lines in which all three statements are true. Hence, the declarations are consistent
  • In the line of the truth table in which b, v, and r are true, you can see that Verdi’s statement is false, while the other two are true.
  • If all three tell the truth, there are still two possibilities. We do know for sure that rossi voted for Berlusconi and bianchi against, but we do not know how verdi voted.

Exercise 2: CNF transformation

A classical constraint satisfaction problem is the Graph Coloring problem. A graph is defined as a tuple G = (V, E) where V = {v1, v2, . . . , vn} is the set of vertices and E = {(vi, vj), . . . ,(vk, vl)} the set of edges connecting pairs of vertices. The graph we work with is undirected (i.e. if (vi , vj) ∈ E then (vj , vi) ∈ E (reflexivity)). Assume that KB is a theory consisting of the general knowledge about colors and graphs and the constraint the coloring function does not assign the same color to connected vertices. Given a graph, the graph coloring problem is now to assign colors which satisfy KB.

Given a graph with nodes ‘i’, ‘a’ and ‘h’. Take the following propositional vocabulary:

  • eia, eah and eih represent whether there is an edge between the nodes.
  • bi , ba and bh represent whether the nodes are colored black (assume there are only two colors, so the negation indicates a node has the opposite color).

  1. Firstly, write down the propositional theory KB.

  2. Transform KB into CNF using DeMorgan’s laws.

Assume that node i represents Italy, a represents Austria and h Hungary and consider the additional knowledge that

  • Italy is connected to Austria
  • Austria is connected to Hungary
  • Italy and Hungary are not connected
  1. Add this knowledge to your KB and CNF (do not eliminate anything!).
  2. Now... simplify the original KB with the additional knowledge before turning it into CNF. What’s the effect?
  3. Use the IDP system to find colorings for the given graph. Do this twice, once with your theory before putting it in CNF and once after. Verify that they admit the same colorings.

Solution

vocabulary V{
	//The possible edges.
	e_ia
	e_ah
	e_ih

	//Whether or not a country is black:
	b_i
	b_a
	b_h
}

theory notInCNF :V{
 // Put your theory BEFORE transformation to CNF here
	e_ia => (b_i <=> ~b_a).
	e_ah => (b_a <=> ~b_h).
	e_ih => (b_i <=> ~b_h).
    
}

theory inCNF:V{
 // Put your theory AFTER transformation to CNF here
    ~e_ia  | ~b_i | ~b_a. 
    ~e_ia  | b_i | b_a. 
    ~e_ah  | ~b_h | ~b_a. 
    ~e_ah  | b_h | b_a. 
    ~e_ih  | ~b_i | ~b_h. 
    ~e_ih  | b_i | b_h. 
}


structure S:V{
	//You can put (partial) assignments here
    //We interpret the edge:
    e_ia = true
    e_ah = true
    e_ih = false
}

procedure main(){
 //Generate the models of both of the theories. We see they indeed agree:
    printmodels(allmodels(notInCNF,S))
    printmodels(allmodels(inCNF,S))
}
