Predicate Logic

Predicate logic VS. Propositional logic

  • Propositional logic reflects only whether a sentence is true or false

  • Predicate logic also looks at the complex conceptual structure within a sentence, e.g. the things, properties, actions and relationships it talks about

  • Predicate logic has greater representational power than propositional logic

    • Basis of logic programming languages, such as Prolog

  • Still: TRUE and FALSE are the only possible values of logical sentences

    • No vagueness and uncertainty (just like in propositional logic)

Predicates and functions

Functions

  • Proposition: 5 > 3
  • Operator : >
  • Function: GreaterThan(5, 3)
  • Domain: R × R (set of real numbers)
  • Codomain: (set of boolean constants)
  • Summary: GreaterThan : R × R → B

Predicates

  • A predicate with specific (instantiated) arguments is a proposition
  • E.g. GreaterThan(3, 4) and GreaterThan(4, 3) are propositions

Predicate domain and codomain

  • Codomain is fixed

    • Set of logical constants (TRUE, FALSE)

  • Domain is can be any set, e.g.

    • people
    • symbols
    • numerical sets (e.g. R, N, I, ...)
    • ...

Predicates and propositions

  • Predicate is a shorthand for a family of propositions, one for each element in the domain
  • E.g. predicate GreaterThanThree : R → B
    • Suppose p represents GreaterThanThree
    • Then p(−2) and p(7) are propositions
  • Suppose:
    • p represents GreaterThanThree
    • q represents LessThanSeven
    • r(x) = p(x) ∧ q(x)
  • What does r mean?
    • r is the property of being between 3 and 7
    • r(x) = p(x) ∧ q(x) = (x > 3) ∧ (x < 7)
    • E.g. r(6.8) = p(6.8) ∧ q(6.8) = TRUE ∧ TRUE = TRUE

Exercise

  • Determine the value of r(−16.5)
  • r(x) = p(x) ∧ q(x) = (x > 3) ∧ (x < 7)

Solution: FALSE

Non-numerical predicates

  • Predicates with non-numerical arguments
  • E.g. Mother : P × P → B
  • Mother(x, y) is TRUE iff x is the mother of y
    • e.g. Mother(Hillary, Chelsea) = TRUE
  • If speaking in general terms with no particular domain (universe) in mind, call it U

Predicates, English and sets

Predicates and English

In natural language, predicates are expressed in many ways

  • E.g. Nouns for family relations as two-argument predicates:

    • Aunt(x, y), Brother(x, y)

  • E.g. Adjectives of nationality as single-argument or two-argument predicates:

    • Chinese(Deng), Nationality(Deng, Chinese), Citizenship(Deng, China)

  • E.g. Adjectives of colour as single-argument or two-argument predicates:

    • Brown(Bear), Colour(Bear, Brown)

  • E.g. Relational adjectives as multi-argument predicates:

  • Big(Ant, Insect), Size(Ant, Insect, Big)

  • E.g. Verbs for geographical relations

    • Borders(India, Pakistan) (= symmetric predicate), RunsThrough(Amazon, Brasil)

The logic provides a framework, it is up to us to decide how to use it

Certain predicates might be more convenient to use than others

Predicate and variable names should be chosen wisely

Truth sets

The truth set of a predicate is the set of values that make the predicate TRUE

Definition: T(p) = {x|p(x) = TRUE}

E.g.: Vowel(x)

  • Vowel('e') = TRUE, Vowel('g') = FALSE
  • T(Vowel) = { 'a', 'e', 'i', 'o', 'u'}

Truth sets of predicates with more than 1 argument consist of ordered pairs

  • E.g.: (5, 3) ∈ T(GreaterThan)
  • (RioGrande, Mexico, USA) ∈ T(Separates)

Exercise

  • Express in predicate logic that “Jumbo is small for an elephant but big for a mammal”

IMPORTANT:

Before you can do such an exercise: decide which predicates to use. Always be very clear about the informal interpretation!

E.g. here we use

Size of arity 3 with the informal interpretation that Size(a, t, s) is true when a is a animal, t is a type of animals (eg Mammal, Elephant) and s is a size.

Solution; Size(Jumbo, Elephant, Small) ∧ Size(Jumbo, Mammal, Big)

Quantifiers

Introduction

So far, predicates were used to make assertions about one thing at a time

Quantifiers allow to make precise, formal assertions about a whole set at once

  • Cf. loops in programming languages

Universal quantifier

  • Symbol: ∀
  • Means: “every”, “for all”
  • E.g. ∀x ∈ U: p(x)
  • For all members x of set U, p(x) is TRUE
  • The set U is called the domain (or universe) of quantification
  • U is an arbitrary set

Example

Express: “Every integer has a square that is greater than or equal to 0”

  • ∀x ∈ I : p(x), where p(x) ↔︎ x has a non-negative square
  • ∀x ∈ I : (x2 ≥ 0)

∀x ∈ U: p(x) ≡ p(x1) ∧ p(x2) ∧ ...

Existential quantifier

  • Symbol: ∃
  • Means: “for some”, “there exists”
  • E.g. ∃x ∈ U : p(x)
    • There exits a member x of set U, for which p(x) is TRUE
    • The set U is called the domain (or universe) of quantification
    • U is an arbitrary set

Example

Express: “There exists an integer between 3 and 7”

  • ∃x ∈ I : p(x), where p(x) ↔︎ x is between 3 and 7
  • ∃x ∈ I : ((x > 3) ∧ (x < 7))

∃x ∈ U : p(x) ≡ p(x1) ∨ p(x2) ∨ ...

Eliminating quantifiers

Universal quantifier

  • For U = {x1 , x2, ...}:

    ∃x ∈ U : p(x) ≡ p(x1) ∨ p(x2) ∨ ...

  • E.g.: For:

    ∃x ∈ I : ((x > 3) ∧ (x < 7))

  • We have

    (3 < 0 < 7) ∨ (3 < 1 < 7) ∨ (3 < 2 < 7) ∨ ...

Negation

  • De Morgan:

    ¬(p(x1) ∧ p(x2)) ≡ ¬p(x1) ∨ ¬p(x2)

    ¬(p(x1) ∨ p(x2)) ≡ ¬p(x1) ∧ ¬p(x2)

  • Generalised:

    ¬(∀x ∈ U : p(x)) ≡ ∃x ∈ U : ¬p(x)

    ¬(∃x ∈ U : p(x)) ≡ ∀x ∈ U : ¬p(x)

Example

  • It is not the case that someone has a car
    • ¬∃x ∈ P : IsCarOwner(x)
  • Someone does not have a car
    • ∃x ∈ P : ¬IsCarOwner(x)
  • No one is without a car
  • ¬∃x ∈ P : ¬IsCarOwner(x)

Exercise

Express in predicate logic. Use negation, whatever quantifiers you need and the one-argument predicate IsCarOwner. Also, state which of your answers are equivalent to each other.

  • It is not the case that everyone has a car
    • ¬∀x ∈ P : IsCarOwner(x)
  • Someone does not have a car
    • ∃x ∈ P : ¬IsCarOwner(x)
  • It is false that everyone is without a car
    • ¬∀x ∈ P : ¬IsCarOwner(x)
    1. ≡ (c)

With U = {a, b, c}, rewrite the following formula without any quantifiers.

  • ¬(∀x ∈ U : p(x)) ≡ ∃x ∈ U : ¬p(x)

Solution: ¬(p(a) ∧ p(b) ∧ p(c)) ≡ ¬p(a) ∨ ¬p(b) ∨ ¬p(c)

Bound and free variables

A variable is either

  • bound by quantification
  • free

E.g.: ∃y ∈ N : (y = x2)

y is bound, x is free

Empty sets

The following equivalences hold no matter what p is:

  • ∀x ∈ ∅ : p(x) ≡ TRUE
  • ∃x ∈ ∅ : p(x) ≡ FALSE

Multiple quantifiers

Introduction

Just as one quantifier can be applied to a predicate with one variable, two or more quantifiers can be applied to a predicate with two or more variables

Examples with universal and existential quantifiers

Definitions:

  • ∀x : ∀y : (Parent(x, y) ∧ Female(x) ↔︎ Mother(x, y))
  • ∀x : ∀y : ∀z : (Mother(y, x) ∧ Father(z, x) ↔︎ Offspring(x, y, z))

Symmetric properties:

  • ∀x : ∀y : (Borders(x, y) ↔︎ Borders(y, x))

Each integer has a square that is also an integer and that is non-negative:

  • ∀x ∈ I : ∃y ∈ N : y = x2

Every living person has a mother, living or not.

  • P is the set of all living people, Q is the set of all people, living or not
  • ∀x ∈ P : ∃y ∈ Q : Mother(y, x)

But every living person has only a single mother?

  • The ∃! quantifier is used to assert that exactly one of something exists:
  • ∀x ∈ P : ∃!y ∈ Q : Mother(y, x)
  • “Every living person has exactly one mother, living or not”

Binding a variable

∃y ∈ N : (y = x2)

  • bound variable y, free variable x
  • “Some non-negative number is the square of x”

∀x ∈ I : ∃y ∈ N : (y = x2)

  • bound variables x and y
  • “All integers have a non-negative square number”

∀x ∈ I : IsPerfectSquare(x) ↔︎ (∃y ∈ N : (y = x2))

  • Definition of the predicate IsPerfectSquare
  • “If a number meets certain conditions, it is a perfect square”

Scope of quantification

The scope of a quantifier extends to the end of the formula, unless otherwise indicated by parentheses.

  • E.g. ∀x ∈ I : IsPerfectSquare(x) ↔︎ (∃y ∈ N : (y = x2))

Order of quantification is important

Order of quantification

If all quantifications are of the same kind (all universal or all existential), the order does not matter

  • ∀x : ∀y : p(x, y) ≡ ∀y : ∀x : p(x, y)
  • ∃x : ∃y : p(x, y) ≡ ∃y : ∃x : p(x, y)

If there are both universal and existential quantifications, the order does matter

  • ∀x ∈ I : ∃y ∈ N : y = x2
    • “Every integer has a square that is a natural number”
  • ∃y ∈ N : ∀x ∈ I : y = x2
    • “There exists a natural number that is the square of every integer” (= FALSE!)
  • ∀x ∈ P : ∃y ∈ Q : Mother(y, x)
    • “Every living person has a mother, living or not”
  • ∃y ∈ Q : ∀x ∈ P : Mother(y, x)
    • “There exists a person, living or not, that is the mother of each living person ” (= FALSE!)

Example

“There is a natural number that is smaller or equal to all other natural numbers”

∃y ∈ N : ∀x ∈ N : (y ≤ x)

C is the network of computers

Direct(x, y) is TRUE if and only if x = y or x is connected directly to y

“There is at least one computer connected directly to all the others”

∃y ∈ N : ∀x ∈ N : Direct(x, y)

The existential quantifier says that there is at least one of something, but sometimes we have more accurate information than that. Sometimes there must be exactly one of something, as noted in the discussion of Example 4.11. Express the fact that there is exactly one integer between 4 and 6 by using an existential quantifier to state that there is at least one such integer and also that if there were two such integers (say x and y), they would have to be equal to each other.

Solution:

∃x ∈ I : 4 < x ∧ x < 6 ∧ ∀y ∈ I : 4 < y ∧ y < 6 ⇒ x = y

Or:

∃x ∈ I : 4 < x ∧ x < 6.

∀x ∈ I, ∀y ∈ I : 4 < y ∧ y < 6 ∧ 4 < x ∧ x < 6 ⇒ x = y.

Remark

The patterns ∃y ∈ S : φ(y) ∧ ψ(y) and ∀x ∈ S : φ(x) ⇒ ψ(x) are very common. The first one says that there is a φ that is also a ψ; the second one says that all φ’s are ψ’s.

Note the difference: ∃ is paired with ∧ and ∀ with ⇒!! If you see a wrong pairing in a solution to an exercise, e.g. ∃x ∈ U : Dog(x) ⇒ Human(x) or ∀x ∈ U : Man(x) ∧ Human(x), then you sentence is probably wrong! (think yourself about what these two nonsense sentences mean)

Exercise:

The quantifier ∃! is sometimes used to assert that exactly one of something exists. That is: ∃!x ∈ I : p(x) means that there is exactly one integer with property p. Use this quantifier to state that:

  • Every living person has exactly one mother, living or not
    • ∀x ∈ P : ∃!y ∈ Q : Mother(y, x)
  • Every integer has exactly one square
    • ∀x ∈ I : ∃!y ∈ N : y = x2

Formulas of predicate logic

Definition: Formulas of Predicate Logic

  • If pred is an n-argument predicate and each of a1, a2, ..., an is either an element of the domain or a variable over the elements, then pred(a1 , a2, ..., an) is a formula
  • If α and β are formulas, then so are (¬α), (α ∧ β) and (α ∨ β)
  • If α is a formula and x is a variable over the elements in the domain, then (∃x : α) and (∀x : α) are formulas
  • Nothing else is a formula

Comments on definition

The definition is kept simple:

  • Parentheses deal with precedence and scope
  • Does not distinguish between free and bound variables
  • All quantification is over the same universe

Difference with propositional logic:

  • Predicate logic deals with domains, variables and quantifiers
  • Elements in a domain may be numbers, people, or whatever is useful
  • Variables in predicate logic range over elements in a domain and are therefore much more flexible than variables in propositional logic, which range over truth values only

Truth Tables

The truth value of a predicate formula can be determined if:

  • the universe is finite (this assumption can be dropped but a more mathematical definition is needed then), and
  • the truth values are known for all atomic formulas without variables (better: we have a structure that interprets all predicate symbols!, and
  • all occurrences of variables are bound by quantification

Algorithm:

  • Eliminate quantifiers and variables (express them in terms of conjunctions and disjunctions)
  • Replace each atomic formula with its truth value
  • Evaluate the resulting proposition

Logic for data structures

Sequences

Sequence:

  • Collection of elements
  • Order of elements is important (↔︎ set)
  • May have repeated elements (↔︎ set)
  • Notation: (a, b, a)

Note:

  • (a, b) and (b, a) are different, whereas {a, b} and {b, a} are the same.
  • The length of the sequence (a, b, a) is 3, whereas the set of elements that appear in this sequence {a, b} has cardinality 2.

Strings

String:

  • Sequence of letters, characters or symbols

General notation for next examples:

  • Sequence of n letters: A = (a1 , a2, ..., an), B = (b1 , b2, ..., bn)
  • Set of first n integers: In : {1, 2, ..., n}
  • x precedes y alphabetically: x ≺ y

Exercises

    1. There is no occurrence of ‘c’ in A
    • ∀i ∈ In : ai ≠ "c"
    1. Some letter appears twice in A
    • ∃i ∈ In : ∃j ≠ i ∈ In : ai = aj
    1. No letter appears twice in A
    • ∀i ∈ In : ∀j ≠ i ∈In : ai ≠ aj
    1. A and B are the same list
    • ∀i ∈ In : ai = bi
    1. The set of letters in A is a subset of the letters in B
    • ∀i ∈ In : ∃j ∈ In : ai = bj
    1. The set of letters in A is the same as in B
    • Idea: A = B ↔︎ (A ⊆ B) ∧ (B ⊆ A)
    • Idea: (∀i ∈ In : ∃j ∈ In : ai = bj) ∧ (∀i ∈ In : ∃j ∈ In : aj = bi)
    1. Each element of A is no later in the alphabet than the corresponding element of B
    • ∀i ∈ In : ai ⪯ bi

Solution:

A lexicographically precedes B if there exists exactly 1 index for which:

  • before this index A and B are equal
  • at this index ai ≺ bi

∃!i ∈ In : (∀j < i : aj = bj ) ∧ ai ≺ bi

Examples

Some letter appears in the same position in A and B:

  • ∃i ∈ In : ai = bi

Within every pair of elements of A, each of the two elements equals the other

  • ∀i ∈ In : ∀j ∈ In : ai = aj

Every element of A is equal to its first element.

  • ∃i ∈ In : ai = a1

Note: ∀i ∈ In : ∀j ∈ In :ai = aj ≡ ∀i ∈ In : ai = a1

A is sorted in strictly alphabetical order

  • ∀i ∈ In-1 : ai ≺ ai+1

Note how comparing with a non-existing element after the last element is avoided!

Graphs

  • Undirected Graph G = (V, E):
    • V: Set of vertices
    • E: Set of edges (unordered pairs of elements of V)
  • E.g.:
    • V = {a, b, c}

Edge(x, y) ↔︎ x and y are connected by an edge (i.e. {x, y} ∈ E)

E.g.:

  • V = {a, b, c, d}
  • Edge(a, b), Edge(b, c), Edge(a, d)

∀x ∈ V : ∃y ∈ V : Edge(x, y)

  • TRUE: all vertices are connected to an other vertex

∃x ∈ V : ∀y ∈ V : x = y ∨ Edge(x, y)

  • FALSE: there is no vertex directly connected to each other vertex

Exercises

    1. Express the fact that Edge is symmetric
    • Edge(x, y) ↔︎ Edge(y, x)
    1. Write a formula stating that for every pair of vertices, there is a path of length 2 connecting these two vertices
    • ∀x ∈ V : ∀y ∈ V : ∃z ∈ V : Edge(x, z) ∧ Edge(z, y)
    1. Give a recursive definition of Path
    • ∀x ∈ V : ∀y ∈ V : Path(x, y) ↔︎ x = y ∨ ∃z ∈ V : Edge(x, z) ∧ Path(z, y)

    1. ∀x ∈ V : ∀y 6= x ∈ V : Edge(x, y)
    • The graph is complete
    1. ∀x ∈ V : ∀y ∈ V : Path(x, y)
    • The graph is connected
    1. ∀x ∈ V : ∃y ∈ V : Edge(x, y)
    • There are no unconnected vertices
    1. ∃x ∈ V : ∀y 6= x ∈ V : Edge(x, y)
    • There exists a vertex that is directly connected to all other vertices

Edge(a, b), Edge(b, c), Edge(b, d)

  • (i): FALSE
  • (ii): TRUE
  • (iii): TRUE
  • (iv): TRUE

Edge(a, b), Edge(b, c), Edge(c, d)

  • (i): FALSE
  • (ii): TRUE
  • (iii): TRUE
  • (iv): FALSE
