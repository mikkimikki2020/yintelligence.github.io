Signal and Image processing

In this series we will introduce and explore some simple applications in signal-and image processing. In the first exercises we will work on a one-dimensional signal, essentially a list of numbers. Then,we will move on to grayscale images, represented by a matrix of numbers （in Python,a nested list）. In the last exercise we will work on colour images, which are also matrices but where each pixel is represented by 3 numbers （in Python this means we will have a list of 3 numbers in a list in a list）.

All files that are needed for this exercise session can be found in the series5_data.zip file.

Question 1: Remove noise from a one-dimensional signal

At the top of your program, write the following line of code:

from pylab import figure,title,subplot,plot,show

  1. Read the values in the file sig1.txt and store them as a list sig1 of floats. Plot the signal by using the following code:

    figure()

show()

    As you can see, this signal is noisy and we will try to remove the noise by filtering the signal. Formally we can write this filtering operation as: \[ y[n]=\sum^{h-1}_{i=0}x[n-i]*f[i] \] In this equation y[n] is the nth sample in the filtered signal, x[n-i] the （n-i）th sample of the signal and f[i] is the ith value in the filter we apply. For this exercise we choose a filter with size h（e.g. h=7） and set f[i] = 1/h. If h=4, then f will look as follows: f = [0.25, 0.25, 0.25, 0.25].

    When the filter is applied, the nth sample of the filtered signal will be equal to the average of the non-filtered signal on location n and the （h-1） non filtered values in front of it. Implement this filter, plot the filtered signal and test your code with different values of h.

  2. Read the information in the file sig2.txt into a list sig2 of floats and plot the signal. As you can see, also here we are dealing with a noisy signal, but the noise is of a different type this time.

    Execute the code from question 1a and store the result in the variable filt1. Write a second filter that uses the median instead of the average and create a filtered signal filt2.

    Question 2: Derivatives

In signal processing and mostly in image processing we often approximate the derivative of a signal by taking the difference between 2 subsequent samples/pixels. Recall that the derivative of a function is defined as: \[ \frac{df(t)}{dt}=\lim_{h→0}\frac{f(t)-f(t-h)}{h} \] The following code generates 5 seconds of a sinus signal sin(2𝜋𝑓∗𝑡) and at the same time the derivative of that signal 2𝜋𝑓∗cos(2𝜋𝑓∗𝑡) with a frequency of 𝑓=2𝐻𝑧. Both signals are continuous, but are sampled at 𝑓𝑠 samples per second. The larger the sample frequency, the more “accurate” we can plot the signal.

Warning: make sure to increment t appropriately otherwise you will end up in an endless loop!!

from math import sin,pi,cos
from pylab import figure,subplot,plot,show,legend

t_end = 5
fs = 100
f = 2
sig = []
dsig = []
t_sig = []
t = 0
while t <= t_end:
    x = sin(2*pi*f*t)
    dx =2*pi*f*cos(2*pi*f*t)
    sig.append(x)
    dsig.append(dx)
    t_sig.append(t)

Calculate the derivative of the signal by filtering the signal with the filter d = [+1,-1] in the same way as in the previous exercise. The filtered sample at location n is equal to the original sample at index n minus the original sample at index （n-1）.This part will be the numerator of the formula for the derivative. The h in the denominator is the time between two consecutive samples and is equal to 1/𝑓𝑠.

Plot the result by using the following code and observe how the value of fs affects the “precision” of the result:

figure()

Question 3: Grayscale images

Put the following imports on the top of your code:

from pylab import figure,imread,imshow,show

Read the coins.png image using the imread function:

I = imread(‘coins.png’)

This image has dimensions 300×246 pixels and contains only gray values （so no colour!）. These gray values are normally stored by means of 8 bits and can assume values between 0 and 255. However,for png images these 8 bit values are re-scaled to float and will accept values between 0 and 1. These pixels are stored in a row-like fashion in a nested list.

  • len(I) should return 246 as our image contains 246 rows.
  • len(I[0]) should return 300 as each row in the image contains 300 pixels.
  • I[0][0] is the pixel in the top left of the image.

The image can be shown with the imshow function, by mentioning that the image has to be rendered with grayscale values:

imshow(I, cmap=’gray’)
show()

We will write an “edge detector”. An edge can be defined as the locations in an image where the grayscale values suddenly change significantly. That change can be characterized by using the concept of derivatives. For a two-dimensional function f(x,y), like our image, the derivative is the equivalent of the gradient: \[ \vec{\nabla}f = (\frac{\partial f}{\partial x},\frac{\partial f}{\partial y}) \] In the previous exercise we saw how to calculate the derivative of one-dimensional signals.

Write a program that can recognize the edges in an image by executing the following steps:

Hint: To quickly create a matrix of zeros with h rows and w columns you can use the following piece of code:

zeros= [[0 for j in range(w)] for i in range(h)]

  1. Calculate the derivative in the x-direction by calculating the derivative for each row and store this in a new image Ix.

  2. Now do the same for each column to produce Iy （the derivative in the y-direction）.

  3. You should now have two images Ix and Iy. The gradient for a pixel with coordinated (i,j) is now the vector: \[ \vec{\nabla}f =(I_x,I_y) \] Calculate for all pixels: \[ |\vec{\nabla}f|^2=I_{x}^{2}+I_{y}^{2} \] and store the result in a new image I_grad.

  4. Make a new image I_edges that contains a 0 for each pixel for which I_grad is smaller than a certain threshold of 0.05 and 1 for pixels that have a value higher than 0.05.

RGB Images

An RGB colour image can be interpreted as a matrix in three dimensions. The first two dimensions are the height and the width of the image. The third dimension is the one that determines the colour of the image: each pixel in the image has a value between 0-255 for RED, a value between 0 and 255 for GREEN and a value between 0 and 255 for BLUE. Each pixel can thus have more than 16.000.000 （256×256×256） different colours.

  3. Define a threshold value that will control how much a pixel in the original image can deviate from a target colour in order to be selected.

  4. Create a new image in which is shown how much each pixel deviates from the target colour. To achieve this, open a new grayscale image of the same size as the original image:

    newim= [[0 for j in range(w)] for i in range(h)]

    In order to calculate the pixel values of that new image we use the following rule: the closer the R, G and B values of the original pixel are to the target value, the lighter the colour will be in the corresponding pixel in the new image. A white pixel would mean that there is a perfect match with the target colour.
